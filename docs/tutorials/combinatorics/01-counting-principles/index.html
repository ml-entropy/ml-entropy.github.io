01. Counting Principles

Before we can compute probabilities, we must learn to count. Not the counting of kindergarten—but the systematic art of enumerating possibilities without listing them. We develop the addition rule, the multiplication rule, the pigeonhole principle, and the stars-and-bars technique, then show why these ideas are the bedrock of probability, information theory, and algorithm design.

Why Counting?

The most fundamental definition of probability is:

$$P(\text{event}) = \frac{|\text{favorable outcomes}|}{|\text{total outcomes}|}$$

This formula is deceptively simple. To use it, you need two numbers: the count of favorable outcomes and the count of total outcomes. For small problems, you can list everything. But for real-world problems, the number of possibilities grows astronomically fast.

  • Probability: What is the chance of being dealt a flush in poker? You need to count all possible 5-card hands (2,598,960) and all flush hands (5,148). Both require counting techniques.
  • Entropy: Shannon's entropy $H = -\sum p_i \log p_i$ measures information content. The number of possible messages of length $n$ from an alphabet of size $k$ is $k^n$—a counting problem that leads directly to information-theoretic bounds.
  • Coding theory: How many distinct binary strings of length $n$ have exactly $k$ ones? The answer is $\binom{n}{k}$, and this directly determines the rate of error-correcting codes.
  • Machine learning: The number of possible decision boundaries, the size of the hypothesis space, the VC dimension—all require combinatorial reasoning.

Analogy. Think of counting principles as the grammar of discrete mathematics. Just as you cannot write coherent sentences without grammar, you cannot reason about probabilities, information, or algorithms without knowing how to count. The rules are few, but their applications are vast.

The Counting-Probability Bridge

Every probability problem in a finite sample space reduces to a counting problem. If you can count, you can compute probabilities. If you can't count, no amount of probability theory will save you. This tutorial builds the counting machinery; the Probability Foundations tutorial puts it to work.

The Addition Rule (Rule of Sum)

The simplest counting principle: if you have two groups of things with no overlap, the total count is the sum of the individual counts. This is so intuitive that it barely seems like a "rule"—yet it's the foundation upon which all more sophisticated techniques are built.

The Addition Rule

If event $A$ can occur in $m$ ways and event $B$ can occur in $n$ ways, and $A$ and $B$ are mutually exclusive (they cannot both happen), then:

$$|A \cup B| = |A| + |B| = m + n$$

Generalized form. For $k$ mutually exclusive events $A_1, A_2, \ldots, A_k$:

$$|A_1 \cup A_2 \cup \cdots \cup A_k| = |A_1| + |A_2| + \cdots + |A_k|$$

Example: Restaurant menu. A restaurant offers 8 appetizers, 12 main courses, and 6 desserts. If you want to order exactly one item (either an appetizer, or a main course, or a dessert), how many choices do you have?

The three categories are mutually exclusive (you can't order a single item that is simultaneously an appetizer and a dessert). By the addition rule:

$$\text{Total choices} = 8 + 12 + 6 = 26$$
When Events Overlap

If $A$ and $B$ are not mutually exclusive, the addition rule overcounts. The correction is the inclusion-exclusion principle: $|A \cup B| = |A| + |B| - |A \cap B|$. We cover this in full generality in Tutorial 05.

The Multiplication Rule (Rule of Product)

The multiplication rule handles sequential or independent choices. If a process consists of multiple stages, and the number of options at each stage doesn't depend on what was chosen before, the total count is the product of the options at each stage.

The Multiplication Rule

If a procedure consists of $k$ sequential stages, where:

  • Stage 1 can be performed in $n_1$ ways,
  • Stage 2 can be performed in $n_2$ ways (regardless of stage 1 choice),
  • $\ldots$
  • Stage $k$ can be performed in $n_k$ ways,

then the total number of ways to complete the entire procedure is: $$n_1 \times n_2 \times \cdots \times n_k$$

Why multiplication? Each choice at stage 1 "branches" into $n_2$ sub-choices at stage 2, which each branch into $n_3$ sub-choices, and so on. The total number of paths through the tree is the product of all branching factors.

Example: License Plates

A license plate consists of 3 letters (A-Z) followed by 4 digits (0-9). How many distinct license plates are possible?

Counting License Plates
1
Identify the stages. There are 7 stages: choose letter 1, letter 2, letter 3, digit 1, digit 2, digit 3, digit 4.
2
Count options at each stage. Each letter position has 26 choices, each digit position has 10 choices.
3
Apply the multiplication rule. $$\underbrace{26 \times 26 \times 26}_{\text{letters}} \times \underbrace{10 \times 10 \times 10 \times 10}_{\text{digits}} = 26^3 \times 10^4 = 17{,}576 \times 10{,}000 = 175{,}760{,}000$$

Example: Passwords

An 8-character password uses lowercase letters (26), uppercase letters (26), digits (10), and special characters (10)—totaling 72 possible characters per position. With repetition allowed:

$$72^8 = 722{,}204{,}136{,}308{,}736 \approx 7.2 \times 10^{14}$$

That's over 700 trillion possibilities. This is why brute-force attacks on even moderately complex passwords are infeasible—a direct consequence of the multiplication rule creating exponential growth.

The Exponential Explosion

The multiplication rule is the source of exponential growth in combinatorics. When $k$ stages each have $n$ options, the total is $n^k$. This grows far faster than any polynomial. With just $n = 2$ (binary choices), $k = 256$ stages yield $2^{256} \approx 1.16 \times 10^{77}$ possibilities—roughly the number of atoms in the observable universe. This exponential explosion is both a curse (it makes exhaustive search impossible) and a blessing (it's what makes cryptography secure).

Counting with Constraints

Real counting problems rarely fit cleanly into a single application of the addition or multiplication rule. Most problems involve constraints—conditions that restrict which outcomes are valid. Two fundamental strategies handle constraints:

Strategy 1: Direct counting with case analysis

Break the problem into mutually exclusive cases, count each case separately, then add them up (the addition rule). This works well when the cases are natural and few.

Example. How many 3-digit numbers (100–999) have at least one digit equal to 5?

Direct counting is messy here (the digit 5 could appear in position 1, 2, 3, or multiple positions, and the cases overlap). This motivates the second strategy.

Strategy 2: Complementary counting

Complementary Counting

Instead of counting what you want, count what you don't want and subtract: $$|\text{desired}| = |\text{total}| - |\text{undesired}|$$ This is particularly powerful when the "undesired" set is easier to count than the "desired" set.

Returning to our example: 3-digit numbers with at least one 5.

Complementary Counting Example
1
Count total 3-digit numbers. The hundreds digit has 9 options (1–9), the tens and units each have 10 options (0–9): $$|\text{total}| = 9 \times 10 \times 10 = 900$$
2
Count 3-digit numbers with NO fives. The hundreds digit has 8 options (1–9 minus 5), the tens and units each have 9 options (0–9 minus 5): $$|\text{no fives}| = 8 \times 9 \times 9 = 648$$
3
Subtract. $$|\text{at least one 5}| = 900 - 648 = 252$$

This strategy generalizes powerfully. It is the conceptual basis of the inclusion-exclusion principle (Tutorial 05) and appears throughout probability theory: $P(A) = 1 - P(A^c)$.

The Pigeonhole Principle

Some of the deepest results in mathematics follow from an absurdly simple observation about counting.

The Pigeonhole Principle

If $n + 1$ objects are placed into $n$ boxes, then at least one box contains two or more objects.

Generalized form: If $n$ objects are placed into $k$ boxes, then at least one box contains $\lceil n/k \rceil$ or more objects.

Proof. Suppose, for contradiction, that every box contains at most one object. Then the total number of objects is at most $n$ (one per box). But we have $n + 1$ objects—contradiction. Therefore at least one box must contain at least two objects. $\blacksquare$

The power of this principle lies not in its proof (which is trivial) but in its applications. The key is identifying the right "objects" and "boxes."

Example: The Birthday Problem Setup

If there are 367 people in a room, at least two share a birthday. Why? There are at most 366 possible birthdays (including Feb 29). With 367 people (objects) and 366 birthdays (boxes), the pigeonhole principle guarantees a collision.

The more interesting question is: how many people do you need for a >50% probability of a shared birthday? That's the famous Birthday Problem, and the surprising answer (just 23!) requires the counting techniques from this tutorial combined with probability.

Example: Hash Collisions

A hash function maps inputs to a fixed-size output (e.g., SHA-256 produces 256-bit hashes). If you have $2^{256} + 1$ distinct inputs, you are guaranteed a collision (two inputs with the same hash). In practice, the Birthday Paradox means collisions become likely much sooner: after about $2^{128}$ inputs.

Pigeonhole in Machine Learning

The pigeonhole principle appears in surprising places in ML. For example: if a neural network with 32-bit floating-point weights has $N$ parameters, it can represent at most $(2^{32})^N = 2^{32N}$ distinct functions. If the hypothesis space is larger than this, some functions are indistinguishable to the network. Similarly, any lossless compression scheme must, by the pigeonhole principle, expand some inputs—not all data can be compressed.

Stars and Bars

How do you distribute identical objects into distinct bins? This arises constantly: distributing budget across departments, assigning tasks to servers, counting the number of solutions to equations with non-negative integer constraints.

Stars and Bars Theorem

The number of ways to distribute $n$ identical objects into $k$ distinct bins (where each bin can hold 0 or more objects) is:

$$\binom{n + k - 1}{k - 1}$$

Equivalently, this counts the number of solutions to $x_1 + x_2 + \cdots + x_k = n$ in non-negative integers.

Deriving the Stars and Bars Formula
1
Represent objects as stars. Think of the $n$ objects as $n$ identical stars in a row: $\star \star \star \cdots \star$.
2
Represent bins as dividers. We need $k - 1$ bars (dividers) to separate the stars into $k$ groups. For example, with $n = 5$ and $k = 3$: $$\star \star \mid \star \mid \star \star$$ means $x_1 = 2$, $x_2 = 1$, $x_3 = 2$.
3
Count the arrangements. We have $n + k - 1$ total symbols ($n$ stars and $k - 1$ bars). We need to choose which $k - 1$ of these positions will be bars. The rest are stars. This gives: $$\binom{n + k - 1}{k - 1} = \binom{n + k - 1}{n}$$

Example: Distributing Identical Candies

How many ways can you give 10 identical candies to 4 children (each child may receive 0 or more candies)?

$$\binom{10 + 4 - 1}{4 - 1} = \binom{13}{3} = \frac{13!}{3! \cdot 10!} = \frac{13 \times 12 \times 11}{3 \times 2 \times 1} = 286$$

Variant: At Least One Per Bin

If each bin must receive at least one object, pre-place one object in each bin. This reduces the problem to distributing $n - k$ objects into $k$ bins with no restrictions:

$$\binom{n - k + k - 1}{k - 1} = \binom{n - 1}{k - 1}$$
Stars and Bars in Probability

The stars-and-bars formula gives the size of sample spaces for many discrete distributions. For example, the number of ways a multinomial outcome $(x_1, \ldots, x_k)$ with $x_1 + \cdots + x_k = n$ can occur is exactly a stars-and-bars count. This connects directly to the Distributions tutorials.

Connection to Probability

We have now assembled a toolkit: the addition rule, multiplication rule, complementary counting, the pigeonhole principle, and stars and bars. Let us see how these connect to probability.

The classical definition of probability for equally likely outcomes is:

$$P(\text{event}) = \frac{|\text{favorable outcomes}|}{|\text{total outcomes}|}$$

Every technique we have learned helps us compute the numerator, the denominator, or both. Here is a concrete demonstration.

Example: At Least One Repeated Digit

A 4-digit PIN (0000–9999) is chosen uniformly at random. What is the probability that at least one digit repeats?

Probability via Complementary Counting
1
Total outcomes. By the multiplication rule, the total number of 4-digit PINs is $10^4 = 10{,}000$.
2
Favorable outcomes (complement). Count PINs with no repeats. The first digit has 10 options, the second has 9 (can't repeat the first), the third has 8, the fourth has 7: $$|\text{no repeats}| = 10 \times 9 \times 8 \times 7 = 5{,}040$$
3
Compute the probability. $$P(\text{at least one repeat}) = 1 - \frac{5{,}040}{10{,}000} = 1 - 0.504 = 0.496$$

Nearly 50%! This is a miniature version of the Birthday Problem. Notice how we combined the multiplication rule (to count total and complement outcomes) with complementary counting (to handle "at least one")—the same pattern that solves problems throughout probability and statistics.

The Bigger Picture

Counting principles are not merely prerequisites for probability—they are probability, for finite sample spaces. Here is the dependency chain:

  • Counting gives us $|S|$ (sample space size) and $|A|$ (event size)
  • Probability is defined as $P(A) = |A|/|S|$ (for uniform distributions)
  • Entropy $H = \log_2 |S|$ (for uniform distributions) measures the "counting complexity" of the sample space
  • Information theory bounds on compression and communication all trace back to counting arguments
Looking Ahead

In the next tutorials, we formalize and extend these ideas. Permutations handle ordered arrangements (where the multiplication rule applies with decreasing options). Combinations handle unordered selections (where we correct for overcounting by dividing). Together with the principles from this tutorial, they form the complete foundation for discrete probability.

Factorial, Permutations, and Combinations from Scratch

import math
from functools import reduce

# --- Factorial ---
def factorial(n):
    """Compute n! from the definition."""
    if n < 0:
        raise ValueError("Factorial is not defined for negative numbers")
    if n == 0 or n == 1:
        return 1
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

# Verify against math.factorial
for n in range(10):
    assert factorial(n) == math.factorial(n)
    print(f"{n}! = {factorial(n)}")

# --- Permutations: P(n, r) = n! / (n-r)! ---
def permutations(n, r):
    """Count ordered arrangements of r items from n."""
    if r > n or r < 0:
        return 0
    # More efficient than n! / (n-r)!:
    result = 1
    for i in range(n, n - r, -1):
        result *= i
    return result

print(f"\nP(10, 3) = {permutations(10, 3)}")   # 720
print(f"P(26, 3) = {permutations(26, 3)}")     # 15600 (3-letter "words")

# --- Combinations: C(n, r) = n! / (r! * (n-r)!) ---
def combinations(n, r):
    """Count unordered selections of r items from n."""
    if r > n or r < 0:
        return 0
    # Use symmetry: C(n, r) = C(n, n-r) to minimize computation
    r = min(r, n - r)
    result = 1
    for i in range(r):
        result = result * (n - i) // (i + 1)
    return result

print(f"\nC(10, 3) = {combinations(10, 3)}")   # 120
print(f"C(52, 5) = {combinations(52, 5)}")     # 2598960 (poker hands)
print(f"C(13, 3) = {combinations(13, 3)}")     # 286 (stars & bars example)

Complementary Counting: 3-Digit Numbers with at Least One 5

# Method 1: Brute force (enumerate and check)
count_brute = sum(1 for n in range(100, 1000) if '5' in str(n))

# Method 2: Complementary counting (no enumeration needed)
total = 900           # 3-digit numbers: 100 to 999
no_fives = 8 * 9 * 9  # hundreds: {1..9}\{5}, tens: {0..9}\{5}, units: same
count_complement = total - no_fives

print(f"Brute force:  {count_brute}")
print(f"Complement:   {count_complement}")
assert count_brute == count_complement  # Both give 252

Stars and Bars: General Implementation

def stars_and_bars(n, k):
    """
    Count ways to distribute n identical objects into k distinct bins.
    Each bin may hold 0 or more objects.

    Formula: C(n + k - 1, k - 1)
    """
    return combinations(n + k - 1, k - 1)

def stars_and_bars_positive(n, k):
    """
    Count ways to distribute n identical objects into k distinct bins.
    Each bin must hold at least 1 object. Requires n >= k.

    Formula: C(n - 1, k - 1)
    """
    if n < k:
        return 0
    return combinations(n - 1, k - 1)

# Examples
print("Stars and Bars examples:")
print(f"  5 objects into 3 bins (>=0 each): {stars_and_bars(5, 3)}")  # 21
print(f"  10 candies to 4 kids (>=0 each):  {stars_and_bars(10, 4)}")  # 286
print(f"  10 candies to 4 kids (>=1 each):  {stars_and_bars_positive(10, 4)}")  # 84

# Enumerate all solutions to x1 + x2 + x3 = 5, xi >= 0
def enumerate_stars_bars(n, k):
    """Generate all solutions to x1 + x2 + ... + xk = n, xi >= 0."""
    if k == 1:
        yield (n,)
        return
    for x in range(n + 1):
        for rest in enumerate_stars_bars(n - x, k - 1):
            yield (x,) + rest

solutions = list(enumerate_stars_bars(5, 3))
print(f"\nAll solutions to x1+x2+x3=5: {len(solutions)} total")
for s in solutions[:10]:
    print(f"  {s[0]} + {s[1]} + {s[2]} = {sum(s)}")
print(f"  ... ({len(solutions)} total)")

Pigeonhole Principle: Birthday Simulation

import random

def birthday_collision_probability(n_people, n_days=365, n_simulations=100_000):
    """
    Estimate the probability that at least two people share a birthday
    in a group of n_people, via Monte Carlo simulation.
    """
    collisions = 0
    for _ in range(n_simulations):
        birthdays = [random.randint(1, n_days) for _ in range(n_people)]
        if len(set(birthdays)) < n_people:  # duplicate found
            collisions += 1
    return collisions / n_simulations

def birthday_collision_exact(n_people, n_days=365):
    """
    Exact probability using complementary counting + multiplication rule.
    P(collision) = 1 - P(no collision)
    P(no collision) = (365/365) * (364/365) * ... * ((365-n+1)/365)
    """
    if n_people > n_days:
        return 1.0
    p_no_collision = 1.0
    for i in range(n_people):
        p_no_collision *= (n_days - i) / n_days
    return 1.0 - p_no_collision

# Find the crossover point (>50% probability)
print("Birthday Problem:")
print(f"{'People':>8} {'Exact':>10} {'Simulated':>10}")
print("-" * 30)
for n in [5, 10, 15, 20, 23, 25, 30, 40, 50, 70]:
    exact = birthday_collision_exact(n)
    simulated = birthday_collision_probability(n)
    marker = " <-- >50%!" if 0.49 < exact < 0.55 else ""
    print(f"{n:>8} {exact:>10.4f} {simulated:>10.4f}{marker}")

# Pigeonhole guarantee
print(f"\n366 people: P = {birthday_collision_exact(366):.4f}")
print("367 people: P = 1.0000 (pigeonhole guarantee)")

Counting for Probability: PIN Repeat Analysis

# Probability that a random 4-digit PIN has at least one repeated digit

# Exact computation
total_pins = 10 ** 4
pins_no_repeat = 10 * 9 * 8 * 7  # multiplication rule with decreasing choices
p_repeat = 1 - pins_no_repeat / total_pins

print(f"Total PINs:          {total_pins:,}")
print(f"PINs without repeat: {pins_no_repeat:,}")
print(f"PINs with repeat:    {total_pins - pins_no_repeat:,}")
print(f"P(at least 1 repeat): {p_repeat:.4f}")

# Generalize: n-digit PINs
print(f"\n{'Digits':>6} {'P(repeat)':>12}")
print("-" * 20)
for n_digits in range(1, 11):
    total = 10 ** n_digits
    if n_digits <= 10:
        no_repeat = 1
        for i in range(n_digits):
            no_repeat *= (10 - i)
        p = 1 - no_repeat / total
    else:
        p = 1.0  # pigeonhole: must repeat with >10 digits
    print(f"{n_digits:>6} {p:>12.6f}")

Exercises & Progress

Test your understanding of counting principles, from basic enumeration to advanced techniques.

Easy 1. Restaurant Combos

A restaurant offers 5 appetizers, 8 main courses, and 4 desserts.

  1. If you order exactly one item (appetizer, main, or dessert), how many choices do you have?
  2. If you order a full meal (one appetizer, one main course, and one dessert), how many different meals are possible?
  3. If you order a full meal but refuse to order the fish main course when you've chosen the soup appetizer (1 specific pairing is forbidden), how many meals are possible?

Hint: Part (a) uses the addition rule. Part (b) uses the multiplication rule. Part (c) uses complementary counting: total meals minus forbidden meals.

Easy 2. Pigeonhole Applications

Use the pigeonhole principle to prove the following:

  1. In any group of 13 people, at least two were born in the same month.
  2. If you have 5 points inside a unit square (1×1), at least two of them are within distance $\frac{\sqrt{2}}{2}$ of each other. Hint: Divide the square into 4 smaller squares.
  3. Among any 11 integers, at least two have the same remainder when divided by 10.

For each: Clearly state what the "objects" and "boxes" are.

Medium 3. License Plates with Constraints

A license plate has the format: Letter-Letter-Digit-Digit-Digit-Letter (e.g., AB123C).

  1. How many such plates are possible with no restrictions?
  2. How many plates have no repeated letters? (Digits may repeat.)
  3. How many plates are palindromic (read the same forwards and backwards, treating each character position independently)? For a 6-character plate $c_1 c_2 c_3 c_4 c_5 c_6$, palindromic means $c_1 = c_6$, $c_2 = c_5$, $c_3 = c_4$.
  4. What fraction of all plates are palindromic?

Hint: For part (c), note that $c_1$ and $c_6$ must be letters and equal, $c_2$ and $c_5$ must be a letter and digit respectively—so check whether a palindrome is even possible given the format constraints.

Medium 4. Stars and Bars Applications

Solve each using the stars-and-bars technique:

  1. How many solutions does $x_1 + x_2 + x_3 + x_4 = 20$ have in non-negative integers?
  2. How many solutions does $x_1 + x_2 + x_3 = 15$ have with each $x_i \geq 2$?
  3. A vending machine accepts only quarters (25¢). In how many ways can 8 people collectively insert exactly 12 quarters? (Each person inserts 0 or more quarters.)
  4. How many monomials of degree 5 in 4 variables ($x_1^{a_1} x_2^{a_2} x_3^{a_3} x_4^{a_4}$ with $a_1 + a_2 + a_3 + a_4 = 5$) exist?

Hint: For part (b), substitute $y_i = x_i - 2$ so that $y_i \geq 0$ and the equation becomes $y_1 + y_2 + y_3 = 9$.

Hard 5. Counting Surjections

A surjection (onto function) from a set $A$ with $|A| = n$ to a set $B$ with $|B| = k$ (where $n \geq k$) is a function where every element of $B$ is mapped to by at least one element of $A$.

  1. How many total functions are there from $A$ to $B$? (No restriction.)
  2. Using inclusion-exclusion, show that the number of surjections is: $$S(n, k) = \sum_{j=0}^{k} (-1)^j \binom{k}{j} (k - j)^n$$ Hint: Let $A_i$ be the set of functions that miss element $b_i \in B$. Then non-surjections are $|A_1 \cup A_2 \cup \cdots \cup A_k|$. Apply inclusion-exclusion.
  3. Compute $S(4, 3)$: the number of surjections from a 4-element set onto a 3-element set. Verify by expanding the formula.
  4. Explain the connection to Stirling numbers of the second kind: $S(n, k) = k! \cdot S_2(n, k)$. Why the factor of $k!$?

Why this matters: Surjections count the number of ways to partition a dataset into exactly $k$ non-empty clusters—a fundamental quantity in clustering algorithms and information theory (alphabet reduction).
