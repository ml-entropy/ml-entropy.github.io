04. Matrix Calculus for ML

Neural networks are compositions of matrix operations. To train them, we need derivatives with respect to vectors and matrices—the language of backpropagation. This tutorial builds that language from scratch.

Why Matrix Calculus?

A modern neural network can have millions—or billions—of parameters. Those parameters are organized into weight matrices and bias vectors. Training the network means computing how the loss changes when any single entry of any weight matrix changes, and then nudging every entry in the direction that reduces the loss. Doing this one parameter at a time would be hopelessly slow. Matrix calculus gives us a way to write these derivatives compactly and compute them efficiently.

In Tutorial 01 we learned derivatives of scalar functions. In Tutorial 02 we extended to functions of several variables, giving us the gradient. Now we go further: the input can be a vector or a matrix, and the output can also be a vector or matrix. The derivative object becomes a matrix (the Jacobian) or even a higher-order tensor. Understanding how these objects compose under the chain rule is what makes backpropagation possible.

The Payoff

Once you know matrix calculus, you can derive the gradient of any neural network architecture on paper. You will never again treat backpropagation as a black box.

Layout Conventions

Before we write a single derivative, we must address a persistent source of confusion: there are two competing conventions for how to arrange partial derivatives into a matrix, and different textbooks use different ones without always saying which. The two conventions are called numerator layout and denominator layout.

The difference boils down to this: when you differentiate a column vector $\mathbf{y} \in \mathbb{R}^m$ with respect to a column vector $\mathbf{x} \in \mathbb{R}^n$, do you get an $m \times n$ matrix (numerator layout) or an $n \times m$ matrix (denominator layout)? We use denominator layout, which is the convention most common in machine learning and optimization. Under this convention, the gradient of a scalar is a column vector that "lives in the same space" as $\mathbf{x}$.

Denominator Layout Convention

The gradient of a scalar $f$ with respect to a column vector $\mathbf{x} \in \mathbb{R}^n$ is a column vector: $$\nabla_{\mathbf{x}} f = \frac{\partial f}{\partial \mathbf{x}} = \begin{pmatrix} \frac{\partial f}{\partial x_1} \\ \vdots \\ \frac{\partial f}{\partial x_n} \end{pmatrix}$$

The Jacobian of $\mathbf{y} \in \mathbb{R}^m$ with respect to $\mathbf{x} \in \mathbb{R}^n$ is the $m \times n$ matrix whose $(i,j)$ entry is $\partial y_i / \partial x_j$.

Why does the convention matter? Because the chain rule involves multiplying Jacobians, and whether you write $J_1 J_2$ or $J_2^T J_1^T$ depends on which layout you chose. If you ever get confused by a transpose that "shouldn't be there," check whether the source uses a different layout convention.

Convention Alert

Wikipedia's matrix calculus article uses numerator layout by default. Many ML papers and textbooks (Bishop, Goodfellow) use denominator layout. Always check which convention a source uses before copying formulas.

The Jacobian Matrix

When we have a vector-valued function $\mathbf{y} = f(\mathbf{x})$ mapping $\mathbb{R}^n \to \mathbb{R}^m$, the derivative is no longer a single number or even a single vector. It is an entire matrix of partial derivatives called the Jacobian.

The Jacobian Matrix

For $\mathbf{y} = f(\mathbf{x})$ with $\mathbf{y} \in \mathbb{R}^m$, $\mathbf{x} \in \mathbb{R}^n$: $$J = \frac{\partial \mathbf{y}}{\partial \mathbf{x}} = \begin{pmatrix} \frac{\partial y_1}{\partial x_1} & \cdots & \frac{\partial y_1}{\partial x_n} \\ \vdots & \ddots & \vdots \\ \frac{\partial y_m}{\partial x_1} & \cdots & \frac{\partial y_m}{\partial x_n} \end{pmatrix} \in \mathbb{R}^{m \times n}$$

Geometric interpretation: The Jacobian is the best linear approximation to $f$ near a point. Just as the scalar derivative gives us $f(x + h) \approx f(x) + f'(x)h$, the Jacobian gives us:

$$f(\mathbf{x} + \mathbf{h}) \approx f(\mathbf{x}) + J \mathbf{h}$$

Each row of $J$ tells you how one output component changes as you perturb the inputs. Each column tells you how all outputs respond to perturbing one particular input. In a neural network, the Jacobian of a layer captures how that layer's outputs respond to small changes in its inputs.

Worked Example: A 2D → 3D Function

Let $f(x_1, x_2) = \begin{pmatrix} x_1^2 + x_2 \\ x_1 x_2 \\ e^{x_1} \end{pmatrix}$. The Jacobian is $3 \times 2$:

Computing the Jacobian
1
Row 1: $\frac{\partial}{\partial x_1}(x_1^2 + x_2) = 2x_1$, $\frac{\partial}{\partial x_2}(x_1^2 + x_2) = 1$
2
Row 2: $\frac{\partial}{\partial x_1}(x_1 x_2) = x_2$, $\frac{\partial}{\partial x_2}(x_1 x_2) = x_1$
3
Row 3: $\frac{\partial}{\partial x_1}(e^{x_1}) = e^{x_1}$, $\frac{\partial}{\partial x_2}(e^{x_1}) = 0$
$$J = \begin{pmatrix} 2x_1 & 1 \\ x_2 & x_1 \\ e^{x_1} & 0 \end{pmatrix}$$

At the point $(x_1, x_2) = (1, 3)$: $J = \begin{pmatrix} 2 & 1 \\ 3 & 1 \\ e & 0 \end{pmatrix}$. This tells us, for instance, that a small perturbation $\delta x_1$ changes $y_1$ by about $2\delta x_1$, $y_2$ by about $3\delta x_1$, and $y_3$ by about $e \cdot \delta x_1$.

Essential Matrix Calculus Identities

The following identities appear over and over in machine learning. Rather than re-derive them each time, it is worth memorizing them. But you should also understand why each one is true—we prove the most important ones in the next section.

Scalar-by-Vector Identities
1
$\frac{\partial}{\partial \mathbf{x}}(\mathbf{a}^T \mathbf{x}) = \mathbf{a}$ — the gradient of a linear function is the coefficient vector
2
$\frac{\partial}{\partial \mathbf{x}}(\mathbf{x}^T \mathbf{x}) = 2\mathbf{x}$ — the gradient of squared norm, analogous to $(x^2)' = 2x$
3
$\frac{\partial}{\partial \mathbf{x}}(\mathbf{x}^T A \mathbf{x}) = (A + A^T)\mathbf{x}$ — reduces to $2A\mathbf{x}$ when $A$ is symmetric
Vector-by-Vector Identities
4
$\frac{\partial}{\partial \mathbf{x}}(A\mathbf{x}) = A$ — the Jacobian of a linear map is the matrix itself
5
$\frac{\partial}{\partial \mathbf{x}}(A\mathbf{x} + \mathbf{b}) = A$ — constant shifts disappear, just like $(x + c)' = 1$
Matrix-by-Matrix Identities (Trace Derivatives)
6
$\frac{\partial}{\partial X}\text{tr}(AX) = A^T$
7
$\frac{\partial}{\partial X}\text{tr}(X^T A) = A$
8
$\frac{\partial}{\partial X}\text{tr}(X^T A X) = (A + A^T)X$
9
$\frac{\partial}{\partial X}\log \det(X) = X^{-T}$ — appears in Gaussian log-likelihoods

Proving the Key Identities

Let us prove the three most important identities from first principles. Understanding these proofs will let you derive any identity you need on the fly.

Proof: $\frac{\partial}{\partial \mathbf{x}}(\mathbf{a}^T \mathbf{x}) = \mathbf{a}$

Step-by-step
1
Write out the dot product: $\mathbf{a}^T \mathbf{x} = \sum_{i=1}^n a_i x_i$
2
Take the partial derivative with respect to $x_k$: $\frac{\partial}{\partial x_k} \sum_{i=1}^n a_i x_i = a_k$
3
Stack all partials into a vector: $\nabla_{\mathbf{x}}(\mathbf{a}^T \mathbf{x}) = \begin{pmatrix} a_1 \\ \vdots \\ a_n \end{pmatrix} = \mathbf{a}$ ✓

Proof: $\frac{\partial}{\partial \mathbf{x}}(\mathbf{x}^T A \mathbf{x}) = (A + A^T)\mathbf{x}$

The Quadratic Form
1
Expand: $\mathbf{x}^T A \mathbf{x} = \sum_{i} \sum_{j} A_{ij} x_i x_j$
2
Differentiate w.r.t. $x_k$: $\frac{\partial}{\partial x_k} = \sum_j A_{kj} x_j + \sum_i A_{ik} x_i$
3
The first sum is the $k$-th element of $A\mathbf{x}$; the second is the $k$-th element of $A^T \mathbf{x}$
4
Therefore: $\nabla_{\mathbf{x}}(\mathbf{x}^T A \mathbf{x}) = A\mathbf{x} + A^T \mathbf{x} = (A + A^T)\mathbf{x}$ ✓

Why this matters for ML: The least-squares loss $\|\mathbf{y} - X\boldsymbol{\beta}\|^2$ can be rewritten as $\boldsymbol{\beta}^T X^T X \boldsymbol{\beta} - 2\mathbf{y}^T X \boldsymbol{\beta} + \mathbf{y}^T\mathbf{y}$. Setting the gradient to zero using our identities gives the normal equations: $\boldsymbol{\beta} = (X^T X)^{-1} X^T \mathbf{y}$.

The Chain Rule for Vectors

The chain rule is the single most important result in matrix calculus for machine learning. In the scalar case, if $y = f(u)$ and $u = g(x)$, then $\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx}$. The vector generalization replaces multiplication with matrix multiplication of Jacobians.

Vector Chain Rule

If $\mathbf{y} = g(\mathbf{x})$ and $z = f(\mathbf{y})$ (scalar output), then: $$\nabla_{\mathbf{x}} z = J_g^T \nabla_{\mathbf{y}} z$$ where $J_g = \frac{\partial \mathbf{y}}{\partial \mathbf{x}}$ is the Jacobian of $g$. More generally, if both functions are vector-valued: $J_{f \circ g} = J_f \cdot J_g$.

Why $J^T$ and not $J$? Because we are using denominator layout. The gradient $\nabla_{\mathbf{x}} z$ is a column vector, and we need the transpose of the Jacobian to make the dimensions work out: if $J_g$ is $m \times n$ and $\nabla_{\mathbf{y}} z$ is $m \times 1$, then $J_g^T$ is $n \times m$, and $J_g^T \nabla_{\mathbf{y}} z$ is $n \times 1$—exactly the right shape for $\nabla_{\mathbf{x}} z$.

This IS Backpropagation

In a neural network with layers $\mathbf{h}_1 \to \mathbf{h}_2 \to \cdots \to L$, backpropagation computes $\nabla_{\mathbf{h}_k} L = J_k^T \nabla_{\mathbf{h}_{k+1}} L$ at each layer, moving backwards from the loss to the inputs. The "backward pass" is literally a sequence of Jacobian-transpose-vector products.

Worked Example: Two-Layer Composition

Let $\mathbf{h} = \sigma(W\mathbf{x} + \mathbf{b})$ and $L = \frac{1}{2}\|\mathbf{h} - \mathbf{t}\|^2$ where $\sigma$ is applied element-wise.

Backpropagation Through Two Layers
1
$\nabla_{\mathbf{h}} L = \mathbf{h} - \mathbf{t}$ (gradient of squared error)
2
Let $\mathbf{z} = W\mathbf{x} + \mathbf{b}$. The Jacobian of $\sigma$ is diagonal: $J_\sigma = \text{diag}(\sigma'(\mathbf{z}))$
3
$\nabla_{\mathbf{z}} L = J_\sigma^T (\mathbf{h} - \mathbf{t}) = \sigma'(\mathbf{z}) \odot (\mathbf{h} - \mathbf{t})$ (element-wise, since $J_\sigma$ is diagonal and symmetric)
4
$\nabla_{\mathbf{x}} L = W^T \nabla_{\mathbf{z}} L$, and $\frac{\partial L}{\partial W} = \nabla_{\mathbf{z}} L \cdot \mathbf{x}^T$

The Hessian Matrix

The gradient tells us the direction of steepest descent. But how curved is the landscape? Is the loss shaped like a gentle bowl or a narrow ravine? The Hessian matrix—the matrix of second derivatives—answers these questions.

The Hessian

For a scalar function $f: \mathbb{R}^n \to \mathbb{R}$, the Hessian is the $n \times n$ matrix: $$H_{ij} = \frac{\partial^2 f}{\partial x_i \partial x_j}$$ The Hessian is symmetric (by Schwarz's theorem) when second partials are continuous.

The Hessian connects to optimization in a fundamental way. The second-order Taylor expansion is:

$$f(\mathbf{x} + \mathbf{h}) \approx f(\mathbf{x}) + \nabla f^T \mathbf{h} + \frac{1}{2} \mathbf{h}^T H \mathbf{h}$$

The eigenvalues of $H$ determine the curvature along each eigenvector direction. If all eigenvalues are positive, the point is a local minimum. If some are positive and some negative, it is a saddle point—which are extremely common in high-dimensional neural network loss landscapes.

Newton's method uses the Hessian to take optimal steps: $\mathbf{x}_{\text{new}} = \mathbf{x} - H^{-1} \nabla f$. This converges much faster than gradient descent near a minimum, but computing and inverting the Hessian is prohibitively expensive for large networks (an $n \times n$ matrix where $n$ can be millions). This is why we use first-order methods like Adam in practice.

Condition Number and Training Difficulty

The ratio $\lambda_{\max} / \lambda_{\min}$ of the Hessian’s largest to smallest eigenvalue is called the condition number. A high condition number means the loss surface is elongated—steep in some directions, flat in others. This makes gradient descent zigzag and converge slowly. Techniques like batch normalization and learning rate warmup help by improving the conditioning of the loss landscape.

Example: Linear Layer Gradients

The linear (fully connected) layer is the most fundamental building block. It computes $\mathbf{y} = W\mathbf{x} + \mathbf{b}$ where $W \in \mathbb{R}^{m \times n}$, $\mathbf{x} \in \mathbb{R}^n$, and $\mathbf{b} \in \mathbb{R}^m$. During the backward pass, we receive $\nabla_{\mathbf{y}} L$ from the next layer and must compute three gradients.

Gradients for $\mathbf{y} = W\mathbf{x} + \mathbf{b}$
1
Gradient w.r.t. input: The Jacobian of $W\mathbf{x}$ w.r.t. $\mathbf{x}$ is $W$ (identity 4). By the chain rule: $\nabla_{\mathbf{x}} L = W^T \nabla_{\mathbf{y}} L$. This is what we pass to the previous layer.
2
Gradient w.r.t. weights: Since $y_i = \sum_j W_{ij} x_j + b_i$, we have $\frac{\partial L}{\partial W_{ij}} = \frac{\partial L}{\partial y_i} x_j$. In matrix form: $\frac{\partial L}{\partial W} = \nabla_{\mathbf{y}} L \cdot \mathbf{x}^T$ (an outer product).
3
Gradient w.r.t. bias: Since $\frac{\partial y_i}{\partial b_i} = 1$, we get $\nabla_{\mathbf{b}} L = \nabla_{\mathbf{y}} L$. The bias gradient is just the incoming gradient.

Numerical Example

Let $W = \begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$, $\mathbf{x} = \begin{pmatrix} 1 \\ -1 \end{pmatrix}$, $\mathbf{b} = \begin{pmatrix} 0 \\ 0 \end{pmatrix}$, and suppose $\nabla_{\mathbf{y}} L = \begin{pmatrix} 1 \\ 1 \end{pmatrix}$.

  • $\nabla_{\mathbf{x}} L = W^T \begin{pmatrix} 1 \\ 1 \end{pmatrix} = \begin{pmatrix} 1 & 3 \\ 2 & 4 \end{pmatrix}\begin{pmatrix} 1 \\ 1 \end{pmatrix} = \begin{pmatrix} 4 \\ 6 \end{pmatrix}$
  • $\frac{\partial L}{\partial W} = \begin{pmatrix} 1 \\ 1 \end{pmatrix}\begin{pmatrix} 1 & -1 \end{pmatrix} = \begin{pmatrix} 1 & -1 \\ 1 & -1 \end{pmatrix}$
  • $\nabla_{\mathbf{b}} L = \begin{pmatrix} 1 \\ 1 \end{pmatrix}$

Example: Softmax Jacobian

The softmax function converts a vector of real numbers (logits) into a probability distribution. It is used at the output of nearly every classification network. Its Jacobian is surprisingly elegant.

The softmax function is $\sigma(\mathbf{z})_i = \frac{e^{z_i}}{\sum_j e^{z_j}}$. To find the Jacobian, we need $\frac{\partial \sigma_i}{\partial z_j}$ for all $i, j$. There are two cases.

Deriving the Softmax Jacobian
1
When $i = j$: Apply the quotient rule: $\frac{\partial \sigma_i}{\partial z_i} = \frac{e^{z_i} \sum_k e^{z_k} - e^{z_i} e^{z_i}}{(\sum_k e^{z_k})^2} = \sigma_i(1 - \sigma_i)$
2
When $i \neq j$: The numerator $e^{z_i}$ does not depend on $z_j$: $\frac{\partial \sigma_i}{\partial z_j} = \frac{-e^{z_i} e^{z_j}}{(\sum_k e^{z_k})^2} = -\sigma_i \sigma_j$
3
Combine using the Kronecker delta: $\frac{\partial \sigma_i}{\partial z_j} = \sigma_i(\delta_{ij} - \sigma_j)$
4
In matrix form: $J = \text{diag}(\boldsymbol{\sigma}) - \boldsymbol{\sigma}\boldsymbol{\sigma}^T$
Softmax + Cross-Entropy Simplification

When combined with cross-entropy loss $L = -\sum_i y_i \log \sigma_i$, the gradient w.r.t. logits simplifies beautifully to $\nabla_{\mathbf{z}} L = \boldsymbol{\sigma} - \mathbf{y}$ (predicted minus true labels). This cancellation is not a coincidence—it follows from the fact that cross-entropy is the natural loss function for the softmax family (an exponential family result).

Automatic Differentiation

In practice, nobody computes Jacobians by hand for large networks. Instead, frameworks like PyTorch and TensorFlow use automatic differentiation (autodiff). It is important to understand that autodiff is not symbolic differentiation (which produces formula expressions) and not numerical differentiation (which uses finite differences). Autodiff computes exact derivatives by decomposing computations into elementary operations and applying the chain rule systematically.

There are two modes of autodiff:

  • Forward mode: Propagates derivatives forward through the computation graph. Efficient when the number of inputs is small. Computes one column of the Jacobian per pass.
  • Reverse mode: Propagates derivatives backward from the output. Efficient when the number of outputs is small (e.g., a scalar loss). Computes one row of the Jacobian per pass. This is exactly backpropagation.
Why Reverse Mode Wins for ML

Neural networks have one scalar output (the loss) but millions of inputs (the parameters). Reverse mode computes the entire gradient $\nabla_\theta L$ in a single backward pass, regardless of how many parameters there are. Forward mode would require one pass per parameter. This is why backpropagation (= reverse-mode autodiff) is the algorithm of choice.

Information-Theoretic Connections

Matrix calculus connects to information theory through the Fisher information matrix, which measures how much information the data carries about the parameters.

Fisher Information Matrix

For a parametric model $p(\mathbf{x}; \boldsymbol{\theta})$, the Fisher information matrix is: $$\mathcal{I}(\boldsymbol{\theta}) = E\left[\nabla_{\boldsymbol{\theta}} \log p(\mathbf{x}; \boldsymbol{\theta}) \cdot \nabla_{\boldsymbol{\theta}} \log p(\mathbf{x}; \boldsymbol{\theta})^T\right]$$ Under regularity conditions, this equals the negative expected Hessian of the log-likelihood: $\mathcal{I}(\boldsymbol{\theta}) = -E\left[\nabla^2_{\boldsymbol{\theta}} \log p(\mathbf{x}; \boldsymbol{\theta})\right]$.

The Fisher information matrix is the Hessian of the KL divergence between $p(\mathbf{x}; \boldsymbol{\theta})$ and $p(\mathbf{x}; \boldsymbol{\theta} + \delta\boldsymbol{\theta})$ at $\delta\boldsymbol{\theta} = 0$. This makes it a Riemannian metric on the space of distributions: it tells you the "statistical distance" between nearby parameter values.

The natural gradient $\tilde{\nabla} L = \mathcal{I}^{-1} \nabla L$ corrects for this geometry. While ordinary gradient descent treats all parameter directions equally, natural gradient descent takes larger steps in directions where the distribution changes slowly (flat Fisher metric) and smaller steps where it changes rapidly (steep Fisher metric). Methods like Adam approximate aspects of natural gradient descent, which partly explains their empirical success.

Common Misconceptions

Misconception 1: "The gradient is a row vector"

Under numerator layout, the gradient is indeed a row vector. Under denominator layout (our convention), it is a column vector. This is a convention choice, not a mathematical fact. Always check which convention a source uses.

Misconception 2: "I can just use shapes to figure out transposes"

Shape-matching is a useful sanity check, but it can mislead when you mix conventions. A more reliable approach: go back to index notation ($\frac{\partial y_i}{\partial x_j}$), work out the formula, then reassemble into matrix form.

Misconception 3: "The Hessian is too expensive to ever use"

While the full $n \times n$ Hessian is impractical for large networks, Hessian-vector products can be computed efficiently (in the same time as a gradient). Methods like conjugate gradient and L-BFGS exploit this to approximate second-order optimization without ever forming the full Hessian.

Misconception 4: "Backprop is specific to neural networks"

Backpropagation is just reverse-mode automatic differentiation applied to any computational graph. It works for any differentiable program—physics simulations, graphics renderers, probabilistic programs. Neural networks are one application, not the algorithm itself.

Problem Set Structure

30 problems in three bands: 10 easy (vocabulary and setup), 10 medium (derivations and applications), and 10 hard (proofs and multi-step problems).

Easy Problems

  1. What is the shape of the Jacobian of $f: \mathbb{R}^5 \to \mathbb{R}^3$?
  2. Compute $\nabla_{\mathbf{x}}(\mathbf{a}^T \mathbf{x})$ where $\mathbf{a} = (2, -1, 3)^T$.
  3. What is $\frac{\partial}{\partial \mathbf{x}}(\mathbf{x}^T \mathbf{x})$ when $\mathbf{x} \in \mathbb{R}^4$?
  4. State the chain rule for the composition $z = f(g(\mathbf{x}))$ where $z$ is scalar.
  5. What is the Jacobian of the identity function $f(\mathbf{x}) = \mathbf{x}$?
  6. If $W$ is $3 \times 5$, what is the shape of $\nabla_{\mathbf{x}} L$ in a linear layer?
  7. What does it mean for the Hessian to be positive definite at a point?
  8. Name the two layout conventions in matrix calculus.
  9. Why is reverse-mode autodiff preferred over forward-mode for training neural networks?
  10. What is the softmax Jacobian entry $\frac{\partial \sigma_i}{\partial z_j}$ when $i = j$?

Middle Problems

  1. Prove that $\frac{\partial}{\partial \mathbf{x}}(\mathbf{x}^T \mathbf{x}) = 2\mathbf{x}$ from the definition.
  2. Compute the Jacobian of $f(x_1, x_2) = (x_1 + x_2, x_1 x_2, x_1^2 - x_2^2)^T$.
  3. Derive the gradient of least-squares loss $L = \|\mathbf{y} - X\boldsymbol{\beta}\|^2$ w.r.t. $\boldsymbol{\beta}$.
  4. For $f(\mathbf{x}) = \sigma(W_2 \sigma(W_1 \mathbf{x}))$ (two-layer network), write out the chain of Jacobians needed for backprop.
  5. Show that the Hessian of $f(\mathbf{x}) = \mathbf{x}^T A \mathbf{x}$ is $A + A^T$.
  6. Compute $\frac{\partial}{\partial X} \text{tr}(X^T X)$ using the trace derivative rules.
  7. Derive the gradient of the softmax cross-entropy loss $L = -\sum_i y_i \log \sigma_i(\mathbf{z})$ w.r.t. $\mathbf{z}$.
  8. Explain why element-wise activation functions have diagonal Jacobians.
  9. For a linear layer, verify that $\frac{\partial L}{\partial W}$ has the same shape as $W$.
  10. Explain the difference between the Fisher information matrix and the Hessian of the loss.

Hard Problems

  1. Derive $\frac{\partial}{\partial X} \log \det(X)$ for symmetric positive definite $X$.
  2. Prove the softmax Jacobian formula $J = \text{diag}(\boldsymbol{\sigma}) - \boldsymbol{\sigma}\boldsymbol{\sigma}^T$ from scratch.
  3. Show that the Fisher information matrix equals the negative expected Hessian of the log-likelihood under regularity conditions.
  4. Derive the natural gradient update for a Gaussian model $p(x; \mu, \sigma^2) = \mathcal{N}(\mu, \sigma^2)$.
  5. For the function $f(X) = \text{tr}(X^T A X B)$, compute $\frac{\partial f}{\partial X}$.
  6. Explain why the condition number of the Hessian affects gradient descent convergence. Relate to the learning rate bound.
  7. Design a computational graph for $L = \log(\sum_i e^{z_i})$ (log-sum-exp) and trace the forward and backward passes, showing how numerical stability is maintained.
  8. Prove that forward-mode and reverse-mode autodiff compute the same Jacobian but with different computational costs. When is forward mode more efficient?
  9. Derive the Gauss-Newton approximation to the Hessian for a nonlinear least-squares problem $L = \frac{1}{2}\|f(\mathbf{x}) - \mathbf{y}\|^2$ and explain why it is always positive semidefinite.
  10. Write a detailed comparison of SGD, Newton's method, and natural gradient descent. For each, state what curvature information it uses and what computational cost it incurs per step.
