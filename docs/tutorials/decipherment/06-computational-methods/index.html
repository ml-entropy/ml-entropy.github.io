Computational Methods in Decipherment

The decipherment stories we have explored so far — Egyptian, cuneiform, Linear B, Maya — were triumphs of human intellect. But as computational power has grown exponentially, a natural question arises: can machines decipher ancient scripts? The answer, emerging from the last two decades of research, is a qualified yes — with important caveats about what "decipherment" means in a computational context and what role human insight still plays.

This chapter surveys the computational approaches to decipherment, from classical frequency analysis through modern neural methods. We will implement key algorithms, analyze their information-theoretic properties, and examine landmark results — particularly the automated decipherment of Ugaritic and the computational re-decipherment of Linear B. We will also explore how large language models and unsupervised translation techniques may enable progress on still-undeciphered scripts.

The central insight connecting all these methods is that decipherment is fundamentally a search problem: find the mapping from unknown signs to known language elements that maximizes some objective function (linguistic plausibility, pattern match, translation quality). The algorithms differ in how they define the search space, the objective function, and the search strategy — but they all operate within this framework.

Computational decipherment

Computational decipherment is the use of algorithms to automatically or semi-automatically determine the mapping between an undeciphered script and a known (or hypothesized) language. The input is a corpus of undeciphered text plus, optionally, a model of the target language. The output is a proposed sign-to-sound (or sign-to-meaning) mapping. Success is measured by the accuracy of the mapping compared to known decipherments, or by the linguistic plausibility of the resulting text. The field sits at the intersection of computational linguistics, cryptography, and machine learning.

Frequency Analysis: The Foundation

The oldest computational approach to decipherment is frequency analysis — the observation that different characters in a writing system appear at different rates, and these rates reflect the underlying language's phonetic or morphological structure. In a simple substitution cipher, the most frequent ciphertext letter likely maps to the most frequent plaintext letter (typically 'e' in English). The same principle applies, with modifications, to ancient scripts.

Frequency analysis was first described by the Arab polymath Al-Kindi in the 9th century CE — making it one of the oldest algorithms in all of computer science. Al-Kindi's insight was that language has statistical invariants: no matter what cipher is used, the frequency distribution of the underlying language shines through.

Frequency Analysis for Script Identification (Python)
import math
from collections import Counter

def frequency_analysis(corpus):
    """
    Compute sign frequencies and entropy for an unknown corpus.
    The frequency distribution alone reveals the script type.
    """
    sign_counts = Counter(corpus)
    total = sum(sign_counts.values())
    n_signs = len(sign_counts)
    
    # Compute probabilities and entropy
    entropy = 0
    for sign, count in sign_counts.most_common():
        p = count / total
        entropy -= p * math.log2(p)
    
    # Frequency ratio: top sign / average frequency
    top_freq = sign_counts.most_common(1)[0][1] / total
    avg_freq = 1 / n_signs
    dominance_ratio = top_freq / avg_freq
    
    print(f"Distinct signs: {n_signs}")
    print(f"Total tokens: {total}")
    print(f"Entropy: {entropy:.2f} bits/sign")
    print(f"Max entropy (uniform): {math.log2(n_signs):.2f} bits/sign")
    print(f"Redundancy: {1 - entropy/math.log2(n_signs):.1%}")
    print(f"Most common sign: {sign_counts.most_common(1)[0]} ({top_freq:.1%})")
    print(f"Dominance ratio: {dominance_ratio:.1f}x")
    
    # Script type diagnostic
    if n_signs < 40:
        script_type = "Alphabet"
    elif n_signs < 100:
        script_type = "Syllabary"
    elif n_signs < 500:
        script_type = "Logosyllabic"
    else:
        script_type = "Logographic"
    
    print(f"\nLikely script type: {script_type}")
    return sign_counts, entropy

Al-Kindi's frequency attack (9th century CE)

Al-Kindi's Manuscript on Deciphering Cryptographic Messages (c. 850 CE) described the first known frequency analysis algorithm: count character frequencies in the ciphertext, sort by frequency, and map to the known frequency ranking of the target language's letters. This works perfectly against simple substitution ciphers, where every instance of a letter is replaced by the same cipher character. Al-Kindi's method is essentially maximum likelihood estimation — find the permutation that maximizes $P(\text{ciphertext} | \text{mapping}, \text{language model})$. Remarkably, this is exactly the same objective function used by modern computational decipherment algorithms — just with more sophisticated language models.

The EM Algorithm for Decipherment

Modern computational decipherment relies heavily on the Expectation-Maximization (EM) algorithm — a general-purpose method for finding maximum-likelihood parameter estimates when some variables are hidden. In the decipherment context, the "hidden variables" are the correct readings of the signs, and the "parameters" are the sign-to-sound mapping.

The EM algorithm alternates between two steps:

  1. E-step (Expectation): Given the current mapping estimate, compute the probability of each possible reading for each sign occurrence in the corpus.
  2. M-step (Maximization): Given these probabilistic readings, update the mapping to maximize the likelihood of the corpus under a language model.
EM Algorithm for Substitution Cipher Decipherment
1
Let $c = c_1 c_2 \cdots c_N$ be the ciphertext (unknown signs) and $\sigma$ be the substitution mapping: $\sigma(c_i) = p_i$ (plaintext). We want to find $\sigma^* = \arg\max_\sigma P(c | \sigma, \text{LM})$, where LM is a language model of the target language.
2
E-step: For each sign $c_i$ and each possible reading $r$, compute: $q_i(r) = P(\sigma(c_i) = r | c, \sigma^{\text{old}}, \text{LM})$. This uses the current mapping estimate and the language model to weight possible readings.
3
M-step: Update the mapping: $\sigma^{\text{new}}(s) = \arg\max_r \sum_{i: c_i = s} q_i(r)$. Each sign is mapped to the reading that best explains all its occurrences, weighted by the E-step probabilities.
4
Convergence: EM guarantees that the likelihood $P(c | \sigma, \text{LM})$ increases (or stays the same) with each iteration. After convergence, $\sigma^*$ is a (locally) optimal decipherment. Multiple random restarts help avoid local optima.
EM Decipherment of a Substitution Cipher (Python)
import math
from collections import Counter, defaultdict
import random

class EMDecipherer:
    """
    EM algorithm for deciphering a simple substitution cipher,
    using character bigram frequencies as a language model.
    """
    
    def __init__(self, cipher_text, language_bigrams):
        self.cipher = cipher_text
        self.bigrams = language_bigrams  # P(char2 | char1)
        self.cipher_chars = sorted(set(cipher_text))
        self.plain_chars = sorted(language_bigrams.keys())
        
        # Initialize with random mapping
        self.mapping = {}
        shuffled = list(self.plain_chars)
        random.shuffle(shuffled)
        for c, p in zip(self.cipher_chars, shuffled):
            self.mapping[c] = p
    
    def score(self):
        """Log-likelihood of current decipherment under bigram LM."""
        text = self.apply_mapping()
        ll = 0
        for i in range(len(text) - 1):
            bigram = (text[i], text[i+1])
            ll += math.log2(self.bigrams.get(bigram, 1e-10))
        return ll
    
    def apply_mapping(self):
        """Apply current mapping to produce candidate plaintext."""
        return ''.join(self.mapping.get(c, c) for c in self.cipher)
    
    def em_step(self):
        """One iteration of EM: try swaps, keep improvements."""
        improved = False
        for i, c1 in enumerate(self.cipher_chars):
            for c2 in self.cipher_chars[i+1:]:
                # Try swapping the mappings
                old_score = self.score()
                self.mapping[c1], self.mapping[c2] = \
                    self.mapping[c2], self.mapping[c1]
                new_score = self.score()
                
                if new_score > old_score:
                    improved = True  # keep swap
                else:
                    # Revert swap
                    self.mapping[c1], self.mapping[c2] = \
                        self.mapping[c2], self.mapping[c1]
        
        return improved
    
    def decipher(self, max_iters=100):
        """Run EM until convergence."""
        for i in range(max_iters):
            if not self.em_step():
                print(f"Converged after {i+1} iterations")
                break
        return self.apply_mapping()

Why EM works for decipherment

The EM algorithm succeeds at decipherment because natural languages have highly non-uniform statistical structure. In English, "th" is a much more common bigram than "qx". Any incorrect mapping will produce implausible bigram sequences, lowering the likelihood score. The algorithm exploits this by searching for the mapping that produces the most "natural-looking" text according to the language model. With a good bigram or trigram model and a ciphertext of 500+ characters, EM can typically achieve >90% character accuracy on simple substitution ciphers — and the same principle scales to ancient scripts.

Landmark: Automated Decipherment of Ugaritic

The most impressive demonstration of computational decipherment was achieved by Regina Barzilay and colleagues at MIT in 2010. They applied a Bayesian decipherment algorithm to Ugaritic — a Semitic language written in a simplified cuneiform alphabet (~30 signs), used at the ancient city of Ugarit (modern Ras Shamra, Syria) around 1300 BCE.

Ugaritic was first deciphered by humans in 1930–32. The MIT team's goal was to see whether a computer could rediscover the correct decipherment automatically, given only the Ugaritic corpus and a model of Hebrew (a related Semitic language). Their algorithm achieved 29 out of 30 correct sign mappings — an extraordinary result.

Why Hebrew works as a bridge for Ugaritic

Ugaritic and Hebrew are both Northwest Semitic languages, separated by only a few centuries of divergence. They share the same triliteral root system, similar morphological patterns, and much core vocabulary. Crucially, the structural similarities (consonant patterns, word shapes) are more important than exact vocabulary matches. The algorithm exploits these structural parallels: it doesn't need to know that Ugaritic mlk means "king" — it only needs to know that the pattern C₁C₂C₃ (where C₁ is a sonorant, C₂ a liquid, C₃ a velar) occurs frequently in both languages. This structural matching is robust to vocabulary divergence, which is why the approach works even for languages separated by centuries.

Neural Approaches: Deep Learning for Decipherment

The latest wave of computational decipherment uses deep learning — particularly sequence-to-sequence models and transformer architectures. These approaches treat decipherment as an unsupervised translation problem: given text in an unknown language, find a mapping to a known language without parallel (bilingual) data.

The key technique is adversarial training: train a "translator" network to convert unknown-script text into known-language text, while simultaneously training a "discriminator" network to distinguish real known-language text from translated text. The translator learns to produce outputs that the discriminator cannot distinguish from real text — which means the translations must be linguistically plausible.

The cognate assumption and its limits

Most computational decipherment algorithms assume that the unknown and known languages are related — sharing vocabulary, morphology, or at least typological features. This is the "cognate assumption": if Ugaritic and Hebrew both have the root mlk for "king," the algorithm can use this shared pattern as an anchor. But for truly isolated languages (like Sumerian or the Indus Valley language, if it is a language isolate), no related language exists. In this case, computational methods must rely on universal linguistic properties — word-length distributions, Zipf's law, syntactic patterns — which are much weaker constraints. This is the fundamental limit of computational decipherment: algorithms need a prior, and the quality of the decipherment depends on the quality of the prior.

Bayesian Framework for Computational Decipherment
1
The decipherment problem in Bayesian terms: find the mapping $\sigma^*$ that maximizes the posterior probability: $\sigma^* = \arg\max_\sigma P(\sigma | \text{corpus}) = \arg\max_\sigma P(\text{corpus} | \sigma) \cdot P(\sigma)$.
2
The likelihood $P(\text{corpus} | \sigma)$ is computed using the language model: apply mapping $\sigma$ to convert the corpus to the target language, then evaluate its probability under the LM. Higher probability → more natural-sounding text → better mapping.
3
The prior $P(\sigma)$ encodes expectations about the mapping. For simple substitution: $P(\sigma) \propto 1$ (uniform over permutations). For logosyllabic scripts: the prior can encode expectations about sign-type distributions (e.g., most signs should be common CV syllables, with fewer rare logograms).
4
The search space for $\sigma$ is typically astronomical ($n!$ for $n$ signs), but EM, MCMC (Markov Chain Monte Carlo), or gradient-based methods can find good solutions efficiently. The key insight: most of the search space can be pruned because linguistically implausible mappings have extremely low likelihood — the language model acts as a powerful filter.

The Future: LLMs and Multimodal Decipherment

Large language models (LLMs) like GPT-4 and Claude represent a new frontier for decipherment. These models have internalized the statistical patterns of dozens of languages, potentially giving them an implicit "prior" over linguistic structure that is far richer than any hand-crafted language model. Several research groups are exploring whether LLMs can assist with undeciphered scripts:

Can LLMs decipher scripts?

Current LLMs cannot directly decipher unknown scripts — they have not been trained on undeciphered corpora, and decipherment requires systematic search that is not well-suited to autoregressive text generation. However, LLMs excel at two supporting tasks: (1) hypothesis generation — suggesting plausible language identifications, morphological patterns, or sign groupings based on partial evidence; and (2) evaluation — assessing whether a proposed decipherment produces linguistically plausible text. The most promising approach combines LLM-based evaluation with traditional search algorithms: use EM or MCMC for the search, with an LLM as the language model that scores candidate decipherments.

Multimodal approaches: reading the pictures

Many undeciphered scripts appear alongside images — the Maya codex illustrations, Indus seal images, Rongorongo tablet carvings. Modern multimodal AI can analyze both text and images simultaneously. For the Indus script, computer vision could identify the animals and objects depicted on seals, providing semantic constraints on the accompanying text (e.g., a seal showing a bull likely contains the word for "bull"). This multimodal decipherment approach mirrors how Knorozov used codex illustrations to verify his Maya readings — but at computational scale.

Evaluating Decipherment Quality (Python)
import math

def evaluate_decipherment(proposed_text, reference_bigrams, reference_words):
    """
    Evaluate a proposed decipherment by measuring how well
    the output matches known language statistics.
    
    Returns a quality score between 0 (random) and 1 (perfect).
    """
    # Metric 1: Bigram perplexity
    log_prob = 0
    n_bigrams = 0
    for i in range(len(proposed_text) - 1):
        bigram = (proposed_text[i], proposed_text[i+1])
        p = reference_bigrams.get(bigram, 1e-8)
        log_prob += math.log2(p)
        n_bigrams += 1
    
    perplexity = 2 ** (-log_prob / max(n_bigrams, 1))
    
    # Metric 2: Word recognition rate
    words = proposed_text.split()
    recognized = sum(1 for w in words if w.lower() in reference_words)
    word_rate = recognized / max(len(words), 1)
    
    # Metric 3: Character entropy (should match target language)
    from collections import Counter
    char_counts = Counter(proposed_text)
    total = sum(char_counts.values())
    entropy = -sum((c/total) * math.log2(c/total) 
                    for c in char_counts.values())
    
    # Combined score
    score = 0.5 * word_rate + 0.3 * (1 / perplexity) + 0.2 * (entropy / 5.0)
    
    print(f"Perplexity: {perplexity:.1f}")
    print(f"Word recognition: {word_rate:.1%}")
    print(f"Character entropy: {entropy:.2f} bits")
    print(f"Overall quality: {score:.3f}")
    
    return score

# A good decipherment: perplexity < 50, word recognition > 80%
# A random mapping: perplexity > 1000, word recognition < 5%
The human-machine partnership

The future of decipherment is not "AI replaces humans" but "AI amplifies humans." Computational methods excel at searching vast combinatorial spaces and detecting statistical patterns too subtle for human perception. But humans excel at hypothesis formation (what language might this be?), contextual reasoning (this seal was found in a temple, so the text likely relates to religion), and cross-disciplinary synthesis (combining archaeological, linguistic, and cultural evidence). The most productive approach is collaborative: AI proposes candidate readings, human experts evaluate and refine them, and the cycle iterates. This is already happening with cuneiform tablet reading, where AI systems pre-process tablet images and suggest sign identifications that human Assyriologists then verify.

Unsupervised machine translation

Unsupervised machine translation (UMT) is the task of translating between two languages using only monolingual corpora — no parallel (bilingual) text. UMT systems learn by: (1) building separate representations of each language's structure, (2) aligning these representations in a shared space using distributional similarity (words that appear in similar contexts in different languages tend to be translations), and (3) training a translator that converts between representations. For decipherment, UMT is the ideal framework: the "source language" is the undeciphered script and the "target language" is a known related language. The alignment step is where decipherment happens — finding the mapping between unknown signs and known language elements.

MCMC Methods for Script Decipherment

Markov Chain Monte Carlo (MCMC) methods have emerged as one of the most powerful computational approaches to script decipherment. The basic idea is elegant: treat the unknown sign-to-sound mapping as a probability distribution, and use MCMC sampling to explore the space of possible mappings, guided by a language model that scores each proposed mapping by how well it produces valid text in a known or hypothesized language.

The approach works as follows. Define the state space as all possible mappings from the undeciphered script's signs to phonemes (or syllables) of a candidate language. At each step, propose a small change to the current mapping (e.g., swap the values assigned to two signs). Score the resulting decipherment using a character-level language model (typically a bigram or trigram model trained on known texts in the candidate language). If the new mapping produces more "language-like" output, accept it; otherwise, accept it with a probability proportional to its relative score (the Metropolis-Hastings criterion). Over many iterations, the chain converges toward mappings that produce valid text.

MCMC Decipherment Framework (Python)
import math, random
from collections import Counter

def build_bigram_model(training_text):
    """Build a character bigram language model."""
    bigrams = Counter()
    unigrams = Counter()
    for i in range(len(training_text) - 1):
        bigrams[training_text[i:i+2]] += 1
        unigrams[training_text[i]] += 1
    
    # Log-probabilities with Laplace smoothing
    vocab = set(training_text)
    V = len(vocab)
    log_probs = {}
    for a in vocab:
        for b in vocab:
            count = bigrams.get(a + b, 0)
            total = unigrams.get(a, 0)
            log_probs[(a, b)] = math.log(
                (count + 1) / (total + V))
    
    return log_probs, vocab

def score_decipherment(cipher_text, mapping, lm):
    """Score a proposed decipherment using language model."""
    log_probs, vocab = lm
    deciphered = ''.join(mapping.get(c, c) for c in cipher_text)
    
    score = 0
    for i in range(len(deciphered) - 1):
        pair = (deciphered[i], deciphered[i+1])
        score += log_probs.get(pair, -15)  # heavy penalty
    
    return score

def mcmc_decipher(cipher_text, lm, n_iterations=10000):
    """
    Use Metropolis-Hastings MCMC to find the best
    sign-to-letter mapping.
    """
    log_probs, vocab = lm
    cipher_chars = list(set(cipher_text))
    plain_chars = list(vocab)
    
    # Random initial mapping
    mapping = dict(zip(cipher_chars,
                       random.sample(plain_chars,
                                     min(len(cipher_chars),
                                         len(plain_chars)))))
    
    best_score = score_decipherment(cipher_text, mapping, lm)
    best_mapping = mapping.copy()
    
    accepted = 0
    for i in range(n_iterations):
        # Propose: swap two random mappings
        new_mapping = mapping.copy()
        a, b = random.sample(cipher_chars, 2)
        new_mapping[a], new_mapping[b] = (
            new_mapping.get(b, b), new_mapping.get(a, a))
        
        new_score = score_decipherment(
            cipher_text, new_mapping, lm)
        
        # Metropolis-Hastings acceptance
        delta = new_score - best_score
        if delta > 0 or random.random() < math.exp(delta):
            mapping = new_mapping
            accepted += 1
            if new_score > best_score:
                best_score = new_score
                best_mapping = new_mapping.copy()
    
    print(f"MCMC: {n_iterations} iterations, "
          f"{accepted} accepted ({100*accepted/n_iterations:.1f}%)")
    return best_mapping, best_score

training = "the quick brown fox jumps over the lazy dog " * 100

The Ugaritic algorithm: Snyder et al. (2010)

The landmark paper by Snyder, Barzilay, and Knight (2010) demonstrated that MCMC-based methods could automatically decipher the Ugaritic script — an ancient alphabetic cuneiform used at the site of Ugarit (Syria, ~1400–1200 BCE). Their algorithm used a bigram language model of Hebrew (a related Semitic language) as the target and searched for the best mapping from Ugaritic signs to Hebrew letters. The system correctly identified 29 of 30 Ugaritic letters with no human intervention — a remarkable achievement. The key insight was that related languages share statistical patterns (bigram frequencies, word length distributions) that constrain the mapping even without a bilingual text. This approach has since been extended to other scripts and represents a genuine paradigm shift: the first time a machine has independently deciphered an ancient writing system.

Transfer Learning for Ancient Scripts

Recent advances in deep learning have opened new possibilities for computational decipherment through transfer learning — the idea that a model trained on one script or language can transfer its knowledge to help with an unfamiliar script. This is particularly promising for undeciphered scripts where direct training data is nonexistent.

The key insight is that writing systems, despite their surface diversity, share deep structural properties: they encode language, and language has universal statistical regularities (Zipf's law, predictable entropy rates, morphological patterns). A neural network trained to recognize these patterns in known scripts can potentially identify them in unknown scripts — serving as a "universal writing system detector" that can classify signs by function (phonogram vs. logogram vs. determinative) and identify structural relationships (which signs are likely to share phonetic features) without knowing the specific phonetic values.

AI limitations in decipherment

Despite the excitement around computational methods, it is important to recognize their limitations. Current AI approaches work best on substitution-like problems: scripts where each sign maps to a fixed phonetic value in a known language. They struggle with logosyllabic systems (where signs can be logograms, syllabograms, or determinatives depending on context), with unknown languages (where no language model can be constructed), and with small corpora (where statistical patterns are unreliable). The Indus script, for example, defeats current computational methods because it is short, the language is unknown, and the script may be logosyllabic. The fundamental limitation is that AI can optimize within a defined search space but cannot make the creative leaps that characterized breakthroughs by Champollion, Ventris, and Knorozov — redefining the problem itself rather than searching within the existing framework.

Neural OCR for Damaged Inscriptions

One of the most immediately practical applications of machine learning to ancient scripts is optical character recognition (OCR) for damaged texts. Ancient inscriptions are rarely in pristine condition: stone monuments are weathered, clay tablets are broken, papyri are fragmentary, and painted texts are faded. Human experts spend enormous time and effort identifying individual signs on damaged surfaces — a process that requires years of training and intimate familiarity with the script's visual conventions.

Deep learning models — particularly convolutional neural networks (CNNs) and vision transformers — can be trained to recognize signs even in degraded conditions. The key advantage of neural OCR is its ability to learn from damaged examples: by training on thousands of instances of each sign at varying levels of damage, the model builds an internal representation of each sign's essential features that is robust to noise, breakage, and weathering. This is analogous to how humans learn to read: we see letters in many fonts, sizes, and handwriting styles, and our visual system extracts the invariant features.

Sign Recognition with Damage Simulation (Python)
import random, math

def simulate_damaged_sign(sign_features, damage_level):
    """
    Simulate damage to a sign's feature vector.
    
    sign_features: list of floats (feature values)
    damage_level: 0.0 (pristine) to 1.0 (destroyed)
    """
    damaged = []
    for f in sign_features:
        if random.random() < damage_level:
            # Feature is damaged: add noise or zero it
            if random.random() < 0.5:
                damaged.append(f + random.gauss(0, 0.5))
            else:
                damaged.append(0.0)  # feature destroyed
        else:
            damaged.append(f + random.gauss(0, 0.05))
    return damaged

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x*x for x in a))
    norm_b = math.sqrt(sum(x*x for x in b))
    if norm_a == 0 or norm_b == 0:
        return 0
    return dot / (norm_a * norm_b)

def recognize_damaged_sign(damaged, reference_signs):
    """
    Identify a damaged sign by comparing to reference
    examples using cosine similarity.
    """
    scores = []
    for name, ref in reference_signs.items():
        sim = cosine_similarity(damaged, ref)
        scores.append((sim, name))
    
    scores.sort(reverse=True)
    return scores[:3]  # top 3 candidates

3D scanning and virtual unwrapping

Advanced imaging technologies are revealing texts that were previously inaccessible. Reflectance Transformation Imaging (RTI) uses multiple light angles to enhance surface details on worn stone inscriptions, often making signs visible that are invisible under normal lighting. 3D scanning with structured light captures the full geometry of clay tablets, allowing scholars to examine them from any angle and under any virtual lighting condition. Most dramatically, virtual unwrapping algorithms can read scrolls and folded documents without physically opening them — a technique that has been applied to carbonized scrolls from Herculaneum (destroyed by Vesuvius in 79 CE) and to sealed clay envelopes from Mesopotamia that were never meant to be opened. These technologies don't decipher scripts, but they reveal texts that would otherwise be lost — expanding the corpus available for analysis.

Case Study: A Full Computational Decipherment Pipeline

To tie together the computational methods discussed in this chapter, let us walk through a complete decipherment pipeline — the sequence of computational steps that would be applied to a newly discovered inscription in an undeciphered script. This pipeline represents the current state of the art in computational epigraphy.

Stage 1: Digitization and sign segmentation. The inscription is photographed under multiple lighting conditions, 3D-scanned if possible, and processed through a neural segmentation model that identifies individual signs and their boundaries. Output: a sequence of sign images.

Stage 2: Sign classification. Each sign image is classified against a trained sign catalogue using a CNN or vision transformer. For known scripts, this achieves ~95% accuracy; for newly discovered scripts, the model clusters similar sign images into types and assigns provisional type IDs. Output: a sequence of sign type IDs.

Stage 3: Structural analysis. Statistical analysis of the sign sequence: frequency distribution, positional preferences, bigram/trigram patterns, and conditional entropy. This stage identifies: (a) whether the system is likely linguistic; (b) approximate script type (alphabetic, syllabic, logosyllabic); (c) probable word boundaries; (d) candidate determinatives (signs with specific positional patterns). Output: a structural profile.

Stage 4: Language hypothesis testing. For each candidate target language, an MCMC decipherment algorithm searches for the optimal sign-to-phoneme mapping, scored by a language model. Multiple candidates are tested in parallel. The language producing the highest-scoring decipherment is selected. Output: a ranked list of (language, mapping, score) triples.

Stage 5: Human validation. The top-scoring decipherment is reviewed by human experts who check: (a) do the decoded words make sense in the proposed language? (b) does the content match the archaeological context? (c) are there internal consistency checks (repeated formulas reading consistently, names matching known historical figures)? Output: validated or rejected decipherment.

The future: foundation models for ancient scripts

The latest frontier in computational epigraphy is the development of foundation models — large neural networks pre-trained on multiple ancient scripts simultaneously, then fine-tuned for specific tasks. The vision: a single model that has "seen" Egyptian hieroglyphs, cuneiform, Maya glyphs, Linear B, and other known scripts, building an internal representation of what writing looks like at an abstract level. Such a model could then be applied to undeciphered scripts, classifying signs by function (phonogram vs. logogram vs. determinative), identifying structural patterns, and even suggesting candidate phonetic values based on similarities to known systems. Early experiments (such as Google DeepMind's work on damaged Akkadian tablets) show promise, but we are still far from a general-purpose "decipherment AI." The fundamental limitation remains: these models can recognize patterns in data, but the creative leaps that characterized Champollion, Ventris, and Knorozov — reframing the problem itself — remain uniquely human.

Key Takeaways: Computational Decipherment

Decipherment is fundamentally a search problem: find the sign-to-sound mapping that maximizes linguistic plausibility under a language model. All algorithms — from Al-Kindi's frequency analysis to neural approaches — share this structure.

The EM algorithm provides a principled framework: alternate between estimating readings (E-step) and updating the mapping (M-step). With a good language model and sufficient corpus, EM achieves >90% accuracy on substitution ciphers.

Landmark result: Barzilay et al. (2010) automatically deciphered Ugaritic using Hebrew as a bridge language, achieving 29/30 correct sign mappings — demonstrating that computational methods can match human-level decipherment.

Neural approaches treat decipherment as unsupervised translation: adversarial training learns a mapping between unknown and known scripts without any bilingual data.

The cognate assumption is both the key enabler and fundamental limit: algorithms need a related language as prior. For true language isolates, only universal linguistic properties are available — much weaker constraints.

LLMs and multimodal AI offer new possibilities: richer priors from massive training data, image-text analysis for illustrated scripts, and human-AI collaborative workflows.

🟢 Easy

🟢1. Bayes rule for decipherment+

Q: State Bayes rule as applied to decipherment: $P(f | \text{corpus}) \propto ?$

$P(f | \text{corpus}) \propto P(\text{corpus} | f) \cdot P(f)$. Likelihood measures fit to linguistic data. Prior encodes constraints (phonetic plausibility, one-to-one mapping).

🟢2. Ugaritic Snyder 2010+

Q: In Snyder et al.'s Ugaritic decipherment, how many of 30 letters were correctly identified?

29 of 30 letters. Single error was a rare letter with insufficient training data. Computational method matched human performance.

🟢3. N-gram language model+

Q: Define an n-gram model and explain why it works for ranking decipherment hypotheses.

N-gram model estimates $P(w_i | w_{i-1}, \ldots, w_{i-n+1})$. For decipherment: hypothesis that produces text with high probability (low perplexity) under the model is more likely correct. Model doesn't need perfect accuracy—just relative ranking.

🟢4. EM algorithm for phonetics+

Q: Describe E-step and M-step of EM for phonetic assignment.

E-step: Compute expected alignments between signs and phonemes given current model. M-step: Update probabilities to maximize expected alignment likelihood. Iterate until convergence.

🟢5. Hill climbing vs simulated annealing+

Q: Distinguish hill climbing and simulated annealing for decipherment search.

Hill climbing: Always accept improvements, reject worsening. Fast but stuck at local optima. Simulated annealing: Accept improvements always, worsening with probability $\exp(\Delta L/T)$, $T$ decreasing. Slower but better global optima.

🟢6. Ugaritic sign space+

Q: How many possible mappings exist for Ugaritic (~30 signs)? Is this tractable?

30! ≈ 2.65 × 10^32 possible mappings. Large but tractable with modern search: constraint propagation, heuristics, and parallel search reduce effective space.

🟢7. Maya sign space+

Q: Maya has ~800 signs. How many possible mappings? Is pure search feasible?

800! ≈ 10^1567 mappings. Astronomically infeasible. But with linguistic constraints (syllabic structure, known language family), effective space collapses dramatically—making it computable.

🟢8. Edit distance for cognates+

Q: What is edit distance? How is it used to detect cognates?

Edit distance measures minimum character substitutions/insertions/deletions to transform one word into another. Cognates (related words) typically have low edit distance despite sound changes and evolution. Threshold-based matching identifies likely cognate pairs.

🟢9. Neural machine translation for scripts+

Q: Describe Luo et al.'s approach: treating decipherment as machine translation. What are advantages and risks?

Advantages: Neural networks learn complex patterns automatically; no manual feature engineering. Risks: Requires large aligned corpus (rare for ancient languages); black-box model (hard to validate); may find spurious patterns.

🟢10. Perplexity and hypothesis ranking+

Q: Define perplexity. Lower or higher perplexity = better hypothesis?

Perplexity $= 2^{-\frac{1}{N}\sum \log P(w_i)}$. Lower perplexity = higher probability under language model = more likely correct decipherment. Low perplexity indicates hypothesis fits linguistic patterns well.

🟡 Medium

🟡11. Language model for dead languages+

Q: Ancient Egyptian training data is limited. List sources for building a language model: modern descendants, related languages, etc. Which is most reliable?

Sources: (1) Modern Coptic (descendant). (2) Related languages (Semitic cognates). (3) Reconstructed phonotactics/morphology (comparative linguistics). (4) Mixture model combining all. Most reliable: modern descendants (fewest sound changes) + multiple related languages (cross-validate).

🟡12. Constraint-based refinement+

Q: In decipherment search, constraints eliminate impossible candidates. Name three constraints: phonetic, structural, linguistic.

Phonetic: Common phonemes must map to frequent signs (typology). Structural: Each script sign maps to exactly one phoneme (one-to-one constraint). Linguistic: Resulting text must show plausible morphology, bigram patterns, word lengths for the target language. Combined, these reduce search from 800! to tractable space.

🟡13. Sound change inference+

Q: Explain sound change inference. Give an example: if you observe p→f in high-vowel contexts, how does this help cognate matching?

Inference: Learn systematic phonological shifts between language stages. Example: p→f in high-vowel contexts means ancient /p/ + high vowel becomes modern /f/ + high vowel. Cognate matching: Ancient word "pa-ti" might match modern "fa-ti". Learned rules predict correspondences, improving match accuracy.

🟡14. Why algorithms beat humans at search+

Q: Can an algorithm explore 10^32 mappings faster than a human expert can explore 1000? Explain the tradeoff.

Algorithm: Explores systematically, pruning dead branches via constraints, testing thousands/second. Human: Explores heuristically, using intuition to prioritize promising directions, but covers <1000 in lifetime. Algorithm finds global optimum fast; human finds local insights deeply. Optimal: Humans generate constraints/hypotheses; algorithms search exhaustively within constraints.

🟡15. Genetic algorithms for decipherment+

Q: Describe a genetic algorithm for decipherment: population, fitness, mutation, crossover.

Population: 100+ candidate sign-to-phoneme mappings. Fitness: Perplexity under language model (lower=better). Mutation: Swap two phoneme assignments randomly. Crossover: Combine top-10 mappings by taking phoneme assignments from each parent. Evolve for 1000 generations. Avoids local optima via population diversity.

🟡16. Proto-Elamite computational pipeline+

Q: Design a computational pipeline for Proto-Elamite: corpus preprocessing, numerical anchors, language model, search.

Pipeline: (1) Preprocess: Segment tablets via OCR/ML, extract glyph sequences. (2) Anchors: Mark known numerals (fixed values). (3) Language model: Train on reconstructed Elamite+related languages. (4) Semantic context: Identify inventory words (sheep, barley) via predictable collocations. (5) Search: EM or simulated annealing, optimizing linguistic plausibility × numeric consistency. (6) Validate: Cross-check against historical records, related scripts.

🟡17. Citizen science + computation+

Q: Explain how citizen science and computational methods complement each other in decipherment (e.g., Proto-Elamite projects).

Citizen science: Humans segment tablets, classify glyphs, identify rare sequences (pattern recognition at scale). Computation: Algorithms learn from human classifications, predict glyph values, identify high-confidence correspondences. Feedback loop: Humans validate computational predictions, refine model. Neither alone suffices; together they scale human insight.

🟡18. MCMC convergence diagnostics+

Q: In MCMC decipherment, how do you determine whether the Markov chain has converged to the correct mapping? Describe at least two diagnostic methods.

(1) Multiple independent chains: Run 5+ chains from random initial mappings and check whether they converge to the same solution. If all chains agree, convergence is likely; disagreement indicates insufficient exploration. (2) Log-likelihood trace plot: Plot the language model score over iterations. Convergence shows a plateau — the score stabilizes. If still increasing, the chain hasn't converged. (3) Gelman-Rubin statistic (R̂): Compare within-chain and between-chain variance. R̂ ≈ 1.0 indicates convergence. (4) Effective sample size: Measure autocorrelation; low effective samples mean the chain is exploring slowly. For decipherment, the strongest signal is agreement between independent chains — if 5 random starts produce the same mapping, confidence is very high.

🟡19. Evaluating candidate languages+

Q: When using computational methods on an undeciphered script, you must choose a candidate target language. How can you systematically evaluate which candidate language produces the best decipherment? What metrics would you use?

(1) Perplexity: For each candidate language, train a language model and run the decipherment algorithm. The language that produces the lowest perplexity (highest probability) text is the best candidate. (2) Word boundary alignment: Check whether decoded text produces valid words at boundaries already identified in the script. Higher word-hit rate = better language match. (3) Morphological consistency: If the script shows inflectional patterns, check whether decoded endings match the candidate language's morphology. (4) Geographic/temporal plausibility: Weight candidates by archaeological context — a Bronze Age Cretan script is more likely to encode an Aegean language than a Central Asian one. Combined scoring: weighted sum of perplexity, word hits, and morphological fit, filtered by historical plausibility.

🟡20. Transfer learning feasibility+

Q: A neural network trained to classify sign functions (logogram vs. syllabogram vs. determinative) on Egyptian hieroglyphs is applied to an undeciphered script. Under what conditions would this transfer learning approach work? When would it fail?

Works when: (1) Both scripts share structural properties (mixed logosyllabic system with determinatives). (2) The network learned general features like frequency distributions, positional patterns, and co-occurrence constraints rather than script-specific visual features. (3) Sufficient undeciphered corpus for the network to detect statistical patterns. Fails when: (1) The undeciphered script has fundamentally different structure (purely alphabetic, or purely logographic). (2) The corpus is too small for reliable statistics. (3) The network overfit to Egyptian-specific features (visual glyph shapes rather than distributional patterns). (4) The undeciphered script may not be linguistic at all (e.g., Indus hypothesis). Best approach: train on multiple known scripts (Egyptian + Maya + Cuneiform) to learn truly universal writing system features.

🔴 Hard

🔴21. Complete Bayesian formulation+

Q: Formulate Bayesian decipherment: Prior P(f) (phonetic plausibility), Likelihood P(C|f) (linguistic fit), and Posterior. Describe sampling or optimization.

Prior: P(f) ∝ \prod phonetic_plausibility(sign_i → phoneme_i). Likelihood: P(C|f) = Language_model(decode(corpus, f)). Posterior: ∝ Prior × Likelihood. Optimization: MAP (maximum a posteriori) via hill climbing, EM, or MCMC sampling. EM: E-step estimates expected alignments; M-step maximizes likelihood. MCMC: Sample from posterior distribution directly.

🔴22. Local optima problem+

Q: Hill climbing can get stuck at local optima. Propose three methods to escape: simulated annealing, genetic algorithms, and multi-start. Which is best for decipherment and why?

(1) Simulated annealing: Accept worse solutions with decreasing probability as T→0. (2) Genetic algorithms: Maintain population, breed good solutions, mutation explores. (3) Multi-start: Run hill climbing from 100 random initializations, return best. For decipherment: Multi-start + simulated annealing best—multi-start is fast; simulated annealing fine-tunes. Genetic algorithms also strong but slower. Optimal: Hybrid (multi-start → simulated annealing on best).

🔴23. Neural NMT vs symbolic search+

Q: Compare neural machine translation (NMT) and symbolic constraint satisfaction for decipherment. Advantages/disadvantages of each?

NMT: (+) Learns complex patterns automatically. (-) Black-box (hard to debug). (-) Requires large aligned corpus (rare). (-) May overfit or find spurious patterns. Symbolic: (+) Interpretable—can trace decisions. (+) Exploits explicit linguistic knowledge. (-) Manual feature engineering tedious. (-) May miss subtle patterns. For ancient scripts: Symbolic better (corpus small, linguistic knowledge available); NMT useful supplement (post-processing, ranking).

🔴24. Morphological constraints+

Q: Design a constraint that exploits known morphology (prefixes, suffixes) to improve decipherment of an unknown script.

Constraint: (1) Identify recurring sequences in corpus (likely morphemes). (2) Model expected frequency of prefixes/suffixes in target language. (3) Penalize mappings that violate morphological patterns (e.g., require vowel sequences for affixes, consonant clusters for roots). (4) During search, prefer mappings where decoded text shows morphology consistent with target language typology. Example: If target language uses agent-marking suffix "-er", decoded text should show this pattern; mappings that produce it score higher.

🔴25. Ugaritic success: Why 29/30?+

Q: Snyder et al. achieved 29/30 on Ugaritic. Analyze: (1) Why did they miss one letter? (2) Could perfect accuracy be achieved? (3) What properties of Ugaritic made success possible?

(1) Rare letter with insufficient EM training data—statistical underdetermined. (2) Perfect accuracy requires either more data or human knowledge (cognate verification). (3) Success factors: (a) Small sign inventory (~30). (b) Abjad (consonants only, simpler than full syllabary). (c) Known target language (Hebrew). (d) Large cognate corpus (~1000 pairs). (e) 1000-year language closeness (minimal sound changes). These conditions are ideal—rarely met for ancient scripts.

🔴26. Can algorithms discover language?+

Q: For Linear A (transliterable but untranslatable), could neural networks discover the underlying language automatically without human guidance?

Unlikely. NNs learn mappings (unknown script → known language), not language discovery. Without a target language as training signal, NN would find spurious correlations. Thought experiment: Train on gibberish+English. NN would find some mapping (random noise → English patterns). No way to validate discovered language is real. Language discovery requires external information (cognate language, bilingual text) or human verification. Computation alone cannot create information.

🔴27. Rare sign problem+

Q: In a corpus of 10,000 glyphs with 800 signs, many signs appear < 10 times. How does this affect EM algorithm? Propose a solution.

Problem: EM requires sufficient data per parameter. Rare signs have too few occurrences—EM estimates are unreliable (high variance). Solution: (1) Hierarchical Bayes—place priors on rare-sign parameters (regularization). (2) Transfer learning—use related languages to seed priors. (3) Manual annotation—linguists mark rare signs, reduce uncertainty. (4) Constraint satisfaction—encode phonetic/typological constraints as priors. Combined: Hierarchical Bayes + linguistic constraints dramatically improve rare-sign estimates.

🔴28. Cognate matching across deep time+

Q: Proto-Elamite (3100 BCE) and Elamite proper (2000 BCE) are 1000 years apart. Design an algorithm to identify cognates despite massive sound changes. What are limits?

Algorithm: (1) Reconstruct sound change patterns using historical linguistics (comparative method). (2) Apply patterns backward from Elamite proper to hypothetical Proto-Elamite forms. (3) Sequence alignment (longest common substring, edit distance with weighted penalties). (4) Score matches by confidence. (5) Combine with semantic context (inventory words). Limits: (a) Sound changes unpredictable beyond ~1000 years (lexical replacement accelerates). (b) Semantic drift—meanings change. (c) Borrowing obscures cognacy. (d) No guarantee of monogenesis. Mitigation: Use multiple related languages; phylogenetic reconstruction; accept partial matches; expert validation.

🔴29. Why algorithms cannot cure information deficit+

Q: Prove or argue: No algorithm, regardless of sophistication, can decipher Indus if the language is unknown. Is there a mathematical formalism for this impossibility?

Proof sketch: Decipherment requires maximizing P(f|corpus). By Bayes: P(f|corpus) ∝ P(corpus|f) × P(f). Without known language, P(corpus|f) cannot be computed (no language model). Multiple mappings f have equal likelihood (symmetric problem). Information-theoretic formalism: Mutual information I(Script; Language) = 0 if language unknown. Any algorithm attempting decipherment without external language information faces exponential degeneracy. No search algorithm can eliminate symmetry that isn't broken by data. Conclusion: Information deficit is insurmountable—no algorithm bridges it.

🔴30. Future: AI + ancient scripts+

Q: Speculate: What role will large language models (LLMs), multimodal AI, and citizen science play in deciphering undeciphered scripts in the next 10 years?

LLMs: (1) Generate cognate candidates from multilingual training. (2) Rank hypotheses via linguistic plausibility. (3) Synthesize domain knowledge (linguistics, history, archaeology). Multimodal AI: (1) Vision models recognize glyph variants across tablets/inscriptions. (2) Link visual features to linguistic structure. Citizen Science: (1) Humans segment, annotate; AI learns from labels. (2) Feedback loop: AI predictions → human validation → refined model. Success scenario: Proto-Elamite decipherment via hybrid approach (cognate discovery + administrative context + ML-guided search). Unlikely: Indus (language unknown), Rongorongo (corpus too small). Most likely: Computational breakthrough on Proto-Elamite within 5 years; Linear A remains blocked by language barrier.
