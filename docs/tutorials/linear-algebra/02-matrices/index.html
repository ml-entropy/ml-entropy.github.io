Linear Transformations

Imagine you have a rubber sheet pinned at the origin. You can stretch it, rotate it, shear it, or reflect it — but you cannot tear it, fold it, or move the pin. Every deformation you can perform under these constraints is a linear transformation. This geometric picture is the single most important intuition in all of linear algebra: matrices are not spreadsheets of numbers — they are actions that reshape space.

Formally, a function $T: \mathbb{R}^n \to \mathbb{R}^m$ is a linear transformation if it satisfies two properties: it must respect addition and it must respect scalar multiplication. These two requirements, taken together, guarantee that the transformation preserves the fundamental structure of the vector space. Any combination of vectors that holds before the transformation still holds after it.

Linearity Conditions

A function $T: \mathbb{R}^n \to \mathbb{R}^m$ is linear if for all vectors $\mathbf{u}, \mathbf{v} \in \mathbb{R}^n$ and all scalars $c \in \mathbb{R}$: $$T(\mathbf{u} + \mathbf{v}) = T(\mathbf{u}) + T(\mathbf{v}) \quad \text{(additivity)}$$ $$T(c\mathbf{v}) = cT(\mathbf{v}) \quad \text{(homogeneity)}$$ Equivalently, these can be combined into a single condition: $T(c_1\mathbf{u} + c_2\mathbf{v}) = c_1 T(\mathbf{u}) + c_2 T(\mathbf{v})$.

What do these conditions mean geometrically? Additivity says that if you first add two vectors and then transform, you get the same result as transforming each separately and then adding. Homogeneity says that stretching a vector by a factor $c$ before or after the transformation gives the same result. Together they ensure: straight lines stay straight, the origin stays fixed, and parallel lines remain parallel. A function like $T(\mathbf{x}) = \mathbf{x} + \begin{pmatrix} 1 \\ 0 \end{pmatrix}$ (a translation) is not linear — it moves the origin.

Let's verify linearity with a concrete example. Consider the function $T\begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} 2x + y \\ 3y \end{pmatrix}$. Take $\mathbf{u} = \begin{pmatrix} 1 \\ 2 \end{pmatrix}$ and $\mathbf{v} = \begin{pmatrix} 3 \\ -1 \end{pmatrix}$. Then $T(\mathbf{u}) = \begin{pmatrix} 4 \\ 6 \end{pmatrix}$, $T(\mathbf{v}) = \begin{pmatrix} 5 \\ -3 \end{pmatrix}$, and $T(\mathbf{u} + \mathbf{v}) = T\begin{pmatrix} 4 \\ 1 \end{pmatrix} = \begin{pmatrix} 9 \\ 3 \end{pmatrix} = T(\mathbf{u}) + T(\mathbf{v})$. It checks out — this function is linear, and we'll soon see it corresponds to the matrix $\begin{pmatrix} 2 & 1 \\ 0 & 3 \end{pmatrix}$.

Why Linearity Matters in ML

Every layer in a neural network applies a linear transformation (matrix multiply) followed by a non-linear activation. The linear part is what the network learns — the weight matrices. Understanding linear transformations is understanding what a neural network does between activations.

Why Matrices Matter for ML

If you've ever trained a neural network, you've manipulated millions of matrices without perhaps realizing it. The single most common operation in deep learning — the forward pass through a fully-connected layer — is a matrix-vector multiplication: $\mathbf{y} = W\mathbf{x} + \mathbf{b}$. The weight matrix $W$ encodes everything that layer has learned. When we say a model has "175 billion parameters," we are overwhelmingly talking about entries of weight matrices.

Weight matrices appear everywhere. In a simple feedforward network with layer sizes $[784, 256, 128, 10]$ (an MNIST classifier), there are three weight matrices of shapes $256 \times 784$, $128 \times 256$, and $10 \times 128$. Each one is a linear transformation from one representation space to another. The input is a 784-dimensional pixel vector; the first matrix projects it into a 256-dimensional "feature space" where, ideally, the digits become more separable. Training adjusts these matrix entries via gradient descent until the composition of all these transformations (interleaved with non-linearities) maps images to correct class labels.

Attention matrices are the heart of the Transformer architecture that powers modern language models. Given a sequence of $n$ token embeddings, the self-attention mechanism computes an $n \times n$ matrix $\text{softmax}(QK^T / \sqrt{d_k})$, where $Q$ and $K$ are themselves linear projections (matrices!) of the input. Each entry $(i,j)$ of this attention matrix represents how much token $i$ should "attend to" token $j$. The attention matrix is a kind of soft routing table — a learned, data-dependent linear combination of value vectors. Understanding matrix multiplication is literally understanding how Transformers think.

Covariance matrices capture the joint variability of features. If you have a dataset with $d$ features, the $d \times d$ covariance matrix $\Sigma$ has entry $\Sigma_{ij} = \text{Cov}(X_i, X_j)$. This matrix is symmetric and positive semi-definite (we'll explore what that means in a later tutorial). Principal Component Analysis (PCA) works by computing the eigenvectors of $\Sigma$ — the directions of maximum variance. Gaussian distributions in $d$ dimensions are entirely determined by a mean vector and a covariance matrix. So understanding matrices is understanding the shape of your data.

The Matrix is the Model

In many classic ML algorithms — linear regression, PCA, linear discriminant analysis, kernel methods — the entire learned model is one or more matrices. Even in deep learning, the non-linear activation functions contain zero learnable parameters. All the learning happens in the matrices.

Matrices as Transformations

Here is one of the most beautiful facts in mathematics: every linear transformation from $\mathbb{R}^n$ to $\mathbb{R}^m$ can be represented by an $m \times n$ matrix $A$, and conversely every $m \times n$ matrix defines a linear transformation. There is a perfect one-to-one correspondence. This means that studying matrices and studying linear transformations are literally the same activity — two languages for the same idea.

The transformation is simply matrix-vector multiplication. Given an $m \times n$ matrix $A$ and a vector $\mathbf{x} \in \mathbb{R}^n$, the output $A\mathbf{x}$ is a vector in $\mathbb{R}^m$:

$$T(\mathbf{x}) = A\mathbf{x}$$

But how do we actually compute this product? The answer reveals the geometric meaning: matrix-vector multiplication is a linear combination of the matrix's columns.

Matrix-Vector Multiplication
1
Write the matrix $A$ as columns: $A = [\mathbf{a}_1 | \mathbf{a}_2 | \cdots | \mathbf{a}_n]$
2
Write $\mathbf{x} = (x_1, x_2, \ldots, x_n)^T$
3
Then: $A\mathbf{x} = x_1\mathbf{a}_1 + x_2\mathbf{a}_2 + \cdots + x_n\mathbf{a}_n$

Let's work through a concrete example. Consider the matrix $A = \begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$ and the vector $\mathbf{x} = \begin{pmatrix} 5 \\ 6 \end{pmatrix}$. Using our column view: $A\mathbf{x} = 5 \begin{pmatrix} 1 \\ 3 \end{pmatrix} + 6 \begin{pmatrix} 2 \\ 4 \end{pmatrix} = \begin{pmatrix} 5 \\ 15 \end{pmatrix} + \begin{pmatrix} 12 \\ 24 \end{pmatrix} = \begin{pmatrix} 17 \\ 39 \end{pmatrix}$. You can verify this by the "row times column" rule: row 1 gives $1 \cdot 5 + 2 \cdot 6 = 17$ and row 2 gives $3 \cdot 5 + 4 \cdot 6 = 39$. Both views produce the same answer, but the column view provides deeper geometric insight.

Key Insight: Columns Tell the Story

The columns of $A$ are where the standard basis vectors land after transformation. The $j$-th column of $A$ is $T(\mathbf{e}_j)$. So the matrix $\begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$ sends $\mathbf{e}_1 = \begin{pmatrix} 1 \\ 0 \end{pmatrix}$ to $\begin{pmatrix} 1 \\ 3 \end{pmatrix}$ and $\mathbf{e}_2 = \begin{pmatrix} 0 \\ 1 \end{pmatrix}$ to $\begin{pmatrix} 2 \\ 4 \end{pmatrix}$. If you know where the basis vectors go, you know the entire transformation.

This "column picture" is how you should always think about matrices. A rotation matrix's columns are unit vectors pointing in the rotated directions of the axes. A projection matrix's columns lie in the subspace being projected onto. A scaling matrix's columns point along the same directions as the original axes, but stretched or compressed. Reading the columns of a matrix tells you exactly what the transformation does to space.

Geometric Gallery of 2D Transformations

Let's build intuition by cataloguing the most important $2 \times 2$ matrices and what they do to the unit square. Each of these appears frequently in computer graphics, robotics, and the geometry of neural network feature spaces.

Scaling: $\begin{pmatrix} s_x & 0 \\ 0 & s_y \end{pmatrix}$ stretches the $x$-axis by $s_x$ and the $y$-axis by $s_y$. If $s_x = s_y$, it's a uniform scaling (zoom in/out). If one of them is negative, we get a reflection along that axis. For instance, $\begin{pmatrix} -1 & 0 \\ 0 & 1 \end{pmatrix}$ reflects across the $y$-axis.

Rotation: $R_\theta = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}$ rotates every vector counterclockwise by angle $\theta$. The columns are $(\cos\theta, \sin\theta)^T$ and $(-\sin\theta, \cos\theta)^T$ — unit vectors at angles $\theta$ and $\theta + 90°$. Rotation matrices are orthogonal: $R_\theta^T = R_\theta^{-1} = R_{-\theta}$.

Shear: $\begin{pmatrix} 1 & k \\ 0 & 1 \end{pmatrix}$ is a horizontal shear. It fixes $\mathbf{e}_1$ but tilts $\mathbf{e}_2$ to $(k, 1)^T$, so vertical lines become slanted. Shears preserve area (determinant = 1) but not angles. They appear in the LU decomposition and in the geometry of recurrent neural networks processing sequential data.

Projection: $\begin{pmatrix} 1 & 0 \\ 0 & 0 \end{pmatrix}$ projects onto the $x$-axis, collapsing all vertical information. This matrix has rank 1, is idempotent ($P^2 = P$), and has eigenvalues 0 and 1. Projections are central to least-squares solutions, PCA, and attention mechanisms (which compute soft projections onto learned subspaces).

Matrix Multiplication as Composition

Suppose you want to first rotate a vector by 30° and then scale it by a factor of 2. Each of these operations is a linear transformation, each represented by its own matrix. But the combined "rotate then scale" operation is also a linear transformation — so it too must be representable by a single matrix. That matrix is the product of the two individual matrices. This is the profound reason matrix multiplication is defined the (seemingly awkward) way it is: it's engineered to encode function composition.

If $A$ represents transformation $S$ and $B$ represents transformation $T$, then $AB$ represents the composition $S \circ T$ (apply $T$ first, then $S$). Notice the right-to-left order — $AB\mathbf{x}$ means "apply $B$ to $\mathbf{x}$, then apply $A$ to the result."

$$(AB)\mathbf{x} = A(B\mathbf{x})$$

The $(i,j)$ entry of $AB$ is computed by taking the dot product of row $i$ of $A$ with column $j$ of $B$:

$$(AB)_{ij} = \sum_{k=1}^{n} A_{ik} B_{kj} = \text{(row } i \text{ of } A) \cdot \text{(column } j \text{ of } B)$$

Let's work through a numerical example. Let $A = \begin{pmatrix} 1 & 2 \\ 0 & 3 \end{pmatrix}$ and $B = \begin{pmatrix} 4 & 1 \\ 2 & 5 \end{pmatrix}$. Then: $(AB)_{11} = 1 \cdot 4 + 2 \cdot 2 = 8$, $(AB)_{12} = 1 \cdot 1 + 2 \cdot 5 = 11$, $(AB)_{21} = 0 \cdot 4 + 3 \cdot 2 = 6$, $(AB)_{22} = 0 \cdot 1 + 3 \cdot 5 = 15$. So $AB = \begin{pmatrix} 8 & 11 \\ 6 & 15 \end{pmatrix}$. Now try $BA$: $(BA)_{11} = 4 \cdot 1 + 1 \cdot 0 = 4$, $(BA)_{12} = 4 \cdot 2 + 1 \cdot 3 = 11$, $(BA)_{21} = 2 \cdot 1 + 5 \cdot 0 = 2$, $(BA)_{22} = 2 \cdot 2 + 5 \cdot 3 = 19$. So $BA = \begin{pmatrix} 4 & 11 \\ 2 & 19 \end{pmatrix} \neq AB$. Order matters!

Matrix Multiplication is Not Commutative

In general, $AB \neq BA$. Order matters! Rotating then scaling is different from scaling then rotating. In a neural network, swapping the order of two weight matrices completely changes the function the network computes. However, matrix multiplication is associative: $(AB)C = A(BC)$, which is why we can meaningfully talk about the product of a chain of matrices without specifying parentheses.

An important property to remember is that matrix multiplication is associative — $(AB)C = A(BC)$ — and distributive — $A(B + C) = AB + AC$. Associativity is what makes deep networks possible: the composition of many layers can be evaluated in any order. Distributivity underlies the linearity of the derivative of a matrix product, which is essential for backpropagation.

Dimensions Must Match

For $AB$ to be defined, the number of columns of $A$ must equal the number of rows of $B$. If $A$ is $m \times n$ and $B$ is $n \times p$, then $AB$ is $m \times p$. A useful mnemonic: the "inner" dimensions must agree, and the "outer" dimensions give the shape of the result.

Special Matrices

Identity Matrix

The identity matrix $I$ is the "do nothing" transformation — it leaves every vector unchanged: $I\mathbf{x} = \mathbf{x}$. It has ones on the main diagonal and zeros everywhere else. The identity matrix plays the same role for matrix multiplication that the number 1 plays for ordinary multiplication: $AI = IA = A$ for any matrix $A$ of compatible dimensions. In ML, initializing a weight matrix to the identity is sometimes used as a "start from no transformation" baseline (e.g., in residual connections, where $\mathbf{y} = f(\mathbf{x}) + I\mathbf{x}$).

$$I_3 = \begin{pmatrix} 1 & 0 & 0 \\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{pmatrix}$$

Inverse Matrix

If $A$ is a square matrix and there exists a matrix $A^{-1}$ such that $A^{-1}A = AA^{-1} = I$, we call $A^{-1}$ the inverse of $A$. Geometrically, $A^{-1}$ "undoes" whatever $A$ does: if $A$ rotates by 45°, $A^{-1}$ rotates by −45°. If $A$ stretches the $x$-axis by 3, $A^{-1}$ compresses it by $1/3$. Not every matrix has an inverse — a matrix that collapses a 2D plane into a line (a rank-1 matrix) has destroyed information, and you cannot undo that. A matrix is invertible if and only if its determinant is non-zero, which is equivalent to saying it has full rank.

For a $2 \times 2$ matrix, there's a simple formula:

$$\begin{pmatrix} a & b \\ c & d \end{pmatrix}^{-1} = \frac{1}{ad - bc} \begin{pmatrix} d & -b \\ -c & a \end{pmatrix}$$

The quantity $ad - bc$ is the determinant. When it equals zero, the matrix squishes space into a lower dimension and no inverse exists. For example, $\begin{pmatrix} 1 & 2 \\ 2 & 4 \end{pmatrix}$ has determinant $1 \cdot 4 - 2 \cdot 2 = 0$ because its rows are proportional — it maps all of $\mathbb{R}^2$ onto a single line.

Transpose

The transpose $A^T$ swaps rows and columns: $(A^T)_{ij} = A_{ji}$. Geometrically, for orthogonal matrices (rotations, reflections), the transpose equals the inverse: $A^T = A^{-1}$. The transpose satisfies a useful "shoe-sock" identity:

$$(AB)^T = B^T A^T$$

This reversal of order appears because composition reverses too — if you put on socks then shoes, you must remove shoes then socks. In ML, the transpose appears constantly. The gradient of a linear layer $\mathbf{y} = W\mathbf{x}$ with respect to $\mathbf{x}$ is $W^T$. During backpropagation, the forward pass uses $W$ and the backward pass uses $W^T$. A matrix that equals its own transpose ($A = A^T$) is called symmetric. Covariance matrices are always symmetric.

Diagonal Matrices

A diagonal matrix has non-zero entries only on the main diagonal: $D = \text{diag}(d_1, d_2, \ldots, d_n)$. Multiplying by a diagonal matrix simply scales each coordinate independently: $D\mathbf{x} = (d_1 x_1, d_2 x_2, \ldots, d_n x_n)^T$. Diagonal matrices are trivial to invert ($D^{-1} = \text{diag}(1/d_1, \ldots, 1/d_n)$, provided all $d_i \neq 0$), trivial to multiply, and trivial to exponentiate. This is why diagonalization — writing $A = PDP^{-1}$ — is so powerful: it reduces a complex matrix to a simple diagonal one in a well-chosen coordinate system.

Orthogonal Matrices

A square matrix $Q$ is orthogonal if $Q^T Q = QQ^T = I$, i.e., its transpose is its inverse. The columns (and rows) of an orthogonal matrix form an orthonormal basis. Geometrically, orthogonal matrices are rigid transformations — they preserve lengths and angles. Rotations and reflections are orthogonal. In ML, orthogonal weight initialization helps prevent vanishing and exploding gradients, because multiplying by an orthogonal matrix neither amplifies nor shrinks the signal.

Matrix Rank

The rank of a matrix is, informally, the number of "truly independent" rows or columns it contains. It measures the dimensionality of the matrix's output — how much of the target space the transformation actually reaches. A $3 \times 3$ matrix of rank 2 maps all of $\mathbb{R}^3$ onto a 2D plane through the origin. A rank-1 matrix collapses everything onto a line. Rank is perhaps the single most important structural property of a matrix.

More precisely, the column space (or range) of $A$ is the span of its columns — the set of all possible outputs $A\mathbf{x}$. The rank is the dimension of this column space. Equivalently, it's the number of linearly independent columns, which (a non-obvious theorem!) always equals the number of linearly independent rows. This is called the column rank equals row rank theorem.

The null space (or kernel) of $A$ is the set of all vectors $\mathbf{x}$ that $A$ sends to the zero vector: $\text{null}(A) = \{\mathbf{x} : A\mathbf{x} = \mathbf{0}\}$. If the null space contains only the zero vector, then $A$ is injective (one-to-one) — different inputs always produce different outputs. If the null space is larger, then $A$ "collapses" some directions, and multiple inputs map to the same output.

Rank-Nullity Theorem

For any $m \times n$ matrix $A$: $$\text{rank}(A) + \text{nullity}(A) = n$$ where $\text{nullity}(A) = \dim(\text{null}(A))$. This says: the dimensions you "use up" (rank) plus the dimensions you "collapse" (nullity) always add up to the total input dimension.

Worked example: Consider $A = \begin{pmatrix} 1 & 2 & 3 \\ 2 & 4 & 6 \end{pmatrix}$. Row 2 is exactly twice row 1, so there is really only one independent row. The rank is 1. By the rank-nullity theorem, the nullity is $3 - 1 = 2$. Indeed, the null space is $\{(x_1, x_2, x_3)^T : x_1 + 2x_2 + 3x_3 = 0\}$, which is a 2D plane in $\mathbb{R}^3$. This matrix maps all of $\mathbb{R}^3$ onto a single line in $\mathbb{R}^2$.

Rank and dimensionality reduction. In ML, we often have a data matrix $X \in \mathbb{R}^{n \times d}$ with $n$ samples and $d$ features. If the data lives near a lower-dimensional subspace — as it often does — then the effective rank of $X$ is much less than $\min(n, d)$. PCA exploits this by finding the best rank-$k$ approximation of $X$. Low-rank matrix factorization powers recommender systems (Netflix prize!), where a sparse user-item matrix is approximated as $X \approx UV^T$ with $U \in \mathbb{R}^{n \times k}$, $V \in \mathbb{R}^{d \times k}$, and $k \ll \min(n, d)$. The assumption is that user preferences lie in a low-dimensional "taste space."

Rank Deficiency = Information Loss

When a matrix has rank less than $\min(m, n)$, it is rank-deficient or singular. This means it irreversibly destroys information — it maps distinct inputs to the same output. From an information-theoretic perspective, a rank-deficient transformation reduces the entropy of the input distribution: the output has fewer degrees of freedom than the input.

Matrix Norms

Just as we measure the "size" of a vector with norms like $\|\mathbf{v}\|_2$, we need ways to measure the "size" of a matrix. But what does "size" mean for a transformation? It depends on what you care about: the maximum stretching, the average distortion, or something else entirely. Different matrix norms capture different aspects of a matrix's behavior, and each plays a distinct role in machine learning.

Frobenius Norm

The simplest matrix norm treats the matrix as a long vector of $mn$ entries and takes the Euclidean norm:

$$\|A\|_F = \sqrt{\sum_{i=1}^{m} \sum_{j=1}^{n} |A_{ij}|^2} = \sqrt{\text{tr}(A^T A)}$$

The Frobenius norm is easy to compute and differentiable everywhere. When you add $L^2$ regularization to a neural network's weights ($\lambda \|W\|_F^2$, also called "weight decay"), you are penalizing the Frobenius norm of the weight matrices. This encourages the network to keep its weights small, which acts as a form of capacity control and helps prevent overfitting. The gradient is simply $\nabla_W \|W\|_F^2 = 2W$, which is why weight decay is equivalent to subtracting a fraction of each weight at every step.

Spectral Norm (Operator Norm)

The spectral norm measures the maximum amount a matrix can stretch any unit vector:

$$\|A\|_2 = \max_{\|\mathbf{x}\|_2 = 1} \|A\mathbf{x}\|_2 = \sigma_{\max}(A)$$

Here $\sigma_{\max}(A)$ is the largest singular value of $A$. The spectral norm tells you the worst-case amplification factor of the matrix. In deep learning, spectral normalization — dividing each weight matrix by its spectral norm — ensures the Lipschitz constant of each layer is at most 1. This technique is crucial for training stable generative adversarial networks (GANs) and for enforcing robustness guarantees. If every layer has spectral norm $\leq 1$, then the entire network is 1-Lipschitz: small input perturbations cannot cause large output changes.

Nuclear Norm (Trace Norm)

The nuclear norm is the sum of all singular values:

$$\|A\|_* = \sum_{i=1}^{\min(m,n)} \sigma_i(A)$$

The nuclear norm is the tightest convex relaxation of the rank function. While minimizing $\text{rank}(A)$ is NP-hard in general, minimizing $\|A\|_*$ is a convex optimization problem that can be solved efficiently. This makes the nuclear norm the go-to tool for low-rank matrix completion — the mathematical foundation of recommendation systems. When Netflix wants to predict which movies you'll enjoy based on a sparse matrix of ratings, nuclear norm minimization fills in the missing entries under the assumption that the true rating matrix is approximately low-rank.

Matrix Norms Summary

Frobenius norm $\|A\|_F$: Total magnitude of all entries. Used for $L^2$ regularization.
Spectral norm $\|A\|_2$: Maximum singular value. Used for Lipschitz constraints.
Nuclear norm $\|A\|_*$: Sum of singular values. Convex proxy for rank minimization.

These three norms satisfy a useful chain of inequalities. For any $m \times n$ matrix $A$ with rank $r$: $\|A\|_2 \leq \|A\|_F \leq \sqrt{r}\, \|A\|_2$ and $\|A\|_F \leq \|A\|_* \leq \sqrt{r}\, \|A\|_F$. When $r = 1$ (a rank-one matrix), all three norms coincide (up to scale). When $r$ is large and the singular values are spread out, the norms can differ substantially. The choice of which norm to use as a regularizer reflects your inductive bias: Frobenius penalizes all entries uniformly, spectral constrains the worst case, and nuclear encourages low rank.

Worked Example: Computing Matrix Norms
1
Let $A = \begin{pmatrix} 3 & 0 \\ 4 & 0 \end{pmatrix}$. This is a rank-1 matrix (column 2 is all zeros).
2
Frobenius: $\|A\|_F = \sqrt{9 + 0 + 16 + 0} = 5$.
3
The singular values of $A$ are $\sigma_1 = 5$ and $\sigma_2 = 0$. So spectral norm $\|A\|_2 = 5$ and nuclear norm $\|A\|_* = 5$.
4
For this rank-1 matrix, all three norms equal 5, confirming $\|A\|_2 = \|A\|_F = \|A\|_*$ when $r = 1$.

Information-Theoretic Connections

Matrices and information theory are deeply intertwined. One of the most natural connections comes through stochastic matrices: matrices whose rows are probability distributions (non-negative entries that sum to 1 in each row). If you have a random variable $X$ that takes values $\{1, 2, \ldots, n\}$, a stochastic matrix $P$ with entries $P_{ij} = \Pr(Y = j \mid X = i)$ defines a channel — a noisy communication system. The input $X$ produces a random output $Y$ according to the transition probabilities in $P$.

Given an input distribution $\mathbf{p} = (p_1, \ldots, p_n)^T$, the output distribution is $\mathbf{q} = P^T \mathbf{p}$ (or $\mathbf{q}^T = \mathbf{p}^T P$ depending on convention). This is matrix-vector multiplication! Shannon's channel coding theorem tells us the maximum rate at which information can be transmitted through this channel — the channel capacity — which equals the maximum mutual information between input and output:

$$C = \max_{\mathbf{p}} I(X; Y) = \max_{\mathbf{p}} [H(Y) - H(Y|X)]$$

A doubly stochastic matrix is one where both the rows and columns sum to 1. Such matrices preserve the uniform distribution: if the input is uniformly distributed, so is the output. The famous Birkhoff-von Neumann theorem states that every doubly stochastic matrix is a convex combination of permutation matrices. In attention mechanisms, the softmax operation produces stochastic matrices — each query's attention weights form a probability distribution over keys. Sinkhorn normalization, which alternately normalizes rows and columns, produces doubly stochastic attention matrices and has been used in optimal transport applications.

Entropy and matrix rank. There is a beautiful connection between the entropy of a matrix's singular value distribution and its "effective rank." Define normalized singular values $\hat{\sigma}_i = \sigma_i / \sum_j \sigma_j$. Then the spectral entropy $H = -\sum_i \hat{\sigma}_i \log \hat{\sigma}_i$ measures how "spread out" the singular values are. When one singular value dominates (rank effectively 1), the entropy is low. When all singular values are equal (full rank, isotropic), the entropy is maximal at $\log(\min(m,n))$. The exponential $e^H$ gives the "effective rank" — the number of dimensions that meaningfully contribute to the transformation.

Data Processing Inequality for Matrices

If $A$ and $B$ are matrices representing sequential transformations, the rank of $AB$ satisfies $\text{rank}(AB) \leq \min(\text{rank}(A), \text{rank}(B))$. This is the linear algebra version of the data processing inequality: you cannot increase information by processing it further. Each successive matrix multiplication can only reduce (or maintain) the dimensionality of the data.

Fisher information matrix. In statistics, the Fisher information matrix $\mathcal{I}(\theta)$ is an $n \times n$ positive semi-definite matrix whose $(i,j)$ entry is $\mathcal{I}_{ij} = \mathbb{E}\left[\frac{\partial \log p(X|\theta)}{\partial \theta_i} \cdot \frac{\partial \log p(X|\theta)}{\partial \theta_j}\right]$. This matrix measures how much information a random variable $X$ carries about an unknown parameter $\theta$. The Cramér-Rao bound says that the covariance of any unbiased estimator of $\theta$ satisfies $\text{Cov}(\hat{\theta}) \succeq \mathcal{I}(\theta)^{-1}$ (in the positive semi-definite ordering). In natural gradient descent — used to train neural networks more efficiently — the parameter update is $\theta \leftarrow \theta - \alpha \mathcal{I}(\theta)^{-1} \nabla_\theta \mathcal{L}$, which uses the inverse Fisher matrix to "correct" the gradient for the curvature of the statistical model. This is a deeply information-theoretic use of matrices.

Common Misconceptions

Even experienced practitioners sometimes hold incorrect beliefs about matrices. Let's address the most common ones head-on, because these misconceptions can lead to subtle bugs in code and flawed reasoning in proofs.

Misconception 1: "Matrix multiplication is commutative"

We've already seen that $AB \neq BA$ in general, but it's worth emphasizing why. Composition of functions is inherently order-dependent: rotating then projecting gives a different result than projecting then rotating. In fact, for random $n \times n$ matrices $A$ and $B$, the probability that $AB = BA$ is zero (measure-theoretically). The matrices that do commute with a given matrix $A$ form its "centralizer," which is typically a very small set. Diagonal matrices commute with each other, and any matrix commutes with the identity and with scalar multiples of the identity, but these are special cases.

Misconception 2: "Every matrix has an inverse"

Only square matrices can possibly have inverses, and even among square matrices, only those with non-zero determinant (equivalently, full rank) are invertible. A $3 \times 5$ matrix cannot have an inverse — it maps $\mathbb{R}^5$ to $\mathbb{R}^3$, losing 2 dimensions of information that can never be recovered. Even a square matrix like $\begin{pmatrix} 1 & 2 \\ 2 & 4 \end{pmatrix}$ is not invertible, because it collapses $\mathbb{R}^2$ onto a 1D line. In numerical computing, a matrix can be "almost singular" (ill-conditioned) — technically invertible but so close to singular that computing the inverse amplifies floating-point errors catastrophically. The condition number $\kappa(A) = \sigma_{\max}/\sigma_{\min}$ measures this; large $\kappa$ means trouble.

Misconception 3: "Rank equals the number of non-zero rows"

This is true for matrices in row echelon form, but not for arbitrary matrices. The matrix $\begin{pmatrix} 1 & 2 \\ 2 & 4 \end{pmatrix}$ has two non-zero rows, but its rank is only 1 because row 2 is a scalar multiple of row 1. Rank counts the number of linearly independent rows (or columns), not merely the non-zero ones. To find the rank, you must perform row reduction (Gaussian elimination) and count the pivots, or compute the singular values and count the non-zero ones.

Misconception 4: "A symmetric matrix is always positive definite"

Symmetry ($A = A^T$) and positive definiteness ($\mathbf{x}^T A \mathbf{x} > 0$ for all $\mathbf{x} \neq 0$) are independent properties, though they often appear together. The matrix $\begin{pmatrix} -1 & 0 \\ 0 & -1 \end{pmatrix}$ is symmetric but negative definite. The matrix $\begin{pmatrix} 1 & 0 \\ 0 & -1 \end{pmatrix}$ is symmetric but indefinite. Positive definiteness requires all eigenvalues to be strictly positive. Covariance matrices are always symmetric and positive semi-definite (eigenvalues $\geq 0$), but not necessarily positive definite — a singular covariance matrix indicates that some features are linearly dependent.

Misconception 5: "$(AB)^{-1} = A^{-1}B^{-1}$"

The correct formula is $(AB)^{-1} = B^{-1}A^{-1}$ — the order reverses, just like for the transpose. This is the "shoe-sock" principle again: to undo "apply $B$ then $A$," you must "undo $A$ then undo $B$." Getting this wrong leads to real bugs, especially in code that chains transformations and then tries to invert them.

The Order Reversal Pattern

Many matrix identities reverse order: $(AB)^T = B^T A^T$, $(AB)^{-1} = B^{-1}A^{-1}$, $(ABC)^{-1} = C^{-1}B^{-1}A^{-1}$. This is not a coincidence — it reflects the fundamental fact that undoing a sequence of actions requires reversing their order.

Exercises

Easy Problems 🟢

  1. Compute the matrix-vector product $\begin{pmatrix} 2 & 1 \\ 0 & 3 \end{pmatrix} \begin{pmatrix} 4 \\ -1 \end{pmatrix}$. Interpret the result geometrically by describing where the standard basis vectors $\mathbf{e}_1$ and $\mathbf{e}_2$ are sent.
  2. Given $A = \begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$ and $B = \begin{pmatrix} 0 & 1 \\ 1 & 0 \end{pmatrix}$, compute both $AB$ and $BA$. Are they equal?
  3. Find the transpose of $A = \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \end{pmatrix}$. What are the dimensions of $A^T$? Verify that $(A^T)^T = A$.
  4. Compute the inverse of $A = \begin{pmatrix} 3 & 1 \\ 5 & 2 \end{pmatrix}$ using the $2 \times 2$ inverse formula. Verify your answer by computing $AA^{-1}$.
  5. Determine the rank of $A = \begin{pmatrix} 1 & 2 & 3 \\ 2 & 4 & 6 \\ 0 & 0 & 0 \end{pmatrix}$. What is the nullity? Describe the column space geometrically.
  6. Compute the Frobenius norm $\|A\|_F$ of $A = \begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$. How does this compare to the Euclidean norm of the vector $(1, 2, 3, 4)^T$?
  7. Verify that $P = \begin{pmatrix} 0.7 & 0.3 \\ 0.4 & 0.6 \end{pmatrix}$ is a stochastic matrix. If the initial distribution is $\mathbf{p} = (1, 0)^T$, what is the distribution after one step (compute $P^T \mathbf{p}$)?
  8. Which of the following matrices are symmetric? (a) $\begin{pmatrix} 1 & 3 \\ 3 & 2 \end{pmatrix}$, (b) $\begin{pmatrix} 1 & 2 \\ 3 & 4 \end{pmatrix}$, (c) $\begin{pmatrix} 5 & 0 & 0 \\ 0 & 3 & 0 \\ 0 & 0 & 1 \end{pmatrix}$.

Medium Problems 🟡

  1. Let $R_\theta = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}$ be a rotation matrix. Show that $R_\theta$ is orthogonal by verifying $R_\theta^T R_\theta = I$. Then show that $R_\alpha R_\beta = R_{\alpha + \beta}$ (rotation composition adds angles).
  2. Prove that if $A$ is an $m \times n$ matrix, then $\text{rank}(A) = \text{rank}(A^T A) = \text{rank}(AA^T)$. Why is $A^T A$ always square and symmetric?
  3. Find a $2 \times 2$ matrix $A$ such that $A^2 = \begin{pmatrix} -1 & 0 \\ 0 & -1 \end{pmatrix}$. (Hint: think about rotations.) Is the solution unique?
  4. Show that for any matrix $A$, the matrix $A^T A$ is positive semi-definite. That is, prove $\mathbf{x}^T (A^T A) \mathbf{x} \geq 0$ for all $\mathbf{x}$. When is it strictly positive definite?
  5. Let $P = \begin{pmatrix} 1/3 & 1/3 & 1/3 \\ 1/4 & 1/2 & 1/4 \\ 1/2 & 0 & 1/2 \end{pmatrix}$ be a transition matrix. Compute $P^2$ and interpret the entries of $P^2$ in terms of two-step transition probabilities. Is $P$ doubly stochastic?
  6. Find the spectral norm $\|A\|_2$ of $A = \begin{pmatrix} 3 & 0 \\ 0 & -2 \end{pmatrix}$. What is the nuclear norm $\|A\|_*$? What is the Frobenius norm $\|A\|_F$? Verify the inequality $\|A\|_2 \leq \|A\|_F \leq \sqrt{r} \|A\|_2$ where $r = \text{rank}(A)$.
  7. A linear layer in a neural network has weight matrix $W \in \mathbb{R}^{128 \times 784}$ and bias $\mathbf{b} \in \mathbb{R}^{128}$. How many parameters does this layer have? What is the rank constraint on $W$? If we factorize $W = UV$ with $U \in \mathbb{R}^{128 \times k}$ and $V \in \mathbb{R}^{k \times 784}$, how many parameters does the factorized version have for $k = 32$?
  8. Prove that $\text{rank}(AB) \leq \min(\text{rank}(A), \text{rank}(B))$. Give an example where $\text{rank}(AB) < \min(\text{rank}(A), \text{rank}(B))$.

Hard Problems 🔴

  1. Prove the Eckart-Young theorem: the best rank-$k$ approximation to a matrix $A$ (in Frobenius norm) is given by the truncated SVD $A_k = \sum_{i=1}^{k} \sigma_i \mathbf{u}_i \mathbf{v}_i^T$, and the approximation error is $\|A - A_k\|_F = \sqrt{\sigma_{k+1}^2 + \cdots + \sigma_r^2}$. What does this theorem have to do with PCA?
  2. Let $P$ be an $n \times n$ doubly stochastic matrix. Prove that the uniform distribution $\mathbf{u} = (1/n, \ldots, 1/n)^T$ is a stationary distribution of $P$ (i.e., $P^T \mathbf{u} = \mathbf{u}$). Show that the entropy of the output distribution is at least as large as the entropy of the input when the input is uniform: $H(P^T\mathbf{u}) \geq H(\mathbf{u})$ (with equality).
  3. Show that for a symmetric matrix $A \in \mathbb{R}^{n \times n}$, the Frobenius norm equals $\|A\|_F = \sqrt{\sum_{i=1}^n \lambda_i^2}$ where $\lambda_i$ are the eigenvalues of $A$. Use this to prove that $\|A\|_2 \leq \|A\|_F \leq \sqrt{n} \|A\|_2$ for symmetric $A$.
  4. Consider the attention mechanism: $\text{Attention}(Q, K, V) = \text{softmax}(QK^T / \sqrt{d_k}) V$. If $Q, K \in \mathbb{R}^{n \times d_k}$ and $V \in \mathbb{R}^{n \times d_v}$, what is the shape of the attention weight matrix $\text{softmax}(QK^T / \sqrt{d_k})$? Prove it is a stochastic matrix (each row sums to 1). What would it mean for it to be doubly stochastic?
  5. Prove that the nuclear norm is the convex envelope of the rank function on the set of matrices with spectral norm at most 1. More precisely, show that $\|A\|_* = \min \{ \sum_i \|B_i\|_2 : A = \sum_i B_i, \text{rank}(B_i) = 1 \}$. Why does this make nuclear norm minimization a good surrogate for rank minimization?
  6. Define the matrix exponential $e^A = \sum_{k=0}^{\infty} \frac{A^k}{k!}$. Show that if $A$ is diagonalizable as $A = PDP^{-1}$, then $e^A = P e^D P^{-1}$ where $e^D = \text{diag}(e^{d_1}, \ldots, e^{d_n})$. Prove that $e^A$ is always invertible with $(e^A)^{-1} = e^{-A}$. How does the matrix exponential relate to continuous-time Markov chains?
  7. Let $A \in \mathbb{R}^{m \times n}$ with singular values $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_r > 0$. Define the effective rank as $\text{erank}(A) = \exp\left(-\sum_{i=1}^{r} \hat{\sigma}_i \log \hat{\sigma}_i\right)$ where $\hat{\sigma}_i = \sigma_i / \sum_j \sigma_j$. Show that $1 \leq \text{erank}(A) \leq \text{rank}(A)$, with the lower bound achieved when one singular value dominates and the upper bound when all singular values are equal.
  8. Prove the matrix determinant lemma: for an invertible $n \times n$ matrix $A$ and vectors $\mathbf{u}, \mathbf{v} \in \mathbb{R}^n$, $\det(A + \mathbf{u}\mathbf{v}^T) = (1 + \mathbf{v}^T A^{-1} \mathbf{u}) \det(A)$. Use this to show that a rank-1 update to an invertible matrix can make it singular if and only if $\mathbf{v}^T A^{-1} \mathbf{u} = -1$. How is this relevant to the Sherman-Morrison formula for updating inverses?
