What is a Vector?

Ask a physicist what a vector is, and they'll say "an arrow with magnitude and direction." Ask a computer scientist, and they'll say "an ordered list of numbers." Ask a mathematician, and they'll give you the most powerful answer of all: a vector is any object that can be added to other vectors and scaled by numbers, provided these operations follow certain rules. This abstract definition is what makes linear algebra so universally applicable — the same theorems that describe arrows in 3D space also describe quantum states, probability distributions, and the high-dimensional embeddings that power modern machine learning.

Why should you care about this abstraction? Because in machine learning, you will encounter vectors everywhere, and they almost never look like arrows. A word embedding in a language model is a vector with 768 or more components — good luck drawing that as an arrow. A dataset with 1,000 features per sample lives in $\mathbb{R}^{1000}$, a space no human can visualize. Yet the same operations — addition, scaling, projection, measuring distance — work identically whether you have 2 dimensions or 2 million. The geometry of high-dimensional spaces is counterintuitive in many ways, but the algebra is rock-solid, and that algebra starts here.

Vector in $\mathbb{R}^n$

A vector $\mathbf{v} \in \mathbb{R}^n$ is an ordered tuple of $n$ real numbers: $$\mathbf{v} = \begin{pmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{pmatrix}$$

The number $n$ is called the dimension of the vector, and each $v_i$ is called a component (or coordinate) of the vector. The notation $\mathbf{v} \in \mathbb{R}^n$ means "$\mathbf{v}$ belongs to the set of all $n$-tuples of real numbers."

The Dual Perspective: Geometric vs. Algebraic

There are two complementary ways to think about vectors, and both are essential. The geometric perspective treats a vector as a directed line segment — an arrow from one point to another. This is invaluable for building intuition in 2D and 3D: you can visualize addition as placing arrows tip-to-tail, and scalar multiplication as stretching or compressing an arrow. The algebraic perspective treats a vector as a column of numbers. This is what you'll use in computation, and it generalizes effortlessly to any number of dimensions. Neither perspective is "correct" on its own — the geometric view guides your intuition while the algebraic view drives your calculations.

Notice the word "ordered" in the definition. The vector $\begin{pmatrix} 3 \\ 5 \end{pmatrix}$ is fundamentally different from $\begin{pmatrix} 5 \\ 3 \end{pmatrix}$, even though they contain the same numbers. Order matters because each position carries meaning. In a feature vector describing a house, position 1 might represent square footage and position 2 might represent number of bedrooms. Swapping those values would describe a completely different house. This is why vectors are tuples (where order matters), not sets (where it doesn't).

Vectors in Machine Learning

In modern ML, vectors are the universal language of data representation. A word embedding maps each word in a vocabulary to a dense vector (e.g., 300 dimensions in Word2Vec), where semantically similar words end up near each other. The famous result $\text{king} - \text{man} + \text{woman} \approx \text{queen}$ is literally vector arithmetic in embedding space. Feature vectors represent data points: each image in a dataset might be a vector of pixel intensities (784 dimensions for a 28×28 grayscale image), or a learned representation from a neural network (2048 dimensions from a ResNet). Even the parameters of a neural network form a single enormous vector — GPT-3's parameter vector lives in $\mathbb{R}^{175,000,000,000}$.

Vector Operations

Two fundamental operations define vector algebra, and everything else in linear algebra is built on top of them. These operations are so important that a "vector space" is defined entirely by how these operations behave.

Addition: Add component-wise. Each component of the result is the sum of the corresponding components of the inputs.

$$\mathbf{u} + \mathbf{v} = \begin{pmatrix} u_1 + v_1 \\ u_2 + v_2 \\ \vdots \\ u_n + v_n \end{pmatrix}$$

Scalar Multiplication: Scale each component by the same number $c$. If $c > 1$, the vector stretches; if $0 < c < 1$, it shrinks; if $c < 0$, it flips direction.

$$c\mathbf{v} = \begin{pmatrix} cv_1 \\ cv_2 \\ \vdots \\ cv_n \end{pmatrix}$$

Notice what is not included in this list: there is no built-in notion of multiplying two vectors together. Vector multiplication is not a single concept — there are multiple products (dot product, cross product, outer product) that serve different purposes, and we'll meet the dot product later in this tutorial. The fact that only addition and scaling are fundamental is a feature, not a bug — it keeps the theory clean and general.

Vector Spaces

A vector space is a set $V$ equipped with two operations — vector addition and scalar multiplication — that satisfy a specific list of axioms. This is one of the most important definitions in all of mathematics, because it captures the essence of "linearity" in a way that applies far beyond arrows in space. Once you prove a theorem about abstract vector spaces, that theorem automatically applies to every concrete example: columns of numbers, polynomials, functions, matrices, probability distributions, and more.

You might wonder: why bother with axioms? Can't we just work with $\mathbb{R}^n$ and be done with it? The answer is that many important mathematical objects naturally form vector spaces without being columns of numbers. The set of all polynomials of degree at most 3 is a vector space — you can add polynomials and scale them by constants, and all the axioms hold. The set of all continuous functions from $[0,1]$ to $\mathbb{R}$ is a vector space. Even the set of all $m \times n$ matrices is a vector space. By proving theorems at the abstract level, we get results that work for all of these simultaneously.

Vector Space Axioms (Complete List)
1
Closure under addition: If $\mathbf{u}, \mathbf{v} \in V$, then $\mathbf{u} + \mathbf{v} \in V$. Adding two vectors in the space always produces another vector in the space — you can never "escape" by adding.
2
Closure under scalar multiplication: If $\mathbf{v} \in V$ and $c \in \mathbb{R}$, then $c\mathbf{v} \in V$. Scaling a vector by any real number keeps it in the space.
3
Commutativity of addition: $\mathbf{u} + \mathbf{v} = \mathbf{v} + \mathbf{u}$. The order of addition doesn't matter.
4
Associativity of addition: $(\mathbf{u} + \mathbf{v}) + \mathbf{w} = \mathbf{u} + (\mathbf{v} + \mathbf{w})$. Grouping doesn't matter when adding three or more vectors.
5
Existence of zero vector: There exists a vector $\mathbf{0} \in V$ such that $\mathbf{v} + \mathbf{0} = \mathbf{v}$ for every $\mathbf{v} \in V$. Every vector space has an "identity" for addition.
6
Existence of additive inverses: For every $\mathbf{v} \in V$, there exists $-\mathbf{v} \in V$ such that $\mathbf{v} + (-\mathbf{v}) = \mathbf{0}$. You can always "undo" addition.
7
Compatibility of scalar multiplication: $a(b\mathbf{v}) = (ab)\mathbf{v}$. Scaling by $a$ then $b$ is the same as scaling by $ab$.
8
Identity of scalar multiplication: $1\mathbf{v} = \mathbf{v}$. Scaling by 1 does nothing.
9
Distributivity over vector addition: $c(\mathbf{u} + \mathbf{v}) = c\mathbf{u} + c\mathbf{v}$. Scaling a sum equals the sum of the scaled parts.
10
Distributivity over scalar addition: $(a + b)\mathbf{v} = a\mathbf{v} + b\mathbf{v}$. Scaling by a sum of scalars distributes.

Non-Obvious Vector Spaces

The power of abstraction becomes clear when you see vector spaces that look nothing like columns of numbers. Consider the set $\mathcal{P}_n$ of all polynomials of degree at most $n$. The polynomial $p(x) = 3x^2 + 2x - 1$ is a "vector" in $\mathcal{P}_2$. Adding two such polynomials gives another polynomial of degree at most 2 (closure), and scaling a polynomial by a constant gives another polynomial of degree at most 2. The zero vector is the zero polynomial $p(x) = 0$. All ten axioms check out. This vector space has dimension $n+1$ because the monomials $\{1, x, x^2, \ldots, x^n\}$ form a basis — but we're getting ahead of ourselves.

An even more exotic example: the set $C[0,1]$ of all continuous functions from $[0,1]$ to $\mathbb{R}$ is a vector space. Here, the "vectors" are entire functions like $f(x) = \sin(\pi x)$ or $g(x) = e^x$. Vector addition is pointwise: $(f+g)(x) = f(x) + g(x)$. Scalar multiplication is also pointwise: $(cf)(x) = c \cdot f(x)$. This vector space is infinite-dimensional — no finite set of functions can span it. Infinite-dimensional spaces are crucial in functional analysis and form the mathematical backbone of kernel methods in ML.

Perhaps the most ML-relevant non-obvious example: the set of all probability distributions over a finite set $\{1, 2, \ldots, k\}$ is not a vector space (why not?), but the set of all signed measures on that set is. This subtlety matters when working with information-theoretic quantities like KL divergence, which measures how one distribution differs from another. We'll return to this connection at the end of this tutorial.

Why Probability Distributions Don't Form a Vector Space

A probability distribution $\mathbf{p} = (p_1, \ldots, p_k)$ satisfies $p_i \geq 0$ and $\sum p_i = 1$. But if you scale $\mathbf{p}$ by $c = 2$, the result has $\sum 2p_i = 2 \neq 1$ — it's no longer a probability distribution. Closure under scalar multiplication fails. The set of probability distributions is a convex set (closed under convex combinations $\lambda \mathbf{p} + (1-\lambda)\mathbf{q}$ for $\lambda \in [0,1]$), but not a vector space.

Linear Combinations and Span

If vector spaces are the stage, then linear combinations are the main act. A linear combination of vectors $\mathbf{v}_1, \ldots, \mathbf{v}_k$ is any expression of the form:

$$c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_k\mathbf{v}_k$$

where $c_1, \ldots, c_k$ are real numbers called coefficients (or weights). This is the only "recipe" allowed in a vector space: take some vectors, scale each one, and add the results. You might think this is restrictive, but it turns out that an enormous amount of mathematics and computation reduces to finding the right coefficients in a linear combination. Solving a system of linear equations? Finding coefficients. Least-squares regression? Finding coefficients. Principal component analysis? You guessed it — coefficients.

The span of a set of vectors is the collection of all linear combinations you can form from those vectors:

$$\text{span}(\mathbf{v}_1, \ldots, \mathbf{v}_k) = \{c_1\mathbf{v}_1 + \cdots + c_k\mathbf{v}_k : c_i \in \mathbb{R}\}$$

Think of the span as the set of all "reachable" vectors starting from a given collection. If you have a single nonzero vector in $\mathbb{R}^2$, its span is a line through the origin — you can only reach points along that line by scaling. Add a second vector that points in a different direction, and suddenly you can reach every point in the plane. The span "grew" from a 1-dimensional line to a 2-dimensional plane because the second vector added genuinely new reach.

Geometric Interpretation

The span of a single nonzero vector is a line through the origin. The span of two non-parallel vectors is a plane through the origin. The span of three vectors that don't all lie in the same plane is all of $\mathbb{R}^3$. Each new independent vector adds a dimension to the span. But adding a vector that's already in the span contributes nothing new — the span doesn't grow.

Worked Example: Computing a Linear Combination

Let's make this concrete. Consider the vectors $\mathbf{v}_1 = \begin{pmatrix} 1 \\ 2 \end{pmatrix}$ and $\mathbf{v}_2 = \begin{pmatrix} 3 \\ -1 \end{pmatrix}$ in $\mathbb{R}^2$. Can we write $\mathbf{b} = \begin{pmatrix} 7 \\ 3 \end{pmatrix}$ as a linear combination of $\mathbf{v}_1$ and $\mathbf{v}_2$?

Finding the Coefficients
1
We need $c_1$ and $c_2$ such that $c_1\begin{pmatrix} 1 \\ 2 \end{pmatrix} + c_2\begin{pmatrix} 3 \\ -1 \end{pmatrix} = \begin{pmatrix} 7 \\ 3 \end{pmatrix}$.
2
This gives us two equations: $c_1 + 3c_2 = 7$ and $2c_1 - c_2 = 3$.
3
From the second equation: $c_2 = 2c_1 - 3$. Substituting into the first: $c_1 + 3(2c_1 - 3) = 7$, so $7c_1 - 9 = 7$, giving $c_1 = \frac{16}{7}$.
4
Then $c_2 = 2 \cdot \frac{16}{7} - 3 = \frac{32-21}{7} = \frac{11}{7}$.
5
Verify: $\frac{16}{7}\begin{pmatrix} 1 \\ 2 \end{pmatrix} + \frac{11}{7}\begin{pmatrix} 3 \\ -1 \end{pmatrix} = \begin{pmatrix} \frac{16+33}{7} \\ \frac{32-11}{7} \end{pmatrix} = \begin{pmatrix} 7 \\ 3 \end{pmatrix}$ ✓

Since $\mathbf{v}_1$ and $\mathbf{v}_2$ are not parallel (neither is a scalar multiple of the other), their span is all of $\mathbb{R}^2$. Every vector in the plane can be expressed as a linear combination of these two vectors. This is exactly what it means for two non-parallel vectors to form a basis for $\mathbb{R}^2$ — a concept we'll formalize shortly.

Why Span Forms a Subspace

Here's an important structural fact: the span of any set of vectors is always a subspace (a vector space contained within a larger vector space). Why? Because if $\mathbf{u}$ and $\mathbf{w}$ are both in the span — meaning both can be written as linear combinations of $\mathbf{v}_1, \ldots, \mathbf{v}_k$ — then $\mathbf{u} + \mathbf{w}$ is also a linear combination of those same vectors (just add the coefficients), and $c\mathbf{u}$ is too (just scale the coefficients). Closure under addition and scalar multiplication are automatically satisfied. The zero vector is the linear combination with all coefficients equal to zero. So the span is always a bona fide vector space in its own right.

Connection to Feature Engineering

In ML, choosing features for a model is essentially choosing which vectors span your representation space. If you include redundant features (ones that are linear combinations of others), they don't expand the span — they add no representational power. This is why multicollinearity is a problem in linear regression: redundant features make the coefficient estimates unstable without improving the model's expressive ability.

Linear Independence

The concept of linear independence is about detecting redundancy. Vectors $\mathbf{v}_1, \ldots, \mathbf{v}_k$ are linearly independent if the only way to combine them to get the zero vector is the trivial way — setting all coefficients to zero:

$$c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_k\mathbf{v}_k = \mathbf{0} \quad \Longrightarrow \quad c_1 = c_2 = \cdots = c_k = 0$$

If there exist coefficients $c_1, \ldots, c_k$, not all zero, that produce the zero vector, then the vectors are linearly dependent. In that case, at least one vector can be "written in terms of" the others — it's redundant. For instance, if $c_1 \neq 0$, we can solve for $\mathbf{v}_1 = -\frac{c_2}{c_1}\mathbf{v}_2 - \cdots - \frac{c_k}{c_1}\mathbf{v}_k$. So $\mathbf{v}_1$ contributes nothing that the other vectors don't already provide.

Key Insight

Linear independence means no vector in the set can be written as a linear combination of the others. Each vector contributes something genuinely "new" to the space — it points in a direction not reachable by any combination of the remaining vectors. Geometrically, adding an independent vector always increases the dimension of the span.

Concrete Example: Testing for Independence

Are the vectors $\mathbf{v}_1 = \begin{pmatrix} 1 \\ 0 \\ 2 \end{pmatrix}$, $\mathbf{v}_2 = \begin{pmatrix} 0 \\ 1 \\ 3 \end{pmatrix}$, and $\mathbf{v}_3 = \begin{pmatrix} 2 \\ 1 \\ 7 \end{pmatrix}$ linearly independent in $\mathbb{R}^3$?

Testing Linear Independence
1
Set up the equation: $c_1\begin{pmatrix} 1 \\ 0 \\ 2 \end{pmatrix} + c_2\begin{pmatrix} 0 \\ 1 \\ 3 \end{pmatrix} + c_3\begin{pmatrix} 2 \\ 1 \\ 7 \end{pmatrix} = \begin{pmatrix} 0 \\ 0 \\ 0 \end{pmatrix}$
2
This gives three equations: $c_1 + 2c_3 = 0$, $c_2 + c_3 = 0$, $2c_1 + 3c_2 + 7c_3 = 0$.
3
From equation 1: $c_1 = -2c_3$. From equation 2: $c_2 = -c_3$. Substituting into equation 3: $2(-2c_3) + 3(-c_3) + 7c_3 = -4c_3 - 3c_3 + 7c_3 = 0$. ✓
4
Equation 3 is satisfied for any value of $c_3$! Choosing $c_3 = 1$ gives $c_1 = -2$, $c_2 = -1$. So $-2\mathbf{v}_1 - \mathbf{v}_2 + \mathbf{v}_3 = \mathbf{0}$.
5
Conclusion: The vectors are linearly dependent. Specifically, $\mathbf{v}_3 = 2\mathbf{v}_1 + \mathbf{v}_2$, so $\mathbf{v}_3$ is redundant.

Connection to Rank and Dimensionality Reduction

The rank of a matrix is the maximum number of linearly independent columns (equivalently, the dimension of the column space). When a dataset matrix has rank much smaller than the number of features, the data lies on a lower-dimensional subspace — many features are redundant combinations of others. This is exactly the situation that Principal Component Analysis (PCA) exploits: it finds the directions of maximum variance (the independent directions), and projects the data onto a smaller set of independent vectors, discarding the redundant dimensions. The number of principal components you keep is essentially the rank of your approximation.

In deep learning, the concept surfaces again in low-rank factorization. If a weight matrix $W$ of size $m \times n$ has effective rank $r \ll \min(m,n)$, you can approximate it as $W \approx AB$ where $A$ is $m \times r$ and $B$ is $r \times n$, dramatically reducing the number of parameters. Techniques like LoRA (Low-Rank Adaptation) use this principle to fine-tune large language models efficiently: instead of updating all 175 billion parameters, you update a low-rank perturbation with only millions of parameters.

Basis and Dimension

A basis for a vector space $V$ is a set of vectors that satisfies two conditions simultaneously:

  • Linearly independent — no vector in the set is redundant
  • Spanning — every vector in $V$ can be written as a linear combination of the basis vectors

A basis is a "just right" collection: it has enough vectors to reach everywhere in the space (spanning), but no more than necessary (independent). Remove any vector from a basis and you lose spanning; add any vector and you lose independence. This Goldilocks property is what makes bases so useful — they provide a minimal, non-redundant coordinate system for the entire space.

Dimension

The dimension of a vector space $V$, denoted $\dim(V)$, is the number of vectors in any basis of $V$. A remarkable theorem of linear algebra guarantees that every basis for a given vector space has the same number of elements, so the dimension is well-defined. For $\mathbb{R}^n$, $\dim(\mathbb{R}^n) = n$.

The standard basis for $\mathbb{R}^3$ consists of the unit vectors along each coordinate axis:

$$\mathbf{e}_1 = \begin{pmatrix} 1 \\ 0 \\ 0 \end{pmatrix}, \quad \mathbf{e}_2 = \begin{pmatrix} 0 \\ 1 \\ 0 \end{pmatrix}, \quad \mathbf{e}_3 = \begin{pmatrix} 0 \\ 0 \\ 1 \end{pmatrix}$$

These are independent (none is a combination of the others) and they span $\mathbb{R}^3$ (any vector $\begin{pmatrix} a \\ b \\ c \end{pmatrix} = a\mathbf{e}_1 + b\mathbf{e}_2 + c\mathbf{e}_3$). But the standard basis is just one of infinitely many possible bases.

Non-Standard Bases

The set $\left\{\begin{pmatrix} 1 \\ 1 \end{pmatrix}, \begin{pmatrix} 1 \\ -1 \end{pmatrix}\right\}$ is also a basis for $\mathbb{R}^2$. To express, say, $\begin{pmatrix} 3 \\ 1 \end{pmatrix}$ in this basis, solve $c_1\begin{pmatrix} 1 \\ 1 \end{pmatrix} + c_2\begin{pmatrix} 1 \\ -1 \end{pmatrix} = \begin{pmatrix} 3 \\ 1 \end{pmatrix}$. We get $c_1 + c_2 = 3$ and $c_1 - c_2 = 1$, yielding $c_1 = 2, c_2 = 1$. So the coordinates of $\begin{pmatrix} 3 \\ 1 \end{pmatrix}$ in this new basis are $(2, 1)$ — different from the standard coordinates $(3, 1)$. The vector itself hasn't changed; only our description of it has.

Change of Basis

Change of basis is the process of translating a vector's coordinates from one basis to another. If $B = \{\mathbf{b}_1, \ldots, \mathbf{b}_n\}$ is a new basis and we arrange the basis vectors as columns of a matrix $P = [\mathbf{b}_1 \mid \mathbf{b}_2 \mid \cdots \mid \mathbf{b}_n]$, then the coordinates $[\mathbf{v}]_B$ of $\mathbf{v}$ in basis $B$ satisfy:

$$\mathbf{v} = P [\mathbf{v}]_B \qquad \Longleftrightarrow \qquad [\mathbf{v}]_B = P^{-1}\mathbf{v}$$

The matrix $P$ is called the change-of-basis matrix. It converts coordinates from the new basis back to the standard basis, and $P^{-1}$ converts from the standard basis to the new one.

Why Basis Choice Matters in ML

PCA is fundamentally a change of basis. The original features form one basis; PCA finds a new basis (the principal components) in which the data's covariance matrix is diagonal — each coordinate captures independent variation. In this new basis, the data's structure is laid bare: the first few coordinates carry most of the variance, and the rest can be discarded. Similarly, the Fourier transform is a change of basis from the "time domain" to the "frequency domain," and the wavelet transform provides yet another basis. Different bases reveal different structures in the same data.

Dot Products and Norms

So far, our vector space has addition and scaling — but no notion of length or angle. These geometric concepts require additional structure: the inner product (also called dot product in $\mathbb{R}^n$). Once we have a dot product, we can measure distances, angles, and projections — tools that are indispensable in ML, where "similarity" between vectors is a core operation.

Dot Product (Inner Product in $\mathbb{R}^n$)

For $\mathbf{u}, \mathbf{v} \in \mathbb{R}^n$, the dot product is: $$\mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^{n} u_i v_i = u_1 v_1 + u_2 v_2 + \cdots + u_n v_n$$ Equivalently, $\mathbf{u} \cdot \mathbf{v} = \mathbf{u}^T \mathbf{v}$ (a $1 \times n$ row times an $n \times 1$ column gives a scalar).

The dot product takes two vectors and returns a single number — a scalar. This number encodes rich geometric information. If the dot product is positive, the vectors point roughly in the same direction; if negative, they point roughly in opposite directions; if zero, they are orthogonal (perpendicular). The magnitude of the dot product tells you how "aligned" the vectors are.

Geometric Interpretation: Projection

The dot product has a beautiful geometric interpretation in terms of projection. If $\theta$ is the angle between $\mathbf{u}$ and $\mathbf{v}$, then:

$$\mathbf{u} \cdot \mathbf{v} = \|\mathbf{u}\| \|\mathbf{v}\| \cos\theta$$

This formula connects algebra (component-wise multiplication and sum) to geometry (lengths and angles). The projection of $\mathbf{u}$ onto $\mathbf{v}$ — the "shadow" that $\mathbf{u}$ casts along $\mathbf{v}$ — has length $\frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{v}\|}$ and the projection vector itself is:

$$\text{proj}_{\mathbf{v}}(\mathbf{u}) = \frac{\mathbf{u} \cdot \mathbf{v}}{\mathbf{v} \cdot \mathbf{v}} \mathbf{v}$$

Projection is the workhorse behind least-squares regression: the "best fit" is the projection of the observation vector onto the column space of the feature matrix. It's also fundamental to the Gram-Schmidt process for constructing orthonormal bases, and to QR decomposition.

Norms: Measuring Length

The dot product gives us a natural notion of length, called the Euclidean norm (or $L^2$ norm):

$$\|\mathbf{v}\| = \sqrt{\mathbf{v} \cdot \mathbf{v}} = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2}$$

But the Euclidean norm is just one member of a family. The $L^p$ norms are:

$$\|\mathbf{v}\|_p = \left(\sum_{i=1}^{n} |v_i|^p\right)^{1/p}$$

The $L^1$ norm $\|\mathbf{v}\|_1 = \sum |v_i|$ (Manhattan distance) promotes sparsity in LASSO regression. The $L^\infty$ norm $\|\mathbf{v}\|_\infty = \max_i |v_i|$ measures the largest component. Each norm defines a different geometry — a different notion of what it means for vectors to be "close."

The Cauchy-Schwarz Inequality

One of the most important inequalities in all of mathematics states:

$$|\mathbf{u} \cdot \mathbf{v}| \leq \|\mathbf{u}\| \|\mathbf{v}\|$$

Equality holds if and only if $\mathbf{u}$ and $\mathbf{v}$ are parallel (one is a scalar multiple of the other). Since $\mathbf{u} \cdot \mathbf{v} = \|\mathbf{u}\|\|\mathbf{v}\|\cos\theta$, the inequality just says $|\cos\theta| \leq 1$, which is obvious geometrically — but the algebraic proof works in any inner product space, including infinite-dimensional ones where "angle" has no visual meaning. The Cauchy-Schwarz inequality guarantees that cosine similarity is always between $-1$ and $1$:

$$\cos\theta = \frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{u}\| \|\mathbf{v}\|} \in [-1, 1]$$
Cosine Similarity in ML

Cosine similarity is ubiquitous in information retrieval, recommendation systems, and NLP. When comparing word embeddings or document vectors, cosine similarity measures how similar the directions of two vectors are, ignoring their magnitudes. Two documents about the same topic will have similar directions in embedding space even if one is much longer (larger magnitude) than the other. This direction-based comparison is precisely what makes cosine similarity preferred over raw dot products for text similarity tasks.

Subspaces

A subspace of a vector space $V$ is a subset $W \subseteq V$ that is itself a vector space under the same operations. In other words, $W$ must be closed under addition and scalar multiplication, and must contain the zero vector. Subspaces are the "natural habitats" of linear algebra — they're the sets where linear operations behave well, and they show up everywhere from the solution sets of linear equations to the range of linear transformations.

Subspace Test

A nonempty subset $W$ of a vector space $V$ is a subspace if and only if:
(1) $\mathbf{0} \in W$ (contains the zero vector),
(2) If $\mathbf{u}, \mathbf{v} \in W$, then $\mathbf{u} + \mathbf{v} \in W$ (closed under addition),
(3) If $\mathbf{v} \in W$ and $c \in \mathbb{R}$, then $c\mathbf{v} \in W$ (closed under scalar multiplication).
Equivalently, conditions (2) and (3) can be combined: $W$ is a subspace iff $\mathbf{0} \in W$ and $a\mathbf{u} + b\mathbf{v} \in W$ for all $\mathbf{u}, \mathbf{v} \in W$ and $a, b \in \mathbb{R}$.

Notice that a subspace must pass through the origin (it contains $\mathbf{0}$). A plane that doesn't pass through the origin is not a subspace — it's an affine subspace, which is a subspace translated away from the origin. This distinction matters: lines through the origin are subspaces of $\mathbb{R}^2$; lines that miss the origin are not.

Column Space and Null Space

Two of the most important subspaces associated with a matrix $A$ (of size $m \times n$) are:

The column space $C(A)$ is the span of the columns of $A$ — it's the set of all vectors $\mathbf{b}$ such that $A\mathbf{x} = \mathbf{b}$ has a solution. Geometrically, the column space is the "reachable set" of the linear transformation defined by $A$. If $\mathbf{b}$ is in the column space, the system is consistent; if not, the best we can do is project $\mathbf{b}$ onto the column space (this is exactly least-squares).

The null space $N(A)$ is the set of all vectors $\mathbf{x}$ such that $A\mathbf{x} = \mathbf{0}$ — the "invisible" directions that $A$ maps to zero. The null space captures the ambiguity in the system: if $\mathbf{x}_0$ is a particular solution to $A\mathbf{x} = \mathbf{b}$, then $\mathbf{x}_0 + \mathbf{n}$ is also a solution for any $\mathbf{n} \in N(A)$. The dimension of the null space — called the nullity — tells you how many "free parameters" the solution has.

The Rank-Nullity Theorem

For any $m \times n$ matrix $A$: $\text{rank}(A) + \text{nullity}(A) = n$. The number of independent columns plus the number of "lost" dimensions equals the total number of columns. This is one of the most elegant bookkeeping results in linear algebra, and it tells you exactly how much information $A$ preserves and how much it destroys.

In ML, the null space matters when your feature matrix has more columns than rows (more features than data points) — the null space is nontrivial, meaning there are infinitely many weight vectors that fit the training data equally well. Regularization (L1, L2, or others) is the standard technique for selecting a "good" solution from this infinite family, typically by preferring weight vectors with small norm.

Information-Theoretic Connections

Throughout this tutorial series, we view mathematics through the lens of information theory. Vectors and vector spaces connect to information theory in several deep and practical ways that go far beyond analogy.

Vectors as Probability Distributions

A probability distribution over $k$ outcomes can be represented as a vector $\mathbf{p} = (p_1, p_2, \ldots, p_k)$ in $\mathbb{R}^k$ with the constraints $p_i \geq 0$ and $\sum p_i = 1$. This vector lives on the probability simplex $\Delta^{k-1}$, a $(k-1)$-dimensional surface embedded in $\mathbb{R}^k$. (It's $(k-1)$-dimensional because the constraint $\sum p_i = 1$ removes one degree of freedom.) The simplex is not a subspace — as we saw earlier, it fails closure under scalar multiplication — but it inherits geometric structure from the ambient vector space.

The entropy of a distribution $\mathbf{p}$ is:

$$H(\mathbf{p}) = -\sum_{i=1}^{k} p_i \log p_i$$

This is a function from the probability simplex to $\mathbb{R}$. Entropy is maximized by the uniform distribution $\mathbf{u} = (1/k, \ldots, 1/k)$ — the "center" of the simplex — and minimized (at zero) by any corner of the simplex where all probability mass is on a single outcome. You can think of entropy as measuring how "spread out" a distribution is, which is directly related to the norm of the distribution vector (more uniform = closer to the center of the simplex = higher entropy).

Vector Norms and Information Measures

There is a beautiful connection between $L^p$ norms and Rényi entropy. The Rényi entropy of order $\alpha$ is defined as:

$$H_\alpha(\mathbf{p}) = \frac{1}{1-\alpha} \log\left(\sum_{i=1}^k p_i^\alpha\right) = \frac{\alpha}{1-\alpha}\log\|\mathbf{p}\|_\alpha$$

When $\alpha \to 1$, the Rényi entropy converges to the Shannon entropy $H(\mathbf{p})$. The quantity $\sum p_i^\alpha$ is precisely $\|\mathbf{p}\|_\alpha^\alpha$ — the $L^\alpha$ norm raised to the $\alpha$ power. So the Rényi entropy is literally a function of the vector norm of the probability distribution! This is not a metaphor: the norm of a probability vector directly encodes information content.

Embedding Spaces in NLP

Modern NLP models represent words, sentences, and documents as vectors in learned embedding spaces. These spaces have remarkable algebraic structure: the vector difference $\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}}$ captures the concept of "royalty," and adding it to $\mathbf{v}_{\text{woman}}$ yields a vector close to $\mathbf{v}_{\text{queen}}$. This works because the embedding space is (approximately) a vector space where linear operations preserve semantic relationships.

From an information-theoretic perspective, a good embedding is one that compresses the relevant information about an input while discarding noise. The Information Bottleneck principle formalizes this: an optimal representation $\mathbf{z}$ of input $\mathbf{x}$ for predicting target $\mathbf{y}$ minimizes $I(\mathbf{x}; \mathbf{z}) - \beta I(\mathbf{z}; \mathbf{y})$, where $I$ denotes mutual information. The embedding vectors that neural networks learn are approximate solutions to this compression-relevance tradeoff. The dimension of the embedding space controls how much information can be retained — a direct connection between vector space dimension and information capacity.

Common Misconceptions

Even experienced students carry misconceptions about vectors that can lead to subtle errors. Let's address the most common ones head-on.

Misconception 1: Vectors Are Arrows

Arrows in 2D or 3D are one example of vectors, but far from the only one. Polynomials are vectors. Functions are vectors. Matrices are vectors (in the vector space of matrices). Random variables are vectors. Even infinite sequences can be vectors. The arrow picture is useful for building intuition in low dimensions, but don't let it limit your thinking. In ML, most vectors have hundreds or thousands of components and cannot be visualized as arrows at all. The algebraic properties — not the geometric picture — are what matter.

Misconception 2: Dimension Always Equals the Number of Components

A vector in $\mathbb{R}^3$ has 3 components, and $\mathbb{R}^3$ has dimension 3. But suppose your vectors all lie on a plane through the origin within $\mathbb{R}^3$ — then the subspace they inhabit is only 2-dimensional, even though each vector has 3 components. The intrinsic dimension of a dataset (the dimension of the smallest subspace containing it) is often much smaller than the ambient dimension (the number of components).

This distinction is at the heart of the manifold hypothesis in deep learning: real-world data (images, text, audio) nominally lives in very high-dimensional spaces but actually occupies low-dimensional manifolds within those spaces. Dimensionality reduction techniques exploit this gap.

Misconception 3: The Zero Vector Is "Nothing"

The zero vector $\mathbf{0}$ is not "nothing" — it's the additive identity, and it lives in every subspace. You can't have a subspace without it. The zero vector is the trivial linear combination (all coefficients zero), and it's always in the span of any set of vectors. A set containing only the zero vector $\{\mathbf{0}\}$ is itself a subspace — the smallest possible one, with dimension 0. Don't confuse "zero vector" with "empty set."

Misconception 4: Linear Independence Means Orthogonality

Orthogonal vectors (with dot product zero) are always linearly independent, but the converse is false. The vectors $\begin{pmatrix} 1 \\ 0 \end{pmatrix}$ and $\begin{pmatrix} 1 \\ 1 \end{pmatrix}$ are linearly independent (neither is a multiple of the other) but not orthogonal ($\begin{pmatrix} 1 \\ 0 \end{pmatrix} \cdot \begin{pmatrix} 1 \\ 1 \end{pmatrix} = 1 \neq 0$). Independence is a weaker condition than orthogonality. In fact, in $\mathbb{R}^n$ you can have at most $n$ mutually orthogonal nonzero vectors, but you can have $n$ linearly independent vectors that point in all sorts of non-perpendicular directions.

Exercises

Easy Problems 🟢

  1. Compute the sum $\begin{pmatrix} 2 \\ -1 \\ 3 \end{pmatrix} + \begin{pmatrix} -1 \\ 4 \\ 2 \end{pmatrix}$ and the scalar multiple $3\begin{pmatrix} 2 \\ -1 \\ 3 \end{pmatrix}$.
  2. Given $\mathbf{u} = \begin{pmatrix} 1 \\ 2 \end{pmatrix}$ and $\mathbf{v} = \begin{pmatrix} 3 \\ -1 \end{pmatrix}$, compute $2\mathbf{u} - 3\mathbf{v}$.
  3. Compute the dot product $\begin{pmatrix} 1 \\ 2 \\ 3 \end{pmatrix} \cdot \begin{pmatrix} 4 \\ -1 \\ 2 \end{pmatrix}$.
  4. Find the Euclidean norm ($L^2$ norm) of the vector $\mathbf{v} = \begin{pmatrix} 3 \\ 4 \end{pmatrix}$.
  5. Determine whether the set $W = \{(x, y) \in \mathbb{R}^2 : x + y = 0\}$ is a subspace of $\mathbb{R}^2$. Justify your answer.
  6. Write the vector $\begin{pmatrix} 5 \\ -2 \\ 7 \end{pmatrix}$ as a linear combination of the standard basis vectors $\mathbf{e}_1, \mathbf{e}_2, \mathbf{e}_3$.
  7. True or false: The set $\{(x, y) \in \mathbb{R}^2 : x \geq 0\}$ (the right half-plane) is a subspace of $\mathbb{R}^2$. Explain.
  8. Compute the $L^1$ norm and $L^\infty$ norm of $\mathbf{v} = \begin{pmatrix} -3 \\ 1 \\ -4 \\ 2 \end{pmatrix}$.

Medium Problems 🟡

  1. Determine whether $\begin{pmatrix} 4 \\ 1 \\ 3 \end{pmatrix}$ is in the span of $\begin{pmatrix} 1 \\ 0 \\ 1 \end{pmatrix}$ and $\begin{pmatrix} 0 \\ 1 \\ -1 \end{pmatrix}$. If yes, find the coefficients.
  2. Are the vectors $\begin{pmatrix} 1 \\ 2 \\ 0 \end{pmatrix}$, $\begin{pmatrix} 0 \\ 1 \\ 1 \end{pmatrix}$, $\begin{pmatrix} 1 \\ 0 \\ -2 \end{pmatrix}$ linearly independent? Prove your answer.
  3. Find the projection of $\mathbf{u} = \begin{pmatrix} 3 \\ 4 \end{pmatrix}$ onto $\mathbf{v} = \begin{pmatrix} 1 \\ 1 \end{pmatrix}$. What is the component of $\mathbf{u}$ orthogonal to $\mathbf{v}$?
  4. Let $B = \left\{\begin{pmatrix} 1 \\ 1 \end{pmatrix}, \begin{pmatrix} 1 \\ -1 \end{pmatrix}\right\}$ be a basis for $\mathbb{R}^2$. Find the coordinates of $\begin{pmatrix} 5 \\ 3 \end{pmatrix}$ in basis $B$.
  5. Show that the set of all $2 \times 2$ symmetric matrices forms a subspace of the vector space of all $2 \times 2$ matrices. What is the dimension of this subspace?
  6. Compute the cosine similarity between $\mathbf{u} = \begin{pmatrix} 1 \\ 2 \\ 2 \end{pmatrix}$ and $\mathbf{v} = \begin{pmatrix} 2 \\ 1 \\ -2 \end{pmatrix}$. Are they more similar or dissimilar?
  7. Find a basis for the null space of the matrix $A = \begin{pmatrix} 1 & 2 & 3 \\ 2 & 4 & 6 \end{pmatrix}$. What is the nullity?
  8. Prove that for any vector $\mathbf{v} \in \mathbb{R}^n$, $\|\mathbf{v}\|_\infty \leq \|\mathbf{v}\|_2 \leq \|\mathbf{v}\|_1$.

Hard Problems 🔴

  1. Prove the Cauchy-Schwarz inequality $|\mathbf{u} \cdot \mathbf{v}| \leq \|\mathbf{u}\| \|\mathbf{v}\|$ for $\mathbf{u}, \mathbf{v} \in \mathbb{R}^n$ using the fact that $\|t\mathbf{u} + \mathbf{v}\|^2 \geq 0$ for all $t \in \mathbb{R}$.
  2. Let $W_1$ and $W_2$ be subspaces of $\mathbb{R}^n$. Prove that $W_1 \cap W_2$ is a subspace. Is $W_1 \cup W_2$ always a subspace? If not, give a counterexample.
  3. Prove that if $\{\mathbf{v}_1, \ldots, \mathbf{v}_k\}$ is linearly independent and $\mathbf{v}_{k+1} \notin \text{span}(\mathbf{v}_1, \ldots, \mathbf{v}_k)$, then $\{\mathbf{v}_1, \ldots, \mathbf{v}_k, \mathbf{v}_{k+1}\}$ is linearly independent.
  4. Show that the set of polynomials $\{1, x - a, (x - a)^2, \ldots, (x - a)^n\}$ is a basis for $\mathcal{P}_n$ for any $a \in \mathbb{R}$. (This is the Taylor basis centered at $a$.)
  5. Let $\mathbf{p} = (p_1, \ldots, p_k)$ be a probability distribution. Show that $H(\mathbf{p}) \leq \log k$ (Shannon entropy is maximized by the uniform distribution), and express this bound in terms of the $L^2$ norm of $\mathbf{p}$.
  6. Prove the rank-nullity theorem: for an $m \times n$ matrix $A$, $\text{rank}(A) + \text{nullity}(A) = n$. (Hint: start by choosing a basis for the null space, extend it to a basis for $\mathbb{R}^n$, and show the images of the extension vectors form a basis for the column space.)
  7. Given a set of word embeddings $\mathbf{w}_1, \ldots, \mathbf{w}_N \in \mathbb{R}^d$, describe an algorithm to find the minimum dimension $k$ such that the projections onto a $k$-dimensional subspace preserve at least 95% of the total variance. Connect your algorithm to the SVD or eigenvalue decomposition.
  8. Prove that any $n$-dimensional subspace of $\mathbb{R}^n$ must be $\mathbb{R}^n$ itself. Use this to explain why a set of $n$ linearly independent vectors in $\mathbb{R}^n$ must span all of $\mathbb{R}^n$.
