07. Regularization

Regularization prevents overfitting by adding constraints to learning. From a Bayesian view, it's just incorporating prior beliefs about good solutions.

Theory Code Exercises

The Overfitting Problem

A model that's too flexible can memorize training data rather than learn underlying patterns. It achieves low training error but high test error.

Bias-Variance Tradeoff

$$\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Noise}$$ Simple models have high bias (underfitting). Complex models have high variance (overfitting). Regularization reduces variance at the cost of some bias.

L2 Regularization (Ridge/Weight Decay)

Add a penalty proportional to the squared magnitude of weights:

$$L_{\text{reg}} = L_{\text{data}} + \frac{\lambda}{2} \|\mathbf{w}\|_2^2 = L_{\text{data}} + \frac{\lambda}{2} \sum_i w_i^2$$
Bayesian Interpretation
1
Assume Gaussian prior: $p(\mathbf{w}) = \mathcal{N}(\mathbf{0}, \sigma_w^2 \mathbf{I})$
2
MAP estimation: $\hat{\mathbf{w}} = \arg\max_{\mathbf{w}} p(\mathbf{w}|\mathcal{D}) \propto p(\mathcal{D}|\mathbf{w}) p(\mathbf{w})$
3
Take negative log: $-\log p(\mathbf{w}|\mathcal{D}) = -\log p(\mathcal{D}|\mathbf{w}) + \frac{1}{2\sigma_w^2}\|\mathbf{w}\|^2 + \text{const}$
4
This is exactly L2 regularization with $\lambda = 1/\sigma_w^2$!

Effect: L2 shrinks weights toward zero but rarely makes them exactly zero. It prefers many small weights over few large weights.

L1 Regularization (Lasso)

$$L_{\text{reg}} = L_{\text{data}} + \lambda \|\mathbf{w}\|_1 = L_{\text{data}} + \lambda \sum_i |w_i|$$

Bayesian view: L1 corresponds to a Laplace prior on weights:

$$p(w) = \frac{\lambda}{2} e^{-\lambda|w|}$$
Sparsity

L1 regularization drives many weights to exactly zero, producing sparse solutions. This is useful for feature selection—zero weights mean the corresponding features are ignored.

Dropout

During training, randomly set each neuron's output to zero with probability $p$:

$$\tilde{h}_i = \begin{cases} 0 & \text{with probability } p \\ h_i / (1-p) & \text{with probability } 1-p \end{cases}$$

The division by $(1-p)$ keeps expected values unchanged.

Interpretations of Dropout

Ensemble view: Training exponentially many sub-networks
Noise injection: Makes model robust to missing features
Bayesian view: Approximate variational inference over network weights

Other Regularization Techniques

Early Stopping

Stop training when validation error starts increasing. Simple and effective— implicitly regularizes by limiting optimization steps.

Data Augmentation

Create more training data through transformations (rotations, crops, noise). Forces the model to learn invariances.

Batch Normalization

Has regularizing effects due to noise from mini-batch statistics. Often allows reducing dropout.

Label Smoothing

Instead of hard labels (0 or 1), use soft labels (0.1 or 0.9). Prevents overconfidence and improves calibration.

$$y_{\text{smooth}} = (1 - \epsilon) y_{\text{hard}} + \frac{\epsilon}{K}$$
