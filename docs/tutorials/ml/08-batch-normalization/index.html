09. Batch Normalization

Batch normalization revolutionized deep learning by enabling much deeper networks to train. Let's understand exactly what it does and why it works.

The Internal Covariate Shift Problem

As training progresses, the distribution of each layer's inputs changes because the parameters of preceding layers change. This makes training difficult—each layer must continuously adapt to new input distributions.

Internal Covariate Shift

The change in the distribution of network activations due to the change in network parameters during training. Batch normalization reduces this by normalizing layer inputs.

The BatchNorm Algorithm

For a mini-batch $\mathcal{B} = \{x_1, \ldots, x_m\}$ of size $m$:

Batch Normalization Transform
1
Compute batch mean: $\mu_\mathcal{B} = \frac{1}{m}\sum_{i=1}^{m} x_i$
2
Compute batch variance: $\sigma_\mathcal{B}^2 = \frac{1}{m}\sum_{i=1}^{m} (x_i - \mu_\mathcal{B})^2$
3
Normalize: $\hat{x}_i = \frac{x_i - \mu_\mathcal{B}}{\sqrt{\sigma_\mathcal{B}^2 + \epsilon}}$
4
Scale and shift: $y_i = \gamma \hat{x}_i + \beta$

The parameters $\gamma$ (scale) and $\beta$ (shift) are learnable. They allow the network to undo the normalization if that's optimal.

Why $\gamma$ and $\beta$?

Without them, the representation power would be limited. With $\gamma = \sigma$ and $\beta = \mu$, we recover the original activations. The network learns the optimal normalization.

Training vs Inference

During training, we use mini-batch statistics. But at inference time, we might have a single example. Solution: maintain running averages.

$$\mu_{\text{running}} = \alpha \mu_{\text{running}} + (1-\alpha) \mu_\mathcal{B}$$ $$\sigma^2_{\text{running}} = \alpha \sigma^2_{\text{running}} + (1-\alpha) \sigma^2_\mathcal{B}$$

At inference, use $\mu_{\text{running}}$ and $\sigma^2_{\text{running}}$ instead of batch statistics.

Backpropagation Through BatchNorm

The backward pass is more complex because each $\hat{x}_i$ depends on the entire batch through $\mu$ and $\sigma^2$.

Gradients
1
$\frac{\partial L}{\partial \gamma} = \sum_i \frac{\partial L}{\partial y_i} \hat{x}_i$
2
$\frac{\partial L}{\partial \beta} = \sum_i \frac{\partial L}{\partial y_i}$
3
$\frac{\partial L}{\partial x_i} = \frac{\gamma}{\sqrt{\sigma^2 + \epsilon}}\left(\frac{\partial L}{\partial y_i} - \frac{1}{m}\sum_j \frac{\partial L}{\partial y_j} - \frac{\hat{x}_i}{m}\sum_j \frac{\partial L}{\partial y_j}\hat{x}_j\right)$

Why Does BatchNorm Work?

The original paper cited internal covariate shift, but the full story is more nuanced:

  • Smoother loss landscape: BatchNorm makes the optimization surface more well-conditioned, allowing larger learning rates.
  • Regularization: The noise from batch statistics acts as regularization, similar to dropout.
  • Gradient flow: Normalized activations help gradients flow better through deep networks.
Variants

Layer Norm: Normalize across features (used in transformers)
Instance Norm: Normalize each sample independently (used in style transfer)
Group Norm: Normalize across groups of channels (works with small batches)
