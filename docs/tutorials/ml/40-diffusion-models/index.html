The Core Idea: Destruction and Reconstruction

Imagine a photograph. Copy it, adding a tiny bit of static. Copy the copy, adding more static. Repeat 1000 times. The final copy is pure noise.

Now imagine learning to reverse each tiny step. If you chain 1000 denoising steps starting from pure noise, you reconstruct a realistic image.

Why Diffusion Works

Forward process is fixed: No learning needed — just add Gaussian noise
Each reverse step is small: Small noise steps have approximately Gaussian reverse distributions
Denoising is well-studied: Neural networks are excellent denoisers
The loss is simple: Just predict the noise that was added (MSE loss)

The Forward Process

Starting from data $x_0 \sim q(x_0)$, add noise gradually over $T$ steps:

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t} \, x_{t-1}, \beta_t I)$$

We can sample $x_t$ directly from $x_0$ using the reparameterization trick:

$$x_t = \sqrt{\bar{\alpha}_t} \, x_0 + \sqrt{1 - \bar{\alpha}_t} \, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

where $\bar{\alpha}_t = \prod_{s=1}^t (1-\beta_s)$.

Signal vs. Noise

$\sqrt{\bar{\alpha}_t}$ = signal strength (decreases from 1 to ~0)
$\sqrt{1-\bar{\alpha}_t}$ = noise strength (increases from ~0 to 1)
Signal-to-noise ratio: $\text{SNR}(t) = \bar{\alpha}_t / (1 - \bar{\alpha}_t)$

Noise Schedules

Linear schedule: $\beta_t = \beta_{\min} + \frac{t-1}{T-1}(\beta_{\max} - \beta_{\min})$, typically $\beta_{\min} = 10^{-4}$, $\beta_{\max} = 0.02$, $T = 1000$.

Cosine schedule: $\bar{\alpha}_t = \frac{f(t)}{f(0)}$ where $f(t) = \cos\left(\frac{t/T + s}{1+s} \cdot \frac{\pi}{2}\right)^2$.

The cosine schedule provides a more uniform rate of information destruction, fixing the problem that the linear schedule wastes early steps.

The Reverse Process

We learn the reverse transition:

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \sigma_t^2 I)$$

When conditioned on $x_0$, the tractable posterior is:

$$q(x_{t-1} | x_t, x_0) = \mathcal{N}(x_{t-1}; \tilde{\mu}_t(x_t, x_0), \tilde{\beta}_t I)$$

where $\tilde{\mu}_t = \frac{\sqrt{\bar{\alpha}_{t-1}} \beta_t}{1 - \bar{\alpha}_t} x_0 + \frac{\sqrt{\alpha_t}(1-\bar{\alpha}_{t-1})}{1 - \bar{\alpha}_t} x_t$

The Training Objective

Ho et al. (2020) showed that a simplified loss works remarkably well:

$$L_{simple} = \mathbb{E}_{t, x_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]$$
DDPM Training Algorithm
1
Sample $x_0$ from data, $t \sim \text{Uniform}\{1, ..., T\}$, $\epsilon \sim \mathcal{N}(0, I)$
2
Compute $x_t = \sqrt{\bar{\alpha}_t} x_0 + \sqrt{1-\bar{\alpha}_t} \epsilon$
3
Loss: $\|\epsilon - \epsilon_\theta(x_t, t)\|^2$ — just predict the noise!
4
Update $\theta$ via gradient descent

Three Parameterizations

The model can equivalently predict different quantities:

  • Predict noise ($\epsilon$-prediction): $\mu_\theta = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}} \epsilon_\theta\right)$
  • Predict clean data ($x_0$-prediction): $\hat{x}_0 = f_\theta(x_t, t)$
  • Predict score ($s$-prediction): $s_\theta(x_t, t) \approx \nabla_{x_t} \log q(x_t)$

All three are related: $\epsilon = -\sqrt{1-\bar{\alpha}_t} \cdot s = \frac{x_t - \sqrt{\bar{\alpha}_t} \hat{x}_0}{\sqrt{1-\bar{\alpha}_t}}$

Score Matching and Langevin Dynamics

The score $s(x) = \nabla_x \log p(x)$ points toward regions of higher probability. Given the score, we can sample using Langevin dynamics:

$$x_{t+1} = x_t + \eta \nabla_x \log p(x_t) + \sqrt{2\eta} \, z, \quad z \sim \mathcal{N}(0, I)$$

This converges to samples from $p(x)$ as $\eta \to 0$ and number of steps $\to \infty$.

The SDE Perspective

The discrete forward process becomes a continuous SDE: $dx = f(x, t) dt + g(t) dw$

The reverse-time SDE (Anderson, 1982):

$$dx = [f(x,t) - g(t)^2 \nabla_x \log p_t(x)] dt + g(t) d\bar{w}$$

The probability flow ODE (deterministic, same marginals):

$$dx = \left[f(x,t) - \frac{1}{2}g(t)^2 \nabla_x \log p_t(x)\right] dt$$

This is equivalent to a continuous normalizing flow — connecting back to flow matching!

Guidance and DDIM

Classifier-Free Guidance: Combine conditional and unconditional predictions:

$$\tilde{\epsilon}_\theta = (1 + w) \epsilon_\theta(x_t, t, y) - w \cdot \epsilon_\theta(x_t, t, \emptyset)$$

This is what makes text-to-image models like DALL-E and Stable Diffusion work.

DDIM (Deterministic Sampling): Setting $\sigma_t = 0$ gives a deterministic mapping — same noise always produces the same output. DDIM can generate good samples in 10-50 steps (vs 1000 for DDPM).

Connections and Unification

All frameworks are mathematically equivalent — same model, different perspectives:

  • DDPM (noise predictor, stochastic, 1000 steps)
  • Score-based (score function, Langevin/reverse SDE, 1000 steps)
  • DDIM (same model, deterministic ODE, 10-50 steps)
  • Flow matching (velocity field, ODE with OT path, 5-20 steps)
Information-Theoretic View

Forward process = adding entropy (destroying information). Reverse process = removing entropy (reconstructing information). Score function = gradient of information content. Training = learning the rate of entropy change at each noise level.

39. Diffusion Models

Diffusion models gradually destroy data with noise, then learn to reverse the process. The simple denoising loss connects to score matching, variational bounds, and flow matching — unifying the modern landscape of generative models.
