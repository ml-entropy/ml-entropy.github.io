10. Convolutional Neural Networks

CNNs exploit the spatial structure of images through local connectivity and weight sharing. Let's understand why convolutions are so effective and derive the math behind them.

Why Convolutions?

Fully connected layers don't scale to images. A 256×256 RGB image has 196,608 pixels. Connecting each to 1000 hidden units requires 196 million parameters—per layer!

Key Insights for Images

Local connectivity: Pixels are most related to nearby pixels
Translation invariance: A cat is a cat regardless of position
Compositionality: Complex features build from simple ones

Convolutions encode these priors directly into the architecture.

The Convolution Operation

A convolution slides a small kernel (filter) across the input:

$$(I * K)_{i,j} = \sum_{m} \sum_{n} I_{i+m, j+n} \cdot K_{m,n}$$

For an input with $C_{in}$ channels and output with $C_{out}$ channels:

$$Y_{c_{out}, i, j} = \sum_{c_{in}=1}^{C_{in}} \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} X_{c_{in}, i+m, j+n} \cdot W_{c_{out}, c_{in}, m, n} + b_{c_{out}}$$
Parameter Sharing

The same kernel is applied at every position. This dramatically reduces parameters and forces the network to learn translation-invariant features.

Output Size Calculation

For input size $W$, kernel size $K$, padding $P$, and stride $S$:

$$W_{out} = \left\lfloor \frac{W - K + 2P}{S} \right\rfloor + 1$$
Common Configurations
1
Same padding: $P = (K-1)/2$, $S = 1$ → output same size as input
2
Valid padding: $P = 0$ → output shrinks by $K-1$
3
Stride 2: Halves the spatial dimensions (downsampling)

Pooling Layers

Pooling reduces spatial dimensions while maintaining important features:

Max Pooling

$$Y_{i,j} = \max_{m,n \in \text{window}} X_{i \cdot s + m, j \cdot s + n}$$

Average Pooling

$$Y_{i,j} = \frac{1}{k^2} \sum_{m,n \in \text{window}} X_{i \cdot s + m, j \cdot s + n}$$

Max pooling is more common—it selects the strongest activation, providing some translation invariance within the pooling window.

Receptive Field

The receptive field is the region of input that affects a given output neuron.

$$R_l = R_{l-1} + (k_l - 1) \prod_{i=1}^{l-1} s_i$$

Where $R_l$ is the receptive field at layer $l$, $k_l$ is kernel size, and $s_i$ is stride.

Design Principle

Stack small kernels (3×3) rather than using large ones. Two 3×3 layers have the same receptive field as one 5×5 layer but fewer parameters and more non-linearity.

Backpropagation Through Convolutions

The backward pass of a convolution is also a convolution—with the kernel flipped!

$$\frac{\partial L}{\partial X} = \frac{\partial L}{\partial Y} * \text{flip}(W)$$

The gradient with respect to the kernel:

$$\frac{\partial L}{\partial W} = X * \frac{\partial L}{\partial Y}$$
