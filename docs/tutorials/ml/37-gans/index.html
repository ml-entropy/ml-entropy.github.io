The Generative Modeling Problem

The fundamental problem: given samples from an unknown distribution $p_{data}(x)$, learn to generate new samples that look like they came from the same distribution.

GANs take the implicit density approach: we never write down $p_g(x)$, but we learn a function that transforms noise into realistic samples.

Three Approaches to Generative Modeling

Explicit density, tractable: Directly model $p_\theta(x)$ with exact evaluation (Normalizing Flows, Autoregressive)
Explicit density, approximate: Approximate $p_\theta(x)$ via lower bound (VAEs)
Implicit density: Learn to sample, no explicit density (GANs)

The Adversarial Framework

The GAN framework involves two players in an adversarial game:

Generator $G$: Takes random noise $z \sim p_z(z)$ and maps it to data space: $x_{fake} = G(z)$. Like a counterfeiter trying to produce fake currency.

Discriminator $D$: Takes a data point $x$ and outputs the probability that it came from the real data: $D(x) \in [0, 1]$. Like a detective catching counterfeits.

The minimax game is:

$$\min_G \max_D V(D, G) = \mathbb{E}_{x \sim p_{data}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$
Parsing the Objective

First term: For real data, D wants $D(x) \to 1$, making $\log D(x) \to 0$ (maximized)
Second term: For fake data, D wants $D(G(z)) \to 0$, making $\log(1-0) = 0$ (maximized). But G wants $D(G(z)) \to 1$, making $\log(1-1) \to -\infty$ (minimized)

The Optimal Discriminator

Proposition: For a fixed generator $G$, the optimal discriminator is:

$$D^*_G(x) = \frac{p_{data}(x)}{p_{data}(x) + p_g(x)}$$
Proof
1
The training objective for $D$ is: $V(D, G) = \int_x [p_{data}(x) \log D(x) + p_g(x) \log(1 - D(x))] dx$
2
For each $x$, maximize $f(D) = a \log D + b \log(1 - D)$ where $a = p_{data}(x)$, $b = p_g(x)$
3
Setting $\frac{a}{D} - \frac{b}{1-D} = 0$ gives $D^* = \frac{a}{a+b} = \frac{p_{data}(x)}{p_{data}(x) + p_g(x)}$

Intuition: When $p_g = p_{data}$, we get $D^*(x) = 1/2$ — the detective is completely stumped.

Jensen-Shannon Divergence

Substituting $D^*$ back into the value function reveals that the generator minimizes:

$$V(D^*, G) = 2 \cdot JSD(p_{data} \| p_g) - \log 4$$

The JSD is a symmetrized, smoothed version of the KL divergence:

$$JSD(P \| Q) = \frac{1}{2} D_{KL}\left(P \| \frac{P+Q}{2}\right) + \frac{1}{2} D_{KL}\left(Q \| \frac{P+Q}{2}\right)$$
Global Optimum

The global minimum of $V(D^*, G)$ is achieved if and only if $p_g = p_{data}$, at which point $V(D^*, G) = -\log 4$ and $D^*(x) = 1/2$ for all $x$.

Training Challenges

GAN training is a saddle-point optimization (minimax), not a simple minimization. This leads to several problems:

Mode Collapse: The generator learns to produce only a few types of samples that fool the discriminator, rather than covering the full data distribution. The minimax game may not converge to the Nash equilibrium — instead, the players oscillate.

Training Instability: Oscillating losses, vanishing gradients (if D becomes too good), and mode dropping.

The Non-Saturating Fix

In practice, $\log(1 - D(G(z)))$ saturates when $D(G(z)) \approx 0$ (early training). Instead of minimizing $\log(1 - D(G(z)))$, the generator maximizes $\log D(G(z))$. This provides stronger gradients early in training.

Wasserstein GAN (WGAN)

JSD has a fundamental flaw: when $p_{data}$ and $p_g$ have non-overlapping supports, $JSD = \log 2$ regardless of how close the distributions are. The gradient is zero — the generator gets no signal!

The Wasserstein-1 distance (Earth Mover's Distance) fixes this:

$$W_1(P, Q) = \inf_{\gamma \in \Pi(P,Q)} \mathbb{E}_{(x,y) \sim \gamma}[\|x - y\|]$$

Intuition: Think of $P$ as a pile of dirt and $Q$ as a hole. $W_1$ is the minimum work needed to move the dirt into the hole.

Using the Kantorovich-Rubinstein duality, the WGAN objective replaces the discriminator with a 1-Lipschitz "critic":

$$\min_G \max_{\|D\|_L \leq 1} \mathbb{E}_{x \sim p_{data}}[D(x)] - \mathbb{E}_{z \sim p_z}[D(G(z))]$$
Gradient Penalty (WGAN-GP)

To enforce the Lipschitz constraint, add a penalty: $\lambda \mathbb{E}_{\hat{x}}[(\|\nabla_{\hat{x}} D(\hat{x})\|_2 - 1)^2]$ where $\hat{x} = \alpha x_{real} + (1-\alpha) x_{fake}$, $\alpha \sim U(0,1)$. This directly penalizes the critic for having gradient norms different from 1.

GAN Variants and Extensions

Conditional GAN (cGAN): Condition both G and D on additional information $y$ (e.g., class label).

DCGAN: Architecture guidelines — strided convolutions, batch normalization, ReLU in G, LeakyReLU in D.

StyleGAN: Mapping network $z \to w$, adaptive instance normalization (AdaIN), noise injection. Produces the most photorealistic face generation.

CycleGAN: Unpaired image-to-image translation. Pix2Pix: Paired image-to-image translation.

Connections to Information Theory

Different GAN formulations minimize different divergences between distributions:

  • Original GAN — Jensen-Shannon divergence
  • f-GAN — Any f-divergence
  • WGAN — Wasserstein-1 distance
  • MMD-GAN — Maximum Mean Discrepancy

From the information-theoretic perspective: the generator compresses the data distribution into a deterministic mapping from noise, the discriminator measures how much information distinguishes real from fake, and at equilibrium there is zero mutual information between the real/fake label and the data.

36. Generative Adversarial Networks

GANs set up a game between two neural networks — a generator and a discriminator — where competition drives both to improve. The generator minimizes the Jensen-Shannon divergence, and Wasserstein GANs fix fundamental training problems with the Earth Mover's Distance.
