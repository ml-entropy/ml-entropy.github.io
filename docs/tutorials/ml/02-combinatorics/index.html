Counting Principles

Two fundamental rules govern all counting:

The Two Rules

Product Rule: If task A has $m$ ways and task B has $n$ ways, then A followed by B has $m \times n$ ways.

Sum Rule: If A has $m$ ways and B has $n$ ways (mutually exclusive), then A or B has $m + n$ ways.

Permutations

A permutation is an ordered arrangement. The number of ways to arrange $n$ distinct objects:

$$n! = n \times (n-1) \times (n-2) \times \cdots \times 1$$

The number of ways to choose and arrange $k$ objects from $n$:

$$P(n,k) = \frac{n!}{(n-k)!}$$
Stirling's Approximation

For large $n$: $n! \approx \sqrt{2\pi n}\left(\frac{n}{e}\right)^n$
Taking logs: $\log(n!) \approx n\log n - n$

Combinations

A combination is an unordered selection. The binomial coefficient:

$$\binom{n}{k} = \frac{n!}{k!(n-k)!} = \frac{P(n,k)}{k!}$$
Key Properties
1
Symmetry: $\binom{n}{k} = \binom{n}{n-k}$
2
Pascal's rule: $\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}$
3
Sum: $\sum_{k=0}^{n} \binom{n}{k} = 2^n$
4
Binomial theorem: $(x+y)^n = \sum_{k=0}^{n} \binom{n}{k} x^k y^{n-k}$

The Binomial Distribution

The probability of $k$ successes in $n$ independent trials with success probability $p$:

$$P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}$$
Understanding the Formula
1
$p^k$ — probability of exactly $k$ successes
2
$(1-p)^{n-k}$ — probability of exactly $n-k$ failures
3
$\binom{n}{k}$ — number of ways to arrange $k$ successes among $n$ trials

Mean: $\mathbb{E}[X] = np$, Variance: $\text{Var}(X) = np(1-p)$

Applications in ML

Model Complexity

How many possible decision trees of depth $d$ with $n$ features? How many neural network architectures? These questions are combinatorial.

VC Dimension

A model can shatter $n$ points if it can achieve all $2^n$ possible labelings. The VC dimension relates to the number of parameters via combinatorial arguments.

Cross-Validation

$k$-fold cross-validation partitions data into $\binom{n}{n/k}$ possible splits. Leave-one-out uses all $n$ possible held-out points.

The Curse of Dimensionality

In $d$ dimensions with $k$ bins per dimension, we have $k^d$ total bins. This exponential growth is why high-dimensional spaces are "mostly empty"— data becomes sparse, and nearest neighbors become meaningless.

Counting is the foundation of discrete probability. From the binomial distribution to understanding model complexity, combinatorics underpins many ML concepts.
