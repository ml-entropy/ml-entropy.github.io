Parsing Algorithms

Parsing is the process of recovering the hidden structure of a sentence from its flat, linear surface form. Every NLP system that needs to understand grammatical relationships — from machine translation to question answering — relies on parsing, whether explicitly or implicitly. In this tutorial, we cover the two major paradigms (constituency and dependency parsing) and the key algorithms for each.

1. What is Parsing?

A parser takes a sentence (a sequence of words) and produces a structural description. There are two main formalisms:

  • Constituency parsing: Groups words into nested phrases (NP, VP, PP). The output is a phrase structure tree.
  • Dependency parsing: Connects each word to its syntactic head. The output is a dependency tree.

Two Views of "The cat sat on the mat"

Constituency:

[S [NP [Det The] [N cat]] [VP [V sat] [PP [P on] [NP [Det the] [N mat]]]]]

Dependency:

sat ←nsubj cat ←det The
sat →obl mat ←det the
mat ←case on

Both capture "The cat" as subject of "sat" and "on the mat" as a locative modifier, but in different structural representations.

2. Constituency Parsing

Constituency parsing builds a tree where internal nodes are phrase labels (S, NP, VP) and leaves are words. The tree is defined by a context-free grammar (see Tutorial 07).

Chomsky Normal Form

Many parsing algorithms require the grammar to be in Chomsky Normal Form (CNF), where every rule has one of two forms:

$$A \to B \; C \quad \text{or} \quad A \to a$$

where $A, B, C$ are non-terminals and $a$ is a terminal. Any CFG can be converted to CNF without changing the language it generates.

Ambiguity

A sentence is structurally ambiguous if the grammar assigns it multiple parse trees. Real grammars for English produce enormous numbers of parses: a 20-word sentence can easily have thousands of valid trees. The parser must either enumerate all of them or find the best one (using a probabilistic model).

3. The CYK Algorithm

CYK (Cocke-Younger-Kasami) Algorithm

A dynamic programming algorithm for parsing with CFGs in Chomsky Normal Form. It fills a triangular chart $\text{table}[i][j]$ where each cell contains the set of non-terminals that can generate the substring $w_i \ldots w_j$.

Complexity: $O(n^3 \cdot |G|)$ where $n$ is the sentence length and $|G|$ is the grammar size.

Algorithm Outline

  1. Initialize: For each word $w_i$, set $\text{table}[i][i] = \{A \mid A \to w_i \in R\}$
  2. Fill: For each span length $\ell = 2, 3, \ldots, n$:
    For each start position $i$:
     For each split point $k$ between $i$ and $i + \ell - 1$:
      If $B \in \text{table}[i][k]$ and $C \in \text{table}[k+1][i+\ell-1]$ and $A \to BC \in R$:
       Add $A$ to $\text{table}[i][i+\ell-1]$
  3. Accept: $S \in \text{table}[1][n]$ iff the sentence is in the language

CYK Example

Grammar (in CNF):

$S \to NP \; VP$, $NP \to Det \; N$, $VP \to V \; NP$
$Det \to \text{the}$, $N \to \text{cat} \mid \text{fish}$, $V \to \text{ate}$

Sentence: "the cat ate the fish"

thecatatethefish
1{Det}{NP}——{S}
2{N}——{VP}
3{V}—{VP}
4{Det}{NP}
5{N}

$S$ appears in $\text{table}[1][5]$, confirming the sentence is grammatical.

Probabilistic CYK

For PCFGs, the algorithm is modified to store the highest-probability non-terminal in each cell (Viterbi parsing). Instead of sets, each cell stores $\max_T P(T)$ and backpointers to reconstruct the best tree. The complexity remains $O(n^3 |G|)$.

4. The Earley Parser

While CYK requires CNF conversion, the Earley parser (Earley, 1970) handles arbitrary CFGs directly using a top-down strategy with bottom-up confirmation.

Earley Parser

Maintains a chart of Earley states, each of the form $[A \to \alpha \bullet \beta, i]$, meaning:

  • We are trying to build an $A$
  • We have matched $\alpha$ so far
  • We still need to match $\beta$
  • This rule application began at position $i$

Three operations build the chart: prediction, scanning, and completion.

The Three Operations

  • Prediction: If the dot is before a non-terminal $B$ in state $[A \to \alpha \bullet B \beta, i]$ at position $j$, add $[B \to \bullet \gamma, j]$ for every rule $B \to \gamma$
  • Scanning: If the dot is before terminal $a$ in $[A \to \alpha \bullet a \beta, i]$ at position $j$ and $w_{j+1} = a$, add $[A \to \alpha a \bullet \beta, i]$ to position $j+1$
  • Completion: If a state $[B \to \gamma \bullet, j]$ is complete at position $k$, find all states $[A \to \alpha \bullet B \beta, i]$ at position $j$ and add $[A \to \alpha B \bullet \beta, i]$ at position $k$

Complexity

Grammar TypeEarley Complexity
General CFG$O(n^3)$
Unambiguous CFG$O(n^2)$
Deterministic CFG (LR)$O(n)$

The Earley parser's ability to handle any CFG without transformation, combined with its adaptive complexity, makes it theoretically elegant, though in practice specialized parsers are often faster.

5. Dependency Parsing

Dependency parsing connects each word to its syntactic head (governor). The result is a tree where words are nodes and directed arcs represent grammatical relations.

Dependency Tree

A dependency tree for sentence $w_1, \ldots, w_n$ is a directed tree where:

  • Each word $w_i$ has exactly one head (parent), except the root
  • Each arc $(w_i, w_j)$ is labeled with a dependency relation (e.g., nsubj, dobj, amod)
  • The tree is connected and acyclic

Common Dependency Relations

RelationDescriptionExample
nsubjNominal subjectcat → sat
objDirect objectate → fish
amodAdjectival modifierbig → cat
detDeterminerthe → cat
advmodAdverbial modifierquickly → ran
caseCase marker / prepositionon → mat
oblOblique nominalsat → mat

Projectivity

A dependency tree is projective if no arcs cross when drawn above the sentence. Most English sentences are projective, but free word order languages frequently produce non-projective trees:

Non-Projective Dependencies

Czech: "Kontrolu závêrečnou provedl výbor"

(Final inspection carried-out committee)

The adjective "závêrečnou" modifies "kontrolu" but is separated by the verb, creating crossing arcs. Projective parsers cannot handle this; non-projective algorithms are needed.

6. Transition-Based Dependency Parsing

Transition-based parsers process the sentence left-to-right using a stack and a buffer, making a sequence of local decisions.

Arc-Standard System

Configuration: $(\sigma, \beta, A)$ where $\sigma$ is a stack, $\beta$ is a buffer (remaining words), and $A$ is the set of created arcs.

Three transitions:

  • Shift: Move the first word of $\beta$ onto $\sigma$
  • Left-Arc$(r)$: Create arc $\sigma_1 \xrightarrow{r} \sigma_2$ and remove $\sigma_2$ from stack
  • Right-Arc$(r)$: Create arc $\sigma_2 \xrightarrow{r} \sigma_1$ and remove $\sigma_1$ from stack

Parsing "The cat sat"

StepStackBufferActionArc
0[ROOT][The, cat, sat]Shift—
1[ROOT, The][cat, sat]Shift—
2[ROOT, The, cat][sat]Left-Arc(det)cat → The
3[ROOT, cat][sat]Shift—
4[ROOT, cat, sat][]Left-Arc(nsubj)sat → cat
5[ROOT, sat][]Right-Arc(root)ROOT → sat

Complexity: $O(n)$ — exactly $2n$ transitions for a sentence of length $n$ (each word is shifted once and reduced once). This linear complexity makes transition-based parsers extremely fast.

Oracle and Training

An oracle is a function that determines the correct transition at each step given the gold-standard tree. Training data is generated by running the oracle on treebank sentences. A classifier (originally SVM, now typically a neural network) learns to predict transitions from features of the current configuration.

Error Propagation

The greedy, left-to-right nature of transition-based parsing means that early errors cascade: a wrong attachment decision cannot be corrected later. Beam search (maintaining the top-$k$ configurations) partially addresses this at the cost of $O(kn)$ complexity. Global training methods (structured perceptron, globally normalized models) also help.

7. Graph-Based Dependency Parsing

Graph-based parsers take a global approach: they score all possible arcs and find the tree with the highest total score.

Maximum Spanning Tree Parsing

Given a sentence $w_1, \ldots, w_n$, construct a complete directed graph where the weight of arc $(i, j)$ is $\text{score}(i, j, r)$ — the score of $w_i$ being the head of $w_j$ with relation $r$. The best parse is the maximum spanning arborescence of this graph.

Eisner's Algorithm (Projective)

For projective trees, Eisner's algorithm (1996) uses dynamic programming with $O(n^3)$ complexity. It builds the tree bottom-up by combining half-spans:

  • An incomplete span $[i, j]$ represents a partial structure where $i$ is the head of $j$ (or vice versa) but $j$ has not yet collected all its dependents
  • A complete span $[i, j]$ represents a word that has collected all dependents on one side

Chu-Liu-Edmonds (Non-Projective)

For non-projective trees, the Chu-Liu-Edmonds algorithm finds the maximum spanning arborescence in $O(n^2)$ time:

  1. For each node, select the highest-scoring incoming arc
  2. If no cycle exists, we're done
  3. If a cycle exists, contract it into a single node, adjust arc weights, and recurse

In practice, modern neural parsers of both types achieve very similar accuracy, with transition-based parsers being faster and graph-based parsers being more robust on long sentences.

Evaluation Metrics

Constituency Parsing: Bracket F1

A bracket is a tuple $(i, j, A)$ representing a constituent of type $A$ spanning words $i$ to $j$.

$$\text{Precision} = \frac{|\text{predicted brackets} \cap \text{gold brackets}|}{|\text{predicted brackets}|}$$ $$\text{Recall} = \frac{|\text{predicted brackets} \cap \text{gold brackets}|}{|\text{gold brackets}|}$$ $$F_1 = \frac{2 \cdot P \cdot R}{P + R}$$

Dependency Parsing: Attachment Scores

  • UAS (Unlabeled Attachment Score): % of words attached to the correct head
  • LAS (Labeled Attachment Score): % of words attached to the correct head with the correct relation label
$$\text{UAS} = \frac{\text{correct heads}}{n}, \quad \text{LAS} = \frac{\text{correct heads with correct labels}}{n}$$

The Neural Revolution in Parsing

Modern parsers use neural networks to score arcs (graph-based) or predict transitions. The key innovation is contextual word representations: pre-trained language models like BERT provide embeddings that encode syntactic information, dramatically improving parsing accuracy. The biaffine attention parser (Dozat & Manning, 2017) uses separate MLPs for heads and dependents, then combines them with a biaffine scoring function, achieving near-human accuracy on English.

Parsing algorithms bridge the gap between formal grammar theory and practical language processing. The CYK and Earley algorithms provide exact solutions for constituency parsing, while transition-based and graph-based methods offer efficient dependency parsing. Modern neural approaches have pushed accuracy to remarkable levels, but the fundamental algorithmic ideas — dynamic programming, greedy search, and maximum spanning trees — remain at the core of every parser built today.
