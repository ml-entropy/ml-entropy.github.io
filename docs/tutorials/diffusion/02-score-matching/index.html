Score Matching & Langevin Dynamics

The score view of diffusion asks a sharper question than “what clean sample produced this noise?” It asks for the local direction in which log-density increases, which turns sampling into repeated motion toward higher-probability regions combined with carefully calibrated randomness.

This perspective is powerful because it removes the normalization constant from the problem. Gradients of log-densities do not care about unknown partition functions, which is exactly why score-based models can reason about complicated distributions without ever writing them in normalized form.

7.1 The Score Function

The score of a distribution is the gradient of its log-density:

$$s(x) = \nabla_x \log p(x)$$

It points toward regions of higher probability — like a compass pointing "uphill."

7.2 Score Matching Objective

Directly matching the score is intractable (we don't know $\nabla_x \log p(x)$). But denoising score matching is tractable:

$$\mathbb{E}_{q(x_t|x_0)}\left[\|s_\theta(x_t, t) - \nabla_{x_t} \log q(x_t|x_0)\|^2\right]$$

Since $\nabla_{x_t} \log q(x_t|x_0) = -\epsilon / \sqrt{1-\bar{\alpha}_t}$, this is equivalent to $\epsilon$-prediction!

7.3 Langevin Dynamics

Given the score function, we can sample using Langevin dynamics:

$$x_{t+1} = x_t + \eta \nabla_x \log p(x_t) + \sqrt{2\eta} \, z, \quad z \sim \mathcal{N}(0, I)$$

This converges to samples from $p(x)$ as $\eta \to 0$ and number of steps $\to \infty$.

Hyvärinen score matching and Fisher divergence

Classical score matching minimizes the squared gap between the true score and a model score field. After an integration-by-parts trick, the objective can be written without the unknown data score itself, which is why the method is appealing despite operating on derivatives of densities.

In diffusion practice, denoising score matching is even more useful. Rather than estimating the score on the raw data manifold — where low-density regions are treacherous — we estimate the score of a smoothed family of noisy distributions. That smoothing is what makes modern diffusion training robust.

Why multiple noise scales help

At very low noise levels, the data distribution can look like a thin, sharp manifold and score estimation becomes unstable away from the data. Injecting larger noise smooths the density, produces gentler score fields, and gives Langevin dynamics room to move globally before refining local detail.

Noise-conditional score networks

NCSN-style models train one score network across many noise scales. Conceptually, the high-noise regime teaches the model global geometry, while the low-noise regime teaches local detail; annealed Langevin dynamics then traverses that hierarchy from broad exploration to sharp refinement.

Diffusion models and score-based models are therefore not rival camps so much as different coordinate systems for the same underlying object. Predicting the forward noise and predicting the smoothed score are linked by a simple scale factor determined by the schedule.

$$s_\theta(x_t,t) = -\frac{\epsilon_\theta(x_t,t)}{\sqrt{1-\bar\alpha_t}}$$

Exact scores for a Gaussian mixture

Before training a neural score network, it is useful to work with a distribution whose score can be computed analytically. Gaussian mixtures are perfect for that purpose because their score is explicit but still rich enough to have multiple modes.

Python 

import torch
import math

def gaussian_mixture_score(x, means, sigma, logits=None):
    # x: [N, D], means: [K, D]
    if logits is None:
        logits = torch.zeros(len(means), device=x.device)
    weights = torch.softmax(logits, dim=0)

    diff = x[:, None, :] - means[None, :, :]
    log_comp = -0.5 * diff.pow(2).sum(dim=-1) / sigma**2
    log_comp = log_comp + torch.log(weights)[None, :]

    responsibilities = torch.softmax(log_comp, dim=1)
    component_scores = -(diff / sigma**2)
    score = (responsibilities[..., None] * component_scores).sum(dim=1)
    return score
Why this implementation matters

This implementation mirrors the analytic derivation: a mixture score is a responsibility-weighted average of component scores.

Denoising score matching loss

For Gaussian corruption, the target score of the noised distribution is available in closed form. That is what makes denoising score matching practical.

Python 

import torch.nn.functional as F

def dsm_loss(model, x0, sigma):
    noise = torch.randn_like(x0)
    xt = x0 + sigma * noise
    target_score = -(xt - x0) / (sigma ** 2)
    pred_score = model(xt, sigma)
    return F.mse_loss(pred_score, target_score)


def epsilon_loss_from_score(pred_score, noise, sigma):
    pred_eps = -sigma * pred_score
    return F.mse_loss(pred_eps, noise)
Why this implementation matters

The two losses differ only by a scale factor. This is the algebraic heart of the claim that diffusion and score matching are the same game in different notation.

Langevin dynamics sampler

Once a score is available, Langevin dynamics turns it into samples. The deterministic part follows the score uphill, while the random part keeps the chain from collapsing to a single mode.

Python 

@torch.no_grad()
def langevin_sample(score_fn, x_init, step_size=1e-2, n_steps=200):
    x = x_init.clone()
    trajectory = [x.clone()]
    for _ in range(n_steps):
        grad = score_fn(x)
        noise = torch.randn_like(x)
        x = x + step_size * grad + math.sqrt(2 * step_size) * noise
        trajectory.append(x.clone())
    return x, trajectory
Why this implementation matters

If the step size is too large, the chain becomes unstable. If it is too small, the chain mixes painfully slowly. That trade-off motivates annealed schedules and better solvers.

Annealed multi-scale sampling

Annealed Langevin dynamics uses a ladder of noise levels. High noise gives broad, global movement; low noise refines details.

Python 

@torch.no_grad()
def annealed_langevin(model, x_init, sigmas, steps_per_level=50, eps=1e-2):
    x = x_init.clone()
    for sigma in sigmas:
        alpha = eps * (sigma / sigmas[-1]) ** 2
        for _ in range(steps_per_level):
            grad = model(x, sigma)
            noise = torch.randn_like(x)
            x = x + alpha * grad + math.sqrt(2 * alpha) * noise
    return x

# Example sigma ladder
sigmas = torch.tensor([1.0, 0.5, 0.25, 0.1, 0.05])
Why this implementation matters

This annealed strategy is historically important because it makes visible how diffusion-like generation can emerge from score fields alone.

Easy Exercises

Start with direct computations and concept checks. These should feel like guided repetitions of the core formulas.

🟢 Exercise 1: Score of a standard normal ▼

Compute the score of $p(x)=\mathcal N(0,1)$.

The log-density is $-x^2/2$ up to a constant, so the derivative is $-x$. The score always points back toward the origin.

🟢 Exercise 2: Score at the mean ▼

What is the score of a Gaussian at its mean?

It is zero because the log-density is locally maximized there. A score field vanishes at stationary points of the density.

🟢 Exercise 3: Read the Langevin update ▼

In Langevin dynamics, which term follows the distribution and which term injects randomness?

The gradient term follows the score and therefore moves toward higher probability. The Gaussian term injects randomness so the chain explores rather than collapsing deterministically.

🟢 Exercise 4: Explain why normalization constants disappear ▼

Why does the score not depend on the partition function of a density?

Because differentiating $\log p(x)$ turns multiplicative constants into additive constants, and derivatives of constants vanish. That makes score methods attractive for unnormalized models.

🟢 Exercise 5: Compute a Gaussian corruption score ▼

If $x = x_0 + \sigma \epsilon$, what is $\nabla_x \log q_\sigma(x\mid x_0)$?

It is $-(x-x_0)/\sigma^2$. For Gaussian corruption the score is simply the negative residual divided by the variance.

🟢 Exercise 6: Direction of the score ▼

If a point lies to the right of a unimodal Gaussian mean, in which direction does the score point?

It points left, back toward the mean. More generally, the score points toward regions of higher density.

🟢 Exercise 7: Interpret NCSN qualitatively ▼

Why train on multiple noise scales instead of only one?

Different scales teach different geometric aspects of the distribution. High noise smooths the density for global guidance, while low noise sharpens local detail.

🟢 Exercise 8: Link epsilon to score ▼

If $\epsilon_\theta$ is known at timestep $t$, how do you recover the score estimate?

Use the scaling relation $s_\theta(x_t,t) = -\epsilon_\theta(x_t,t)/\sqrt{1-\bar\alpha_t}$. The schedule determines the conversion factor.

🟢 Exercise 9: Low-density regions ▼

Why are low-density regions hard for direct score estimation?

Because the score can change rapidly there and the model receives little data support. Errors in those regions can badly distort sampling trajectories.

🟢 Exercise 10: Annealing intuition ▼

What is the intuitive purpose of annealed Langevin dynamics?

It starts with a smooth landscape that is easy to navigate globally and then gradually sharpens the landscape so fine detail can be resolved near the end.

Medium Exercises

Now move from recall to derivation. These questions ask you to justify formulas, compare viewpoints, and explain why the method works.

🟡 Exercise 11: Responsibility-weighted mixture score ▼

Why does the score of a Gaussian mixture become a responsibility-weighted average of component scores?

The derivative of the mixture density is the sum of derivatives of its components. Dividing by the mixture density introduces posterior responsibilities, which act as adaptive weights on the component score vectors.

🟡 Exercise 12: Integration by parts idea ▼

What role does integration by parts play in classical score matching?

It transfers derivatives from the unknown data density onto the model score field under suitable boundary conditions. That is how the objective becomes computable without direct access to the true score.

🟡 Exercise 13: DSM as denoising ▼

Why can denoising score matching be viewed as a denoising problem?

For Gaussian corruption, the target score is proportional to the negative residual between noisy and clean samples. Estimating that residual is exactly what a denoiser does.

🟡 Exercise 14: Annealed step size scaling ▼

Why are step sizes in annealed Langevin often scaled with the noise level?

Because the geometry of the score field changes with smoothing. Large-noise levels permit larger global moves, while low-noise levels require smaller steps to avoid overshooting fine structure.

🟡 Exercise 15: From score to density geometry ▼

What geometric information is lost if you know only the density values but not the score field?

You lose the local directional guidance encoded by gradients. The score says how to move, not just how good the current location is.

🟡 Exercise 16: Failure mode near a manifold ▼

Why can a score network trained only at tiny noise levels fail catastrophically during sampling?

The data manifold is extremely thin, so the network has little incentive to learn meaningful directions away from it. Sampling then wanders into unsupported regions where the estimated score is unreliable.

🟡 Exercise 17: Diffusion equivalence ▼

Explain why a DDPM noise predictor can be reinterpreted as a score network.

The forward corruption kernel is Gaussian, so the score of the noised conditional is proportional to the negative forward noise. Predicting that noise therefore yields the score after dividing by the appropriate schedule-dependent scale.

🟡 Exercise 18: Langevin stationary distribution ▼

Why does Langevin dynamics sample from the target distribution in the small-step limit?

The associated Fokker-Planck equation has the target density as its stationary solution. As discretization error vanishes and mixing is sufficient, the chain approaches that equilibrium.

🟡 Exercise 19: Score scaling with variance ▼

How does the magnitude of the Gaussian score change when corruption variance increases?

The score magnitude shrinks because it is divided by $\sigma^2$. Smoother distributions therefore produce gentler, more globally informative score fields.

🟡 Exercise 20: Sampling versus optimization ▼

Why is Langevin dynamics not just gradient ascent on log-density?

Pure gradient ascent would converge to a mode and stop exploring. Langevin keeps adding noise so the chain spreads according to the whole target distribution instead of collapsing to a point estimate.

Hard Exercises

These problems push you toward research-level thinking: asymptotics, design choices, edge cases, and deeper theoretical interpretation.

🔴 Exercise 21: Fisher divergence interpretation ▼

Why is classical score matching minimizing a Fisher divergence rather than KL divergence?

The objective compares gradients of log-densities directly, which is exactly the Fisher divergence. This is a derivative-based discrepancy and behaves differently from KL because it emphasizes local geometry rather than only density ratios.

🔴 Exercise 22: Boundary conditions in score matching ▼

What technical assumption is hidden inside the integration-by-parts derivation?

One assumes boundary terms vanish at infinity so that the surface integral drops out. In practice this means densities and score fields must decay sufficiently fast.

🔴 Exercise 23: Noise-level design ▼

How would you choose a ladder of noise levels for an NCSN-style model on a highly multimodal dataset?

You want the largest scale high enough to connect distant modes through smoothing and the smallest scale low enough to recover sharp detail. A geometric or log-spaced ladder is often sensible because densities change multiplicatively across scales.

🔴 Exercise 24: Score error amplification ▼

Why can small score errors create large sampling artifacts after many Langevin steps?

Each update uses the model estimate as a direction field, so systematic bias compounds over time. Small rotational or scaling errors can therefore send trajectories into the wrong basin or prevent proper mixing.

🔴 Exercise 25: High-dimensional score estimation ▼

Why does score estimation become more difficult in high dimension even with Gaussian smoothing?

Volume grows rapidly away from the data concentration region, so the model must generalize score behavior over a much larger ambient space. The signal available from finite data becomes comparatively sparse.

🔴 Exercise 26: Annealed versus reverse diffusion ▼

In what sense is annealed Langevin a precursor to reverse diffusion sampling?

Both methods start from a heavily smoothed distribution and progressively refine samples toward the data manifold. Reverse diffusion makes the same idea explicit through a learned time-indexed denoising process with schedule-controlled noise.

🔴 Exercise 27: Consistency of score conversion ▼

Why must the score-to-noise conversion include the exact schedule coefficient rather than an arbitrary constant?

The scale of the score is determined by the variance of the forward corruption kernel. Using the wrong coefficient miscalibrates every reverse step and effectively asks the sampler to denoise at the wrong SNR.

🔴 Exercise 28: Mode hopping trade-off ▼

How does increasing Langevin noise help and hurt mode exploration?

More noise helps the chain cross low-density barriers between modes, but too much noise overwhelms the gradient signal and destroys refinement. Good samplers balance exploration and local correction.

🔴 Exercise 29: Smooth densities as curriculum ▼

Why can one view multi-scale score training as a curriculum learning strategy?

High-noise distributions are easier because their geometry is smoother and more global. Training then gradually exposes the model to harder, sharper densities as noise decreases.

🔴 Exercise 30: Research extension ▼

Propose a principled way to combine score matching with learned feature spaces rather than raw pixels.

Train the score network in a latent representation whose encoder preserves semantics while reducing dimensionality. The key requirement is that the latent corruption process and its score remain well defined so the sampler still corresponds to a valid generative path.
