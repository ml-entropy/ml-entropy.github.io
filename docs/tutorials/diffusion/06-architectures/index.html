Diffusion Architectures

The diffusion objective is deceptively simple: predict the noise. That simplicity is exactly why architecture matters so much. Almost all of the modeling power is hidden in the question “what kind of network can infer the right denoising direction from a partially corrupted sample?”

A good diffusion architecture must preserve spatial detail, reason globally, understand time, and incorporate conditioning signals. The dominant designs — U-Nets, latent diffusion systems, and diffusion transformers — are all different answers to that same set of constraints.

Why the U-Net became the default backbone

A diffusion U-Net combines an encoder that builds larger receptive fields with a decoder that reconstructs high-resolution detail. Skip connections make the architecture especially well suited to denoising because low-level structure that is still present in the noisy input can be copied forward rather than rediscovered from scratch.

This is a subtle but powerful inductive bias. Denoising is not pure abstraction; even at moderate noise levels, local edges, color statistics, and geometric hints survive. The U-Net is designed to preserve those hints while still using deep bottleneck layers to reason about global semantics. The encoder progressively downsamples the spatial resolution, building up abstract feature representations, while the decoder upsamples back to the original resolution. Skip connections bridge matching encoder-decoder levels, ensuring that the fine-grained spatial information captured early in the encoder is directly available to the decoder without being forced through the informational bottleneck.

Worked example: skip connection information flow

Consider a 4-level U-Net operating on 64×64 latent feature maps. The encoder downsamples at each level: 64×64 → 32×32 → 16×16 → 8×8. The bottleneck sees only 8×8 spatial features — a compression ratio of 64× compared to the input resolution.

Without skip connections, the decoder must reconstruct all fine-grained detail from this 8×8 compressed representation alone. Every sharp edge, every texture gradient, every subtle color transition at 64×64 resolution must somehow be inferred from 64 spatial positions. In practice, this produces blurry, over-smoothed outputs because the bottleneck simply cannot preserve enough spatial information to reconstruct high-frequency details faithfully.

With skip connections, the decoder at each level receives two inputs: the upsampled features from below and the original encoder features at the same resolution, concatenated along the channel axis. At the 64×64 level, the decoder has access to both the semantically rich (but spatially coarse) bottleneck information that has been progressively upsampled, and the spatially precise (but semantically shallow) encoder features that captured the original fine structure. The network can then learn to combine coarse semantic guidance with fine spatial detail — the bottleneck decides what should be there, and the skip features help decide exactly where it goes.

This is why U-Nets produce sharp images: they do not force all information through the bottleneck. The fine detail flows around the bottleneck via skip connections, while the bottleneck handles only the global, semantic reasoning. For denoising specifically, this is ideal — at moderate noise levels, most of the fine structure is still present in the input and can be passed directly through the skip connections, so the network only needs to use the bottleneck for the parts that require long-range reasoning.

Time conditioning with sinusoidal embeddings

The same noisy tensor can mean very different things at different timesteps. Early in the trajectory the network should make gentle local corrections, while late in the trajectory it should infer broad semantic structure from almost pure noise, so the timestep must be injected explicitly into the model.

Sinusoidal embeddings are popular because they map scalar timesteps into a rich continuous representation with multiple frequencies. Those embeddings are then projected and injected into residual blocks, often through FiLM-like scale-and-shift modulation. The key insight is that different frequencies in the embedding allow the network to distinguish fine-grained timestep differences — a model needs to know whether it is at step 200 vs. 210, not just "somewhere in the middle" — and the continuous nature of the representation means the network can interpolate smoothly between timestep behaviors even for values it has not seen during training.

FiLM conditioning: Adaptive Group Normalization
1
The standard Group Normalization operation normalizes features $h$ within groups of channels to zero mean and unit variance. In a diffusion residual block, we augment this with adaptive affine parameters that are functions of the timestep embedding $t_{\text{emb}}$: $$\text{AdaGN}(h, t) = \gamma(t) \cdot \text{GroupNorm}(h) + \beta(t)$$ Here $\gamma(t)$ and $\beta(t)$ are learned affine transforms of the timestep embedding — typically a linear layer applied to $t_{\text{emb}}$ that outputs per-channel scale and shift values.
2
This mechanism is called Feature-wise Linear Modulation (FiLM). It lets the network modulate feature statistics differently at each noise level. At early timesteps (low noise), the network should preserve and refine existing structure, so $\gamma$ values tend to be close to 1 and $\beta$ close to 0 — gentle corrections. At late timesteps (high noise), the network must hallucinate structure from almost-random input, so the affine parameters can be more aggressive, effectively rescaling which feature channels are active.
3
The practical benefit is that timestep conditioning penetrates every residual block, not just the input. Without FiLM, the network would need to learn timestep-dependent behavior purely from the initial concatenation or addition of the timestep embedding — a much weaker conditioning signal. With FiLM, every normalization layer in every block can adapt its behavior to the current noise level, giving the model fine-grained control over how it processes features across the full diffusion trajectory.
Why GroupNorm, not BatchNorm?

Diffusion model training typically uses small effective batch sizes. High-resolution images consume substantial GPU memory, and each batch element may be at a different timestep with different noise characteristics, so the "batch statistics" that BatchNorm relies on become extremely noisy and unreliable. GroupNorm sidesteps this problem entirely: it computes normalization statistics per sample, dividing channels into groups and normalizing within each group independently. This makes it completely invariant to batch size — whether you train with batch size 1 or batch size 256, the normalization statistics are identical for a given input.

This is a practical design constraint, not a theoretical preference. Early experiments with BatchNorm in diffusion models showed training instabilities and degraded sample quality, particularly at small batch sizes. GroupNorm resolved these issues with negligible computational overhead. Nearly all modern diffusion architectures — both U-Net and DiT variants — use GroupNorm (or the closely related LayerNorm) as their default normalization layer.

Self-attention and cross-attention

Convolutions are excellent at local computation, but diffusion also needs global coordination. Attention layers solve that problem by allowing any spatial location to interact with any other location, which helps maintain object coherence, layout, and long-range consistency. Without attention, a convolutional network can only propagate information across distant spatial positions through many successive layers — and even very deep networks can struggle to maintain globally coherent structure across a full image. Attention layers short-circuit this limitation, giving the network direct access to global context in a single operation.

Cross-attention extends this idea to conditioning. Image features generate queries, text or other conditioning features generate keys and values, and the resulting attention weights tell the model which condition tokens matter at each spatial position. This is the mechanism that makes text-to-image generation possible: the model can learn, for example, that spatial positions corresponding to a "red car" should attend strongly to the text tokens "red" and "car" while ignoring tokens about other parts of the prompt. The attention weights are learned end-to-end, so the model automatically discovers the right alignment between text semantics and spatial regions.

Self-attention in diffusion U-Nets
1
Given a spatial feature map $h \in \mathbb{R}^{n \times d}$ (where $n$ is the number of spatial positions and $d$ is the channel dimension), we compute queries, keys, and values with learned projections: $$Q = W_Q\, h, \quad K = W_K\, h, \quad V = W_V\, h$$ Because $Q$, $K$, and $V$ all come from the same feature map, this is self-attention — every spatial location attends to every other location in the same layer.
2
The attention weights are computed as a scaled dot-product: $$\text{Attn}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^T}{\sqrt{d_k}}\right) V$$ The $\sqrt{d_k}$ scaling prevents the dot products from growing large in magnitude, which would push softmax into saturated regions with near-zero gradients. Each row of the resulting attention matrix tells one spatial position how much to attend to every other position — giving every pixel a global receptive field in a single layer, something a convolutional kernel cannot achieve regardless of depth.
3
This global receptive field complements the local convolutions elsewhere in the U-Net. Convolutions capture texture, edges, and local patterns efficiently, while self-attention captures long-range coherence — ensuring, for example, that the left eye and right eye of a face remain symmetric even when they are far apart in pixel space. In practice, self-attention is applied only at lower-resolution feature maps (e.g., 16×16 or 32×32) to keep the $O(n^2 d)$ cost manageable.
4
For cross-attention, queries still come from the image features, but keys and values come from the conditioning signal $c$ (e.g., text embeddings from CLIP): $$Q = W_Q\, h, \quad K = W_K\, c, \quad V = W_V\, c$$ The attention formula is identical, but now the weights $\text{softmax}(QK^T / \sqrt{d_k})$ tell each spatial position which parts of the conditioning signal are most relevant to it. This is how text-to-image models "steer" the denoising process — each pixel location selectively reads from the words that describe the content it should produce.

Latent diffusion

Pixel-space diffusion is expensive because every denoising step operates on large images. Latent diffusion first compresses images with an autoencoder and then runs the diffusion process in the latent space, where spatial resolution is drastically reduced but semantic structure is preserved. The autoencoder is trained separately (often as a VAE or VQ-VAE) to learn a compact representation where nearby points in latent space correspond to visually similar images, which gives the diffusion model a well-structured space to operate in.

This is the architectural insight behind Stable Diffusion. By moving the expensive iterative process into a lower-dimensional space, the model becomes practical on commodity hardware without abandoning the expressive power of large denoisers. A typical compression ratio is 8× per spatial dimension — a 512×512 image becomes a 64×64 latent grid with 4 channels — which reduces the per-step memory and compute by roughly two orders of magnitude compared to pixel-space diffusion at the same resolution. The autoencoder and decoder are applied only once per sample (encode at training time, decode at generation time), so their cost is amortized across all denoising steps.

Worked example: why latent diffusion changes the budget so much

Imagine diffusing a 512x512 RGB image directly. Even before the network becomes deep, every denoising step must process hundreds of thousands of spatial positions, and that cost is paid again and again across the whole reverse trajectory. If an autoencoder compresses the image to a 64x64 latent grid, the iterative diffusion loop now runs on 64 times fewer spatial locations.

That is why latent diffusion was such a practical breakthrough. The encoder and decoder cost is paid only once per sample, while the expensive repeated denoising computation happens in a much smaller space. The autoencoder is therefore not decorative compression; it is what makes large modern diffusion systems economically feasible.

Common misconception: latent diffusion is not "just compression"

A weak autoencoder can permanently destroy fine structure before the denoiser ever sees it. Good latent diffusion depends on a latent space that is compact enough to save compute and faithful enough that the diffusion prior is not forced to spend capacity repairing codec artifacts.

Diffusion Transformers (DiT)

DiT replaces the convolutional U-Net with a transformer over image patches or latent patches. Instead of skip-connected multiscale feature pyramids, the model relies on token mixing, attention, and adaptive layer-normalization conditioned on time and labels. The architecture is strikingly simple compared to a U-Net: there is no encoder-decoder structure, no multi-scale processing, and no skip connections. Instead, there is a single sequence of identical transformer blocks that process a flat sequence of patch tokens — all the architectural complexity is delegated to the attention mechanism and the conditioning interface.

The attraction of DiT is scaling behavior. Transformers often benefit more predictably from increased model size and training compute, which is why diffusion research increasingly treats the backbone as a scalable sequence model rather than a handcrafted convolutional hierarchy. Empirically, the original DiT paper showed that simply increasing the number of transformer blocks and the hidden dimension produced consistent FID improvements on ImageNet, with the scaling curve showing no signs of saturation at the model sizes tested. This stands in contrast to U-Net scaling, where adding capacity requires careful decisions about where to add channels, which resolutions to add attention at, and how to balance encoder vs. decoder depth.

How a DiT actually processes an image

A DiT begins by chopping an image or latent into patches, projecting each patch to a token vector, adding positional structure, and then running many transformer blocks over the resulting sequence. Timestep and class information do not merely appear at the input; through adaptive normalization they modulate every block so the whole computation knows what noise regime and condition it is operating under.

That is the key conceptual shift from U-Nets. A U-Net bakes multiscale image structure into the backbone directly, while a DiT treats the denoiser as a general-purpose conditional sequence model whose tokens happen to come from visual patches. Self-attention handles global communication, and adaptive LayerNorm gates decide how strongly each block should act at the current timestep.

A minimal mental model of a DiT forward pass
1
Patchify the image or latent and project each patch into a token embedding.
2
Add positional information and construct a conditioning vector from time, class, or text features.
3
Use adaptive normalization to modulate attention and MLP sublayers inside every transformer block.
4
Project the updated tokens back to patches and unpatchify them into a denoising prediction.
Worked example: parameter count and compute comparison

A concrete comparison helps ground the U-Net vs. DiT trade-off. Stable Diffusion 1.5 uses a convolutional U-Net with approximately 860M parameters. The separately trained VAE encoder/decoder adds ~84M parameters, bringing the total to ~944M. In contrast, DiT-XL/2 (the largest variant in the original DiT paper) has approximately 675M parameters and also relies on a pretrained VAE — but the denoiser itself is smaller.

The compute characteristics differ more sharply than raw parameter counts suggest. In the U-Net, self-attention is applied at intermediate resolutions (typically 32×32 and 16×16), giving $n = 1024$ or $n = 256$ spatial tokens per attention layer. The attention cost is $O(n^2 d)$ per layer. But many of the U-Net's FLOPs come from convolutions, which scale linearly with spatial resolution and are extremely hardware-efficient on modern GPUs.

DiT operates entirely in token space. A 64×64 latent with patch size 2 produces $\frac{64}{2} \times \frac{64}{2} = 1024$ tokens. Every transformer block applies full self-attention over all 1024 tokens, so the attention cost is $O(1024^2 \cdot d)$ per block — and DiT-XL has 28 such blocks. By comparison, a pixel-space model at 256×256 with patch size 4 would produce 4096 tokens, making attention 16× more expensive per layer. This is precisely why latent diffusion is essential for DiT: operating on a compressed 64×64 latent rather than 256×256 pixels keeps the token count tractable.

The bottom line: the U-Net packs more parameters and leverages convolutional efficiency, while DiT uses fewer parameters but routes more of the computation through attention. At matched FLOP budgets, DiT tends to produce better FID scores on ImageNet-class benchmarks — but the U-Net's convolutional layers run faster on current hardware, so wall-clock time comparisons are more nuanced than FLOP comparisons alone.

Scaling laws and practical design

In practice, architecture choices interact with data scale, latent resolution, memory budget, and conditioning method. There is no single best design: U-Nets remain strong and efficient, latent diffusion makes large images affordable, and transformer backbones shine when scale and compute are abundant.

Scaling laws matter because backbone choice changes how effectively extra parameters, extra data, and extra training time translate into a better denoiser. U-Nets often dominate in the efficiency regime, where multiscale locality and skip connections buy a lot of quality per FLOP. DiTs become more attractive when the dataset, optimizer budget, and hardware scale are large enough that generic token mixing keeps improving smoothly.

The right way to think about diffusion architecture is therefore not “which block is fashionable?” but “which inductive biases best match my domain, my hardware budget, and my conditioning interface?” That question is what modern systems engineering around diffusion is really about.

What does this mean for practitioners?

If you are compute-limited, choose the backbone whose bias gives you the most efficiency per denoising step. If you are scale-limited in the opposite direction — enormous data, enormous clusters, and a frontier-quality target — then a transformer backbone may have more headroom even if it is not the most elegant small model.

The scaling hypothesis: why frontier models prefer transformers

DiT's central advantage over the convolutional U-Net is predictable scaling: doubling model size yields a smooth, predictable improvement in FID, and this relationship holds across multiple orders of magnitude. U-Nets, by contrast, do not scale as predictably because their inductive biases — skip connections, multi-scale feature pyramids, and handcrafted resolution schedules — create diminishing returns once the model is large enough that the bottleneck is data and optimization rather than architecture. At small and medium scales the U-Net's biases are a bargain: they deliver strong performance per FLOP. But at extreme scales the same biases become ceilings, and the generic token-mixing of a transformer keeps climbing.

This is why frontier systems (DALL·E 3, Stable Diffusion 3, Sora) increasingly use transformer-based backbones. The bet is not that transformers are inherently better denoisers, but that they are better scaling vehicles — and at frontier budgets, scaling predictability matters more than per-FLOP efficiency. For most practitioners who operate below frontier scale, the U-Net remains an excellent and often superior choice.

Diffusion Architectures

Diffusion architectures are really denoising architectures under tight constraints: they must preserve detail, reason globally, understand time, and absorb conditioning signals. U-Nets and DiTs are two dominant solutions to that design problem. The U-Net leverages multiscale locality and skip connections as powerful inductive biases, making it extremely efficient in the moderate-compute regime and still dominant in production systems like Stable Diffusion 1.x and 2.x. Diffusion Transformers trade those hand-crafted biases for the scaling predictability of general-purpose sequence models, and frontier labs increasingly adopt them when compute and data budgets are large enough to exploit smooth scaling laws. Ultimately, the choice between these backbones is an engineering trade-off between efficiency and scalability — and latent diffusion ensures that either backbone can operate at practical cost by moving the expensive iterative denoising into a compressed representation space.

Sinusoidal timestep embeddings

Timestep embeddings turn one scalar into a vector that can modulate many layers. The implementation mirrors transformer position embeddings but is used for denoising time instead of token position.

Python 

import math
import torch

def timestep_embedding(t, dim, max_period=10000):
    half = dim // 2
    freqs = torch.exp(-math.log(max_period) * torch.arange(half, device=t.device) / half)
    args = t.float()[:, None] * freqs[None, :]
    emb = torch.cat([torch.cos(args), torch.sin(args)], dim=-1)
    if dim % 2 == 1:
        emb = torch.cat([emb, torch.zeros_like(emb[:, :1])], dim=-1)
    return emb
Why this implementation matters

This embedding gives the network a smooth representation of time that generalizes across nearby timesteps.

A residual block with time conditioning

Most diffusion U-Nets inject timestep information inside residual blocks rather than only at the input. That keeps temporal context available throughout the network depth.

Python 

import torch.nn as nn

class ResBlock(nn.Module):
    def __init__(self, in_ch, out_ch, temb_dim):
        super().__init__()
        self.norm1 = nn.GroupNorm(32, in_ch)
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, padding=1)
        self.temb = nn.Linear(temb_dim, out_ch)
        self.norm2 = nn.GroupNorm(32, out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, padding=1)
        self.act = nn.SiLU()
        self.skip = nn.Conv2d(in_ch, out_ch, 1) if in_ch != out_ch else nn.Identity()

    def forward(self, x, temb):
        h = self.conv1(self.act(self.norm1(x)))
        h = h + self.temb(self.act(temb))[:, :, None, None]
        h = self.conv2(self.act(self.norm2(h)))
        return h + self.skip(x)
Why this implementation matters

The time projection enters additively here, but many variants also use scale-and-shift modulation of normalized activations.

A minimal diffusion U-Net skeleton

Real production U-Nets are deeper and more carefully tuned, but the high-level skeleton is simple: downsample, process a bottleneck, then upsample while concatenating skip features.

Python 

class TinyUNet(nn.Module):
    def __init__(self, base=64, temb_dim=256):
        super().__init__()
        self.in_conv = nn.Conv2d(4, base, 3, padding=1)
        self.down1 = ResBlock(base, base, temb_dim)
        self.down2 = ResBlock(base, base * 2, temb_dim)
        self.mid = ResBlock(base * 2, base * 2, temb_dim)
        self.up1 = ResBlock(base * 4, base, temb_dim)
        self.out_norm = nn.GroupNorm(32, base)
        self.out_conv = nn.Conv2d(base, 4, 3, padding=1)
        self.pool = nn.AvgPool2d(2)
        self.up = nn.Upsample(scale_factor=2, mode="nearest")

    def forward(self, x, temb):
        h0 = self.in_conv(x)
        h1 = self.down1(h0, temb)
        h2 = self.down2(self.pool(h1), temb)
        hmid = self.mid(h2, temb)
        h = self.up(hmid)
        h = torch.cat([h, h1], dim=1)
        h = self.up1(h, temb)
        return self.out_conv(torch.nn.functional.silu(self.out_norm(h)))
Why this implementation matters

Even this toy model shows the core idea: coarse reasoning in the middle, high-resolution detail passed through skip connections.

Cross-attention for text conditioning

Cross-attention fuses image tokens with condition tokens. In text-to-image systems, the condition tokens come from a language model.

Python 

class CrossAttention(nn.Module):
    def __init__(self, dim, context_dim, heads=8):
        super().__init__()
        self.heads = heads
        self.scale = (dim // heads) ** -0.5
        self.to_q = nn.Linear(dim, dim, bias=False)
        self.to_k = nn.Linear(context_dim, dim, bias=False)
        self.to_v = nn.Linear(context_dim, dim, bias=False)
        self.to_out = nn.Linear(dim, dim)

    def forward(self, x, context):
        q = self.to_q(x)
        k = self.to_k(context)
        v = self.to_v(context)
        q = q.view(x.size(0), x.size(1), self.heads, -1).transpose(1, 2)
        k = k.view(context.size(0), context.size(1), self.heads, -1).transpose(1, 2)
        v = v.view(context.size(0), context.size(1), self.heads, -1).transpose(1, 2)
        attn = (q @ k.transpose(-2, -1)) * self.scale
        attn = attn.softmax(dim=-1)
        out = attn @ v
        out = out.transpose(1, 2).reshape(x.size(0), x.size(1), -1)
        return self.to_out(out)
Why this implementation matters

Cross-attention is the mechanism that makes prompts spatially meaningful rather than just globally conditioning the whole image.

A tiny DiT-style block

Diffusion transformers often use adaptive LayerNorm zero conditioning, which modulates both the attention block and the MLP using time or class embeddings.

Python 

class DiTBlock(nn.Module):
    def __init__(self, dim, heads, mlp_ratio=4.0):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim, elementwise_affine=False)
        self.attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim, elementwise_affine=False)
        hidden = int(dim * mlp_ratio)
        self.mlp = nn.Sequential(nn.Linear(dim, hidden), nn.GELU(), nn.Linear(hidden, dim))
        self.modulation = nn.Linear(dim, 6 * dim)

    def forward(self, x, cond):
        shift1, scale1, gate1, shift2, scale2, gate2 = self.modulation(cond).chunk(6, dim=-1)
        h = self.norm1(x) * (1 + scale1[:, None, :]) + shift1[:, None, :]
        attn_out, _ = self.attn(h, h, h)
        x = x + gate1[:, None, :] * attn_out
        h = self.norm2(x) * (1 + scale2[:, None, :]) + shift2[:, None, :]
        x = x + gate2[:, None, :] * self.mlp(h)
        return x
Why this implementation matters

The conditioning vector modulates normalization statistics instead of only being concatenated at the input. That is one of the key DiT design choices.

🟢 Exercise 1: Why skip connections help ▼

Why are skip connections especially natural in a denoising network?

Because noisy inputs still contain useful local detail that should be preserved and refined rather than recomputed from scratch. Skip connections carry that detail directly to later layers.

🟢 Exercise 2: Time embedding purpose ▼

Why does a diffusion model need an explicit timestep embedding?

The denoising task changes with noise level, so the network must know whether it is making a coarse semantic guess or a small local correction. The timestep embedding provides that context.

🟢 Exercise 3: Encoder versus decoder ▼

What is the main job of the encoder side of a U-Net?

It builds larger receptive fields and more abstract features by progressively reducing spatial resolution. This helps the model reason about global structure.

🟢 Exercise 4: Decoder role ▼

What is the main job of the decoder side of a U-Net?

It reconstructs high-resolution features and combines coarse semantic understanding with preserved detail from skip connections. This is the refinement half of the architecture.

🟢 Exercise 5: Self-attention intuition ▼

Why add self-attention to a diffusion U-Net?

Attention lets distant spatial locations communicate directly, which helps maintain global coherence and object-level structure across the image.

🟢 Exercise 6: Cross-attention intuition ▼

What extra ability does cross-attention add beyond self-attention?

It lets image features interact with an external conditioning signal such as text. The denoiser can therefore decide which prompt tokens matter at which spatial positions.

🟢 Exercise 7: Latent diffusion goal ▼

Why diffuse in latent space instead of pixel space?

Latent space is much smaller, so each denoising step is cheaper while still representing the important semantics of the image. This makes large-scale diffusion practical.

🟢 Exercise 8: Patch tokens in DiT ▼

What does a DiT operate on instead of convolutional feature maps?

It operates on patch tokens or latent tokens treated as a sequence. That allows it to use a transformer backbone rather than a U-Net pyramid.

🟢 Exercise 9: GroupNorm usage ▼

Why is GroupNorm common in diffusion U-Nets?

It is stable across a wide range of batch sizes and works well in image models where batch size may be limited by memory. That makes it more convenient than BatchNorm in many diffusion setups.

🟢 Exercise 10: FiLM-style conditioning ▼

What does FiLM-like conditioning do to activations?

It scales and shifts them using a conditioning vector such as the timestep embedding. This gives the condition a direct influence on intermediate features throughout the network.

Medium Exercises

Now move from recall to derivation. These questions ask you to justify formulas, compare viewpoints, and explain why the method works.

🟡 Exercise 11: Sinusoidal embedding structure ▼

Why do sinusoidal embeddings use multiple frequencies?

The different frequencies let the network represent both coarse and fine distinctions in timestep. Nearby times remain related while distant times can still be separated cleanly.

🟡 Exercise 12: Residual learning benefit ▼

Why are residual blocks preferable to plain stacks of convolutions in deep diffusion networks?

Residual pathways stabilize optimization, preserve information flow, and make it easier for the model to learn corrections rather than full transformations. That is especially helpful in very deep denoisers.

🟡 Exercise 13: Attention placement ▼

Why do many U-Nets place attention only at select resolutions instead of everywhere?

Attention is expensive at high spatial resolutions. Using it at medium or low resolutions captures global structure while keeping memory and compute manageable.

🟡 Exercise 14: Latent autoencoder requirement ▼

What property must the autoencoder have for latent diffusion to work well?

It must compress aggressively enough to save compute but preserve semantics and reconstruct detail faithfully enough that the diffusion prior is not forced to learn around systematic codec artifacts.

🟡 Exercise 15: Cross-attention query choice ▼

Why do image features produce the queries in text-to-image cross-attention?

The image side is asking which pieces of text matter for its current spatial features. Using image queries therefore lets different regions retrieve different conditioning information from the shared text context.

🟡 Exercise 16: DiT scaling appeal ▼

Why are transformers attractive as diffusion backbones at large scale?

They often scale predictably with more parameters, more data, and more compute. This makes them appealing for frontier models where massive training budgets are available.

🟡 Exercise 17: Skip-connection trade-off ▼

What can go wrong if skip connections dominate too strongly?

The decoder may rely excessively on shallow detail and underuse the bottleneck’s global reasoning. That can hurt semantic coherence, especially at high noise levels where local detail is unreliable.

🟡 Exercise 18: Condition injection location ▼

Why is injecting conditioning only at the input usually insufficient?

The denoising computation is deep and changes with layer depth, so conditioning must remain available throughout the network. Otherwise late layers may forget or underuse the prompt or timestep information.

🟡 Exercise 19: Memory budget design ▼

If GPU memory is tight, what architectural choices become attractive?

Latent diffusion, narrower channels, attention at fewer resolutions, gradient checkpointing, and smaller patch/token sizes are all common ways to retain quality while fitting hardware limits.

🟡 Exercise 20: Architecture-task match ▼

Why might a U-Net remain preferable to a transformer for some domains?

When local spatial bias, efficiency, and limited compute matter more than massive scale, the U-Net’s inductive bias can be more effective and cheaper than a transformer sequence model.

Hard Exercises

These problems push you toward research-level thinking: asymptotics, design choices, edge cases, and deeper theoretical interpretation.

🔴 Exercise 21: Time-conditioning design ▼

Why is the timestep embedding not merely a nuisance input but a core architectural axis?

The denoising problem changes qualitatively across time, so the network effectively performs a whole family of related tasks. Architecture must therefore support condition-dependent computation at every depth rather than treating time as a superficial label.

🔴 Exercise 22: Latent dimension trade-off ▼

How does the choice of latent dimensionality in latent diffusion affect sample quality and compute?

Smaller latents reduce cost but force the autoencoder to discard or distort more detail, while larger latents preserve fidelity at higher iterative cost. The best point depends on reconstruction quality, model size, and target resolution.

🔴 Exercise 23: DiT versus U-Net inductive bias ▼

Why can a DiT scale better while still losing to a U-Net in some practical settings?

Transformers benefit strongly from scale, but they lack the built-in multiscale convolutional bias that U-Nets exploit efficiently. At moderate budgets or on strongly local image tasks, the handcrafted bias can still win.

🔴 Exercise 24: adaLN-Zero motivation ▼

Why is zero-initialized adaptive normalization attractive in DiT blocks?

It starts the conditioned residual branch near zero, which stabilizes optimization at initialization and lets conditioning effects grow gradually rather than disrupting early training dynamics.

🔴 Exercise 25: Cross-attention bottleneck ▼

What makes cross-attention a computational bottleneck in large text-to-image systems?

Every image token may attend to every condition token, so cost scales with both spatial token count and prompt length. This becomes expensive at high resolutions or with rich multimodal context.

🔴 Exercise 26: Autoencoder failure mode ▼

Why can a weak latent autoencoder bottleneck limit the final diffusion model no matter how strong the denoiser is?

If the autoencoder fails to preserve important image information, the diffusion prior can only model what remains in latent space. The denoiser cannot recover details that the encoder already destroyed irreversibly.

🔴 Exercise 27: Architectural scaling law ▼

Why do scaling laws matter for architecture selection?

They tell you whether more compute is likely to pay off for a given backbone. An architecture with better scaling may be a worse small model but a much better frontier model.

🔴 Exercise 28: Patch size in DiT ▼

How does patch size affect a diffusion transformer?

Smaller patches preserve finer spatial structure but create longer token sequences and higher compute. Larger patches are cheaper but risk losing local detail and making spatial reasoning coarser.

🔴 Exercise 29: Research extension ▼

Propose an architecture for diffusion on non-image data such as point clouds or molecules.

The backbone should respect the domain’s symmetry and structure, such as permutation invariance for sets or graph message passing for molecules. The core diffusion logic remains, but the denoiser’s inductive bias must match the data geometry.

🔴 Exercise 30: System-design question ▼

If you had to ship a practical text-to-image model under a tight inference budget, what architectural stack would you choose?

A sensible stack is latent diffusion with a strong autoencoder, a compact U-Net or medium-size DiT backbone, selective attention, and classifier-free guidance. This combination balances speed, controllability, and quality more effectively than a pure pixel-space model.
