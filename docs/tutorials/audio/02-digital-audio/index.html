Analog vs. Digital

An analog audio signal is a continuous function of time — the voltage from a microphone varies smoothly and can take any value at any instant. A digital audio signal is a sequence of discrete numbers, each representing the signal's amplitude at a specific moment.

Why digitize? Digital audio can be copied perfectly (no generational loss), stored compactly, processed with algorithms, and transmitted without degradation. The entire modern audio industry — streaming, recording, synthesis — rests on this conversion.

The Two Steps of Digitization

Sampling — discretize time: measure the signal at regular intervals.
Quantization — discretize amplitude: round each measurement to the nearest allowed level.

Sampling

Sampling captures the signal's value at evenly spaced instants. If we sample $f_s$ times per second (the sample rate), the time between consecutive samples is:

$$T_s = \frac{1}{f_s}$$

The $n$-th sample is: $x[n] = x_a(nT_s)$ where $x_a(t)$ is the original analog signal.

Quantization

Once sampled, each value must be stored as a finite number. With $n$ bits, we have $L = 2^n$ quantization levels. The continuous amplitude range is divided into $L$ equal steps:

$$\Delta = \frac{V_{\max} - V_{\min}}{2^n - 1}$$

The quantization error for any sample is at most $\pm\Delta/2$. This error acts as quantization noise.

The bit depth determines how many amplitude levels are available. More bits = more levels = finer resolution = less quantization noise:

The SNR Formula

For an ideal $n$-bit quantizer with a full-scale sinusoidal input, the signal-to-quantization-noise ratio is:

$$\text{SNR} = 6.02n + 1.76 \text{ dB}$$

Each additional bit adds ~6 dB of dynamic range. 16 bits: $6.02 \times 16 + 1.76 = 98.1$ dB.

Derivation sketch: The quantization error $e[n]$ is uniformly distributed over $[-\Delta/2, +\Delta/2]$. Its variance (noise power) is:

$$\sigma_q^2 = \frac{\Delta^2}{12}$$

For a full-scale sinusoid with amplitude $A = 2^{n-1}\Delta$, the signal power is $A^2/2$. The SNR ratio becomes:

$$\text{SNR} = \frac{A^2/2}{\Delta^2/12} = \frac{(2^{n-1}\Delta)^2/2}{\Delta^2/12} = \frac{3}{2} \cdot 2^{2n}$$

In decibels: $10\log_{10}\!\left(\frac{3}{2} \cdot 2^{2n}\right) = 10\log_{10}(1.5) + 20n\log_{10}(2) = 1.76 + 6.02n$ dB.

PCM — Pulse Code Modulation

PCM is the standard way to represent digital audio: each sample is a binary number. In 16-bit signed PCM, values range from $-32{,}768$ to $+32{,}767$. The WAV file format stores raw PCM data with a header specifying sample rate, bit depth, and channel count.

File size calculation:

$$\text{Size (bytes)} = f_s \times n_{\text{bits}}/8 \times n_{\text{channels}} \times t_{\text{seconds}}$$

Example: 1 minute of CD-quality audio (44,100 Hz, 16-bit, stereo):

$44100 \times 2 \times 2 \times 60 = 10{,}584{,}000$ bytes $\approx 10.1$ MB per minute.

Dithering

Here's a paradox: adding noise can improve audio quality. When reducing bit depth (e.g., 24-bit master → 16-bit CD), the quantization error becomes correlated with the signal, producing audible distortion artifacts. Adding a tiny amount of random noise (dither) before quantization breaks this correlation, converting the distortion into a quiet, benign hiss.

TPDF Dither

Triangular Probability Density Function dither (sum of two uniform random numbers) eliminates both the mean and variance of quantization error's correlation with the signal. It adds noise with amplitude of 2 LSBs (least significant bits), but this noise is completely uncorrelated with the signal — a good tradeoff.

Medium (11–20)

11Quantization Noise PowerMedium

Derive that the quantization noise power for uniform quantization is $\sigma_q^2 = \Delta^2/12$.

Quantization error $e$ is uniformly distributed on $[-\Delta/2, \Delta/2]$. Mean: $\mu = 0$. Variance:

$$\sigma_q^2 = \frac{1}{\Delta}\int_{-\Delta/2}^{\Delta/2} e^2\, de = \frac{1}{\Delta}\left[\frac{e^3}{3}\right]_{-\Delta/2}^{\Delta/2} = \frac{1}{\Delta} \cdot \frac{2(\Delta/2)^3}{3} = \frac{\Delta^2}{12}$$

12Storage ComparisonMedium

A podcast is 60 minutes long. Compare uncompressed file sizes at: (a) 16 kHz/16-bit/mono (speech), (b) 44.1 kHz/16-bit/stereo (music), (c) 96 kHz/24-bit/stereo (hi-res).

(a) $16000 \times 2 \times 1 \times 3600 = 115.2$ MB

(b) $44100 \times 2 \times 2 \times 3600 = 635$ MB

(c) $96000 \times 3 \times 2 \times 3600 = 2074$ MB $\approx 2.0$ GB

13Bit Depth for Target SNRMedium

You need at least 110 dB of SNR. What is the minimum bit depth required?

$110 = 6.02n + 1.76 \Rightarrow n = (110 - 1.76)/6.02 = 108.24/6.02 = 17.98$

Round up: 18 bits minimum.

14Dither AmplitudeMedium

Explain why TPDF dither has a peak amplitude of 2 LSB but an RMS amplitude of $\text{LSB}/\sqrt{6}$.

TPDF is the sum of two independent uniform random variables, each on $[-\text{LSB}/2, +\text{LSB}/2]$. The sum ranges from $-\text{LSB}$ to $+\text{LSB}$ (peak = 2 LSB peak-to-peak).

Variance of each uniform: $\text{LSB}^2/12$. Variance of sum: $2 \times \text{LSB}^2/12 = \text{LSB}^2/6$. RMS = $\text{LSB}/\sqrt{6} \approx 0.408 \times \text{LSB}$.

15WAV File HeaderMedium

A WAV file's data chunk is 7,056,000 bytes. If it's 16-bit stereo at 44,100 Hz, what is the audio duration?

Bytes per second: $44100 \times 2 \times 2 = 176{,}400$ B/s.

Duration: $7{,}056{,}000 / 176{,}400 = 40.0$ seconds.

16Quantization of a Specific SampleMedium

A 4-bit ADC maps input range [0, 3.0V] to levels 0–15. What binary code does it output for an input of 1.7V? What is the quantization error?

$\Delta = 3.0/15 = 0.2$ V. Level = $\lfloor 1.7/0.2 + 0.5 \rfloor = \lfloor 9.0 \rfloor = 9$ (rounding). Binary: 1001.

Reconstructed: $9 \times 0.2 = 1.8$ V. Error: $1.7 - 1.8 = -0.1$ V.

(With truncation instead of rounding: level = 8 (1000), error = +0.1 V.)

17Effective Number of BitsMedium

A real-world 16-bit ADC has measured SNR of 89 dB (due to thermal noise, clock jitter, etc.). What is its effective number of bits (ENOB)?

$\text{ENOB} = (\text{SNR} - 1.76)/6.02 = (89 - 1.76)/6.02 = 87.24/6.02 \approx 14.5$ bits.

Despite being a "16-bit" ADC, it effectively provides only 14.5 bits of useful resolution.

18Oversampling BenefitMedium

Oversampling by factor $K$ spreads quantization noise over a wider bandwidth, effectively adding $10\log_{10}(K)/6.02$ bits of resolution. How much oversampling is needed to gain 1 extra effective bit?

1 bit = 6.02 dB. Need $10\log_{10}(K) = 6.02$, so $K = 10^{0.602} \approx 4$.

Oversampling by 4× gains approximately 1 extra bit of resolution.

19Sample Rate ConversionMedium

You have audio at 44,100 Hz and need to convert it to 48,000 Hz. What is the simplest rational resampling ratio? Describe the process conceptually.

Ratio: $48000/44100 = 480/441 = 160/147$.

Process: (1) Upsample by 160 (insert 159 zeros between each sample), (2) lowpass filter to remove imaging, (3) downsample by 147 (keep every 147th sample). In practice, a polyphase filter implementation avoids computing samples that will be discarded.

20Floating Point AudioMedium

32-bit IEEE 754 floating point has 23 mantissa bits. Near a signal value of 1.0, what is the effective bit depth? Near 0.001?

Near 1.0: the mantissa provides 23 bits of precision, so the effective resolution is ~23 bits (SNR $\approx 6.02 \times 23 + 1.76 \approx 140$ dB).

Near 0.001: the exponent scales down, but relative precision stays 23 bits. The absolute step size is much smaller: $\sim 2^{-23} \times 2^{-10} \approx 10^{-10}$. This is the key advantage of floating point — constant relative precision regardless of signal level.

Hard (21–30)

21Prove the SNR FormulaHard

Rigorously derive $\text{SNR} = 6.02n + 1.76$ dB for an $n$-bit quantizer with full-scale sinusoidal input, starting from first principles.

Full-scale sinusoid: $x(t) = A\sin(\omega t)$ where $A = (2^n - 1)\Delta/2 \approx 2^{n-1}\Delta$.

Signal power: $P_s = A^2/2 = 2^{2n-2}\Delta^2/2 = 2^{2n-3}\Delta^2$.

Quantization noise power: $P_q = \Delta^2/12$.

$$\text{SNR} = \frac{P_s}{P_q} = \frac{2^{2n-3}\Delta^2}{\Delta^2/12} = 12 \cdot 2^{2n-3} = \frac{3}{2} \cdot 2^{2n}$$

$$\text{SNR}_{\text{dB}} = 10\log_{10}\!\left(\frac{3}{2}\right) + 10 \cdot 2n\log_{10}(2) = 1.761 + 6.021n$$

22Noise ShapingHard

In first-order noise shaping, the quantization error is fed back and subtracted from the next sample's input. Show that this shapes the noise spectrum by the filter $H(z) = 1 - z^{-1}$, which is a highpass filter. Why is this beneficial for audio?

Let $e[n]$ be the quantization error at step $n$. Without noise shaping, output = input + $e[n]$. With first-order feedback, the effective error becomes $e[n] - e[n-1]$.

In z-domain: $E_{\text{shaped}}(z) = E(z)(1 - z^{-1}) = E(z)H(z)$.

$|H(e^{j\omega})| = |1 - e^{-j\omega}| = 2|\sin(\omega/2)|$

This is zero at DC ($\omega = 0$) and maximum at Nyquist ($\omega = \pi$). Noise is pushed to high frequencies where (a) human hearing is less sensitive and (b) it can be filtered out. Combined with oversampling, this allows very high effective resolution in the audible band from low-bit quantizers — the principle behind sigma-delta ($\Sigma\Delta$) ADCs used in modern audio equipment.

23Jitter AnalysisHard

Clock jitter causes the actual sampling instants to deviate from ideal. If jitter is Gaussian with standard deviation $\sigma_t$ and the signal is $x(t) = A\sin(2\pi f t)$, show that the resulting SNR degradation is approximately $-20\log_{10}(2\pi f \sigma_t)$.

The error from sampling at $t + \delta t$ instead of $t$: $\epsilon \approx x'(t)\delta t = 2\pi f A\cos(2\pi ft)\delta t$.

Error power: $\langle\epsilon^2\rangle = (2\pi f A)^2 \langle\cos^2\rangle \langle\delta t^2\rangle = (2\pi f A)^2 \cdot \frac{1}{2} \cdot \sigma_t^2$.

Signal power: $A^2/2$. SNR:

$$\text{SNR}_{\text{jitter}} = \frac{A^2/2}{2\pi^2 f^2 A^2 \sigma_t^2} = \frac{1}{4\pi^2 f^2 \sigma_t^2}$$

$$\text{SNR}_{\text{dB}} = -20\log_{10}(2\pi f \sigma_t)$$

This shows jitter's impact is worse at higher frequencies. For a 20 kHz signal with 100 ps jitter: $\text{SNR} = -20\log_{10}(2\pi \times 20000 \times 10^{-10}) \approx 122$ dB — acceptable for 20-bit audio but not for higher resolution.

24Sigma-Delta ADC ResolutionHard

A 1-bit sigma-delta ADC oversamples at 6.144 MHz for a signal bandwidth of 24 kHz (256× oversampling). With Nth-order noise shaping, the effective SNR improvement is approximately $(2N+1) \times 10\log_{10}(\text{OSR}) - 10\log_{10}(\pi^{2N}/(2N+1))$ dB. For a 3rd-order modulator, estimate the ENOB.

OSR = 256. $10\log_{10}(256) = 24.1$ dB. For $N=3$:

Improvement $\approx 7 \times 24.1 - 10\log_{10}(\pi^6/7) = 168.7 - 10\log_{10}(133.1) = 168.7 - 21.2 = 147.5$ dB.

1-bit baseline SNR $= 6.02 \times 1 + 1.76 = 7.78$ dB.

Total SNR $\approx 7.78 + 147.5 = 155.3$ dB.

ENOB $= (155.3 - 1.76)/6.02 \approx 25.5$ bits. Extraordinary resolution from a simple 1-bit quantizer!

25Dither and Truncation DistortionHard

A 24-bit recording is truncated to 16 bits without dither. Show that for a −90 dBFS sinusoid (near the noise floor), the THD (Total Harmonic Distortion) can exceed −50 dB. Why does dither fix this?

At −90 dBFS in 16-bit, the signal amplitude is about $2^{16} \times 10^{-90/20} = 65536 \times 3.16 \times 10^{-5} \approx 2$ LSBs peak.

With only ~2 quantization levels, the output is essentially a square wave switching between two adjacent levels. A square wave's 3rd harmonic is only −9.5 dB below the fundamental; including all odd harmonics, THD $\approx$ −4 dB relative to the fundamental. Even if we're generous and the signal spans ~4 levels, THD is still very poor (−30 to −50 dB).

Dither (random noise of ~1 LSB) decorrelates the quantization error from the signal. The error becomes white noise spread uniformly across all frequencies, independent of signal content. While the noise floor rises slightly, the tonal distortion products vanish — perceived quality improves dramatically.

26ADC Aperture TimeHard

An ADC's aperture time is the time window during which it captures the input. If the aperture time is $\tau = 50$ ns and the input signal changes by more than 1 LSB during $\tau$, you get aperture error. For a 16-bit ADC with ±1V range sampling a full-scale sinusoid, what is the maximum frequency that avoids aperture error?

1 LSB = $2/(2^{16}-1) \approx 30.5$ μV. Maximum slew rate of a sinusoid: $dv/dt|_{\max} = 2\pi f A = 2\pi f \times 1$.

Require: $(dv/dt)_{\max} \times \tau \leq 1$ LSB:

$2\pi f \times 50 \times 10^{-9} \leq 30.5 \times 10^{-6}$

$f \leq 30.5 \times 10^{-6}/(2\pi \times 50 \times 10^{-9}) = 97.1$ Hz.

This is extremely low! This is why sample-and-hold circuits are essential — they "freeze" the voltage so the ADC has time to digitize it.

27Optimal Bit AllocationHard

You have a fixed budget of 1,411,200 bits per second (CD bit rate). You can trade sample rate for bit depth. Compare the quality tradeoffs of: (a) 44,100 Hz × 16 bit × 2 ch, (b) 88,200 Hz × 8 bit × 2 ch, (c) 22,050 Hz × 32 bit × 2 ch.

All three use the same bit rate: $f_s \times n \times 2 = 1{,}411{,}200$.

(a) Standard CD: 22.05 kHz bandwidth, 96 dB SNR — well-matched to human hearing.

(b) 44.1 kHz bandwidth (ultrasonic, mostly wasted), only 48 dB SNR — terrible quality, audible quantization noise.

(c) 11.025 kHz bandwidth (cuts off above ~11 kHz, muffled), but 192 dB SNR (absurd overkill).

Option (a) is optimal because it balances bandwidth (just above the 20 kHz hearing limit) with sufficient SNR (96 dB covers the ~90 dB dynamic range of human hearing). This is essentially why the CD standard chose these exact parameters.

28Companding: μ-LawHard

The μ-law companding function is $F(x) = \text{sgn}(x)\frac{\ln(1 + \mu|x|)}{\ln(1 + \mu)}$ where $\mu = 255$ for telephone. Show that this gives approximately uniform SNR across all signal levels, unlike linear PCM where quiet signals have worse SNR.

For linear PCM, quantization step $\Delta$ is constant. SNR $= 20\log_{10}(|x|/\Delta)$ — drops as $|x|$ decreases.

For companding, the effective step size after expansion: $\Delta_{\text{eff}} \propto dF^{-1}/dy \propto (dF/dx)^{-1}$.

$dF/dx = \frac{\mu}{\ln(1+\mu)(1+\mu|x|)}$. So $\Delta_{\text{eff}} \propto (1 + \mu|x|)/\mu$.

For $\mu|x| \gg 1$: $\Delta_{\text{eff}} \propto |x|$. Therefore $\text{SNR} \propto |x|/\Delta_{\text{eff}} \propto |x|/|x| = \text{constant}$!

The logarithmic compression makes quantization step proportional to signal level, giving roughly constant relative precision (like floating point) — essential for speech where signal level varies enormously.

29Multi-Bit vs. 1-Bit DACHard

Explain why many high-end DACs use 1-bit (DSD/sigma-delta) architecture instead of multi-bit R-2R ladders, despite needing extreme oversampling. What is the fundamental advantage regarding linearity?

A 1-bit DAC has only two output levels: +V and −V. Because there are only two levels, it is inherently perfectly linear — two points always define a straight line, so there can be no differential nonlinearity (DNL) or integral nonlinearity (INL) errors.

Multi-bit DACs require that the step size between every adjacent level be exactly equal. For a 16-bit R-2R DAC, the MSB resistor must match the LSB resistor to within 1 part in 65,536 (<0.0015%). This is extremely difficult to manufacture.

The tradeoff: 1-bit DACs need massive oversampling (64×–256×) and high-order noise shaping to achieve equivalent resolution, requiring sophisticated digital filtering. But since digital filters are implemented in silicon (cheap, perfect, reproducible), this is often preferred over the analog precision required for multi-bit DACs.

30Information-Theoretic CapacityHard

Shannon's channel capacity theorem states $C = B\log_2(1 + \text{SNR})$ bits/s for a channel of bandwidth $B$. For CD audio ($B = 22.05$ kHz, SNR = $2^{16}$), what is the theoretical maximum information rate? Compare with the actual CD bit rate.

$$C = 22050 \times \log_2(1 + 65536) \approx 22050 \times 16.0 = 352{,}800 \text{ bits/s per channel}$$

For stereo: $\approx 705{,}600$ bits/s.

Actual CD bit rate: 1,411,200 bits/s (stereo). This is about 2× the Shannon capacity.

Why the difference? The SNR formula assumes the entire dynamic range is used optimally. In practice, real audio signals don't fill the full dynamic range at all times, and we use linear (not optimal) quantization. The "wasted" capacity is the price of simplicity — linear PCM is trivial to implement even if it's not information-theoretically optimal. Audio codecs like FLAC exploit this redundancy, achieving ~50% compression on typical music.
